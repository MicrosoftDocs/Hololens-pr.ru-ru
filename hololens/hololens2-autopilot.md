# Windows Autopilot для HoloLens 2

Когда вы настраиваете устройства HoloLens 2 для программы Windows Autopilot, пользователи могут выполнить простую процедуру подготовки устройств из облака.

Эта программа Autopilot поддерживает режим саморазвертывания Autopilot для подготовки устройств HoloLens 2 как общих устройств в рамках вашего клиента. Режим саморазвертывания использует предварительно установленный образ и драйверы OEM устройства в процессе подготовки. Пользователь может подготовить устройство, не надевая его и не используя интерфейс первого запуска (OOBE). Дополнительные сведения о Windows Autopilot для Windows 10 см. [здесь](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot).

Когда пользователь запускает саморазвертывание Autopilot, этот процесс выполняет следующие действия:

1. Присоединение устройства к Azure Active Directory (Azure AD).

   > [!NOTE]  
   > Autopilot для HoloLens не поддерживает присоединение к Active Directory или гибридное присоединение к Azure AD.
   
1. Регистрация устройства в Microsoft Intune (или другой службе MDM) с помощью Azure AD.

1. Скачивание политик для устройств, приложений для пользователей, сертификатов и сетевых профилей.

1. Подготовка устройства.

1. Демонстрация экрана входа пользователю.

## Закрытая предварительная версия Windows Autopilot для HoloLens 2

Выполните следующие действия, чтобы настроить среду для закрытой предварительной версии:

1. Проверьте соответствие требованиям Windows Autopilot для HoloLens 2.

1. Зарегистрируйтесь в закрытой предварительной программе Windows Autopilot для HoloLens 2.

1. Проверьте участие клиента в тестировании (его регистрацию для участия в программе).

1. Зарегистрируйте свои устройства в программе Windows Autopilot.

1. Создайте группу устройств.

1. Создание профиля развертывания.

1. Проверка конфигурации ESP.

1. Настройка собственного профиля конфигурации для устройств HoloLens (известная проблема).

1. Проверьте состояние профиля устройств HoloLens.


### 1. Проверка соответствия требованиям Windows Autopilot для HoloLens 2

**Ознакомьтесь со следующими разделами статьи требований Windows Autopilot:**

- [Сетевые требования](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot-requirements#networking-requirements)  
- [Требования к лицензированию](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot-requirements#licensing-requirements)  
- [Требования к конфигурации](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot-requirements#configuration-requirements)

**Ознакомьтесь с разделом "[Требования](https://docs.microsoft.com/windows/deployment/windows-autopilot/self-deploying#requirements)" статьи о режиме саморазвертывания Windows Autopilot.** Ваша среда должна соответствовать этим требованиям и стандартным требованиям Windows Autopilot. Вам не требуется проверять в статье разделы "Пошаговое руководство" и "Проверка". Описанные ниже в этой статье процедуры содержат соответствующие действия для HoloLens. Сведения о том, как зарегистрировать устройства и настроить профили, см. в разделах [4. Регистрация устройств в Windows Autopilot](#4-register-devices-in-windows-autopilot) и [6. Создание профиля развертывания](#6-create-a-deployment-profile) этой статьи. В этих разделах приведены инструкции для HoloLens.

> [!IMPORTANT]  
> Windows Autopilot для HoloLens 2 содержит конкретные требования к операционной системе. Autopilot использует платформу Windows Holographic версии 2004 (сборка 19041.1103 или более поздняя), предварительно установленную на устройствах HoloLens. На устройствах, выпущенных до конца сентября 2020 г., предварительно установлена Windows Holographic версии 1903. Обратитесь к своему распространителю, чтобы узнать, когда вы сможете получить устройства с поддержкой Autopilot. Если вы хотите принять участие в закрытой предварительной программе, ознакомьтесь со следующими инструкциями и требованиями.

Сведения о программе Autopilot для выпусков ОС HoloLens.
- Для использования Autopilot на устройстве должно быть установлено программное обеспечение [Windows Holographic версии 2004](hololens-release-notes.md#windows-holographic-version-2004) или более поздней версии.

- Для использования Autopilot через Wi-Fi на устройстве должно быть установлено программное обеспечение [Windows Holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2) или более поздней версии. В этих сборках можно по-прежнему использовать адаптеры Ethernet. 

- В сборках [Windows Holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2) включена новая возможность управления устройством [Поставщик служб шифрования Tenantlockdown и Autopilot](hololens2-autopilot.md#tenantlockdown-csp-and-autopilot).  

Чтобы проверить версию сборки на вашем устройстве или обновить ее, подключите устройство к компьютеру с Windows 10 и запустите [расширенный помощник по восстановлению](https://www.microsoft.com/store/productId/9P74Z35SFRS8). 

**Если вы хотите попробовать предварительную версию Autopilot, перед первым запуском и подготовкой убедитесь, что устройства HoloLens соответствуют следующим требованиям:**

- Убедитесь, что ваше устройство использует Windows Holographic версии 2004 (сборка 19041.1103 или более поздняя). Если последняя версия ОС не установлена предварительно, вы должны выполнить обновление вручную, используя [расширенного помощника по восстановлению (ARC)](https://www.microsoft.com/p/advanced-recovery-companion/9p74z35sfrs8?rtc=1&activetab=pivot:overviewtab). Инструкции доступны [здесь](https://docs.microsoft.com/hololens/hololens-recovery#clean-reflash-the-device). 

- Ваши устройства должны быть зарегистрированы в Windows Autopilot. Сведения о том, как зарегистрировать устройства, см. в разделе [4. Регистрация устройств в Windows Autopilot](#4-register-devices-in-windows-autopilot). Рекомендуется, чтобы ваш торговый посредник или распространитель регистрировал устройства для вас.  

- В [Windows Holographic версии 2004](hololens-release-notes.md#windows-holographic-version-2004) необходимо подключить устройства к Интернету перед включением HoloLens и запуском процесса подготовки Autopilot. Подключите устройство к Ethernet с помощью адаптера USB-C — Ethernet, чтобы использовать проводное подключение к Интернету.

- В [Windows Holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2) устройства могут подключаться непосредственно к Wi-Fi при первом включении в режиме OOBE для обнаружения Autopilot. 

- Устройства еще не являются участниками Azure AD и не зарегистрированы в Intune (или в другой системе управления мобильными устройствами). Эти действия выполняют при саморазвертывании Autopilot. Чтобы убедиться, что очищена вся информация, связанная с устройством, проверьте страницы **Устройства** на порталах Azure AD и Intune.

- Чтобы настроить профили режима саморазвертывания Autopilot и управлять ими, убедитесь, что у вас есть доступ к [Центру администрирования Microsoft Endpoint Manager](https://endpoint.microsoft.com).


### 2. Регистрация в программе Windows Autopilot для HoloLens 2

**Чтобы принять участие в программе, вы должны зарегистрировать клиент в закрытой предварительной программе. Это включает для Autopilot элементы управления пользовательским интерфейсом Intune, предназначенные для HoloLens (другое название — MEM).** Для этого выберите [запрос приватной предварительной версии Windows Autopilot для HoloLens](https://aka.ms/APHoloLensTAP) или используйте следующий QR-код для отправки запроса.  

![QR-код Autopilot](./images/hololens-ap-qrcode.png)  

Майкрософт тестирует клиенты раз в неделю. По завершении фокус-тестирования вы получите уведомление по электронной почте. 

В этом запросе укажите следующие данные:

- Домен клиента
- Идентификатор клиента
- Количество устройств HoloLens 2, участвующих в оценке
- Количество устройств HoloLens 2, которые планируется развернуть с помощью режима саморазвертывания Autopilot

### 3. Проверка участия клиента в тестировании

Чтобы проверить участие в тестировании для программы Autopilot после отправки запроса, выполните следующие действия:

1. Войдите в [Центр администрирования Microsoft Endpoint Manager](https://endpoint.microsoft.com).

1. Выберите **Устройства** > **Windows** > **Регистрация Windows** > **Профили развертывания Windows Autopilot** > **Создать профиль**.  
   
   ![Раскрывающийся список создания профиля содержит элемент HoloLens.](./images/hololens-ap-enrollment-profiles.png)
   
   Вы увидите список, включающий **HoloLens**. Если этот параметр не отображается, свяжитесь с нами с помощью одного из параметров [обратной связи](hololens2-autopilot.md#feedback-for-autopilot).

### 4. Регистрация устройств в Windows Autopilot

На этапе подготовки зарегистрировать устройства в Windows Autopilot можно двумя основными способами: 

1. **Свяжитесь со своим распространителем или продавцом при размещении заказа, чтобы зарегистрировать свои устройства**.

   или
   
2. **Получите хэш оборудования (другое название — код оборудования) и зарегистрируйте устройство вручную**. 

Дополнительные сведения о регистрации устройств см. в статье [Добавление устройств в Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/add-devices).  

**Извлечение аппаратного хэша устройства**

Устройство может записывать свой аппаратный хэш в CSV-файл при первом запуске или позже, когда владелец устройства запускает процесс сбора журнала диагностики (описано в следующей процедуре). Как правило, владелец устройства является первым пользователем, входящим в устройство.

1. Запустите устройство HoloLens 2.

1. На устройстве одновременно нажмите кнопки включения и уменьшения звука, а затем отпустите их. Устройство собирает журналы диагностики и хэш оборудования и хранит эти данные в наборе ZIP-файлов. 

   1. Подробные сведения и обучающее видео по выполнению этой операции см. в статье [Диагностика в автономном режиме](hololens-diagnostic-logs.md#offline-diagnostics). 
   
1. Подключите устройство к компьютеру с помощью кабеля USB-C.

1. На компьютере откройте проводник. Откройте **Этот компьютер\\\<*HoloLens device name*>\\Внутреннее хранилище\\Документы** и найдите файл AutopilotDiagnostics.zip.  

   > [!NOTE]  
   > ZIP-файл может быть недоступен сразу. Если файл еще не готов, в папке "Документы" может присутствовать файл HoloLensDiagnostics.temp. Чтобы обновить список файлов, обновите окно.

1. Извлеките содержимое файла AutopilotDiagnostics.zip.

1. В извлеченных файлах найдите CSV-файл с префиксом DeviceHash в имени файла. Скопируйте этот файл на диск компьютера, где вы сможете получить к нему доступ позже.  

   > [!IMPORTANT]  
   > Данные в CSV-файле должны использовать следующий формат заголовка и строки:
   > ```
   > Device Serial Number,Windows Product ID,Hardware Hash,Group Tag,Assigned User <serialNumber>,<ProductID>,<hardwareHash>,<optionalGroupTag>,<optionalAssignedUser>
   >```

**Регистрация устройства в Windows Autopilot**

1. В Центре администрирования Microsoft Endpoint Manager откройте **Устройства** > **Windows** > **Регистрация Windows** и выберите **Устройства** > **Импорт** в разделе **Программа Windows Autopilot Deployment**.

1. В разделе **Добавление устройств Windows Autopilot** выберите CSV-файл DeviceHash, нажмите **Открыть** и выберите **Импорт**.  
   
   ![Импорт аппаратного хэша с помощью команды импорта.](./images/hololens-ap-hash-import.png)
   
1. После завершения импорта выберите **Устройства** > **Windows** > **Регистрация Windows** > **Устройства** > **Синхронизация**. Завершение процесса может занять несколько минут в зависимости от количества синхронизируемых устройств. Чтобы просмотреть зарегистрированное устройство, нажмите **Обновить**.  
   
   ![Просмотр списка устройств с помощью команд синхронизации и обновления.](./images/hololens-ap-devices-sync.png)  

### 5. Создание группы устройств

1. В Центре администрирования Microsoft Endpoint Manager выберите **Группы** > **Создать группу**.

1. Для параметра **Тип группы** выберите **Группа безопасности** и введите имя группы с описанием.

1. Для параметра **Тип членства** выберите **Назначенное** или **Динамическое устройство**.

1. Выполните одно из следующих действий.  
   
   - Если на предыдущем шаге вы выбрали **Назначенное** для параметра **Тип членства**, нажмите **Члены** и добавьте устройства Autopilot в группу. Устройства Autopilot, которые еще не зарегистрированы, указаны с помощью серийного номера устройства в качестве его имени.
   - Если на предыдущем шаге вы выбрали **Динамическое устройство** для параметра **Тип членства**, нажмите **Динамические члены устройства** и введите код для **расширенного правила**, который выглядит примерно так:
     - Если вы хотите создать группу, включающую все устройства Autopilot, введите: `(device.devicePhysicalIDs -any _ -contains "[ZTDId]")`
     - Поле тега группы Intune сопоставляется с атрибутом **OrderID** на устройствах Azure AD. Если вы хотите создать группу, включающую все устройства Autopilot с определенным тегом группы (OrderID устройства Azure AD), требуется ввести: `(device.devicePhysicalIds -any _ -eq "[OrderID]:179887111881")`
     - Если вы хотите создать группу, включающую все устройства Autopilot с определенным идентификатором заказа на покупку, введите: `(device.devicePhysicalIds -any _ -eq "[PurchaseOrderId]:76222342342")`

     > [!NOTE]  
     > Эти правила предназначены для атрибутов, являющихся уникальными для устройств Autopilot.
1. Нажмите **Сохранить**, а затем — **Создать**.

### 6. Создание профиля развертывания

1. В Центре администрирования Microsoft Endpoint Manager выберите **Устройства** > **Windows** > **Регистрация Windows** > **Профили развертывания Windows Autopilot** > **Создать профиль** > **HoloLens**.
1. Введите имя профиля и описание, а затем нажмите **Далее**.  
   
   ![Добавление имени профиля и описания](./images/hololens-ap-profile-name.png)
1. На странице **Запуск при первом включении (OOBE)** большинство параметров заранее настроены, чтобы упростить OOBE для этой оценки. При желании вы можете настроить следующие параметры:  

   - **Язык (регион)**. Выберите язык для OOBE. Рекомендуется выбрать язык из списка [поддерживаемых языков для HoloLens 2](hololens2-language-support.md).
   - **Автоматическая настройка клавиатуры**. Чтобы обеспечить соответствие клавиатуры выбранному языку, щелкните **Да**.
   - **Применить шаблон имени устройства**. Чтобы автоматически настроить имя устройства при OOBE, выберите **Да** и введите фразу и заполнители шаблона в поле **Введите имя**. Например, введите префикс и `%RAND:4%`&mdash; заполнитель для четырехзначного случайного номера.
     > [!NOTE]  
     > Если вы используете шаблон имени устройства, процесс OOBE дополнительно перезапустит устройство после применения имени устройства и до присоединения устройства к Azure AD. Этот перезапуск позволяет применить новое имя.  

   ![Настройка параметров OOBE](./images/hololens-ap-profile-oobe.png)
1. После настройки параметров нажмите **Далее**.
1. На странице **Теги области** при желании добавьте теги областей, которые нужно применить к этому профилю. Дополнительные сведения о тегах областей см. в статье [Использование тегов областей и управления доступом на основе ролей для распределенных ИТ](https://docs.microsoft.com/mem/intune/fundamentals/scope-tags.md). По завершении нажмите **Далее**.
1. На странице **Назначения** установите **Выбранные группы** для параметра **Назначить для**.
1. В разделе **ВЫБРАННЫЕ ГРУППЫ** нажмите **+ Выбрать группы для включения**.
1. В списке **Выбор групп для включения** выберите группу устройств, созданную для устройств Autopilot HoloLens и нажмите **Далее**.  
  
   Если вы хотите исключить группы, нажмите **Выбрать группы для исключения** и выберите группы, которые нужно исключить.

   ![Назначение группы устройств для профиля.](./images/hololens-ap-profile-assign-devicegroup.png)
   
1. На странице **Проверка и создание** проверьте параметры и нажмите **Создать** для создания профиля.  
   
   ![Проверка и создание](./images/hololens-ap-profile-summ.png)

### 7. Проверка конфигурации ESP

Страница состояния регистрации (ESP) отображает состояние завершенного процесса настройки устройства, выполняемого при первом входе в устройство пользователя, управляемого MDM. Убедитесь, что ваша конфигурация ESP выглядит примерно следующим образом, и проверьте правильность назначений.  

> [!div class="mx-imgBorder"]
> ![Конфигурация ESP](./images/hololens-ap-profile-settings.png)

### 8. Проверка состояния профиля устройств HoloLens

1. В Центре администрирования Microsoft Endpoint Manager выберите **Устройства** > **Windows** > **Регистрация Windows** > **Устройства**.

1. Проверьте, что устройства HoloLens указаны в списке, а состоянием их профиля является **Назначенное**.  

   > [!NOTE]  
   > Назначение профиля устройству может занять несколько минут.  

   > [!div class="mx-imgBorder"]   
   > ![Назначения устройств и профилей.](./images/hololens-ap-devices-assignments.png)

## Пользовательский интерфейс Windows Autopilot для HoloLens 2

После выполнения указанных выше инструкций пользователям HoloLens 2 потребуется выполнить следующие действия для подготовки устройств HoloLens.  

1. Для использования Autopilot требуется доступ к Интернету. Чтобы предоставить доступ к Интернету, воспользуйтесь одним из следующих способов:

    - Подключите устройство к сети Wi-Fi в режиме OOBE; дождитесь, пока устройство автоматически обнаружит Autopilot. Взаимодействие с OOBE требуется только в этот момент, после этого настройка Autopilot будет выполнена автоматически. Обратите внимание, что по умолчанию устройство HoloLens 2 ожидает обнаружения Autopilot в течение 10 секунд после обнаружения подключения к Интернету. Если в течение 10 секунд профиль Autopilot не будет обнаружен, в OOBE будет показан текст лицензионного соглашения. В этом случае перезагрузите устройство, чтобы сделать еще одну попытку обнаружить Autopilot. Обратите внимание, что ожидание обнаружения Autopilot в режиме OOBE может длиться бесконечно лишь в случае, если для устройства установлена политика TenantLockdown.
    
    - Чтобы воспользоваться проводным подключением к Интернету, подключите устройство к Ethernet с помощью адаптера USB-C — Ethernet. Дождитесь автоматической настройки Autopilot в HoloLens 2.
    
    - Чтобы воспользоваться беспроводным подключением к Интернету, подключите устройство к беспроводной сети с помощью адаптера USB-C — Wi-Fi. Дождитесь автоматической настройки Autopilot в HoloLens 2.

       > [!NOTE]
       > Использование Autopilot повлияет на [владельца устройства](security-adminless-os.md#device-owner).
   
       > [!IMPORTANT]  
       > На устройствах, пытающихся использовать сети Wi-Fi в OOBE для Autopilot, должно быть установлено программное обеспечение [Windows Holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2).
       >
       > Если устройство использует адаптер Ethernet, необходимо подключить такое устройство к сети перед первым запуском в режиме OOBE. На первом экране OOBE устройство определяет, выполняется ли подготовка в качестве устройства Autopilot. Если устройство не может подключиться к сети или вы решили не подготавливать его в качестве устройства Autopilot, вы не сможете в дальнейшем выбрать подготовку Autopilot. Вместо этого вам потребуется заново начать эту процедуру, чтобы подготовить устройство в качестве устройства Autopilot.

1. Устройство должно автоматически запустить OOBE. Не взаимодействуйте с OOBE. Откиньтесь на спинку и расслабьтесь! Позвольте HoloLens 2 определить сетевое подключение и завершить OOBE автоматически. В процессе OOBE устройство может перезапускаться. Экраны OOBE должны выглядеть примерно следующим образом.
   
   ![Шаг OOBE 1](./images/autopilot-welcome.jpg)
   ![Шаг OOBE 2](./images/autopilot-step-complete.jpg)
   ![Шаг OOBE 3](./images/autopilot-device-setup.jpg)

1. В конце OOBE вы можете войти в устройство, используя свое имя пользователя и пароль.

## CSP Tenantlockdown и Autopilot
- Устройства сохраняются в клиенте организации за счет принудительной привязки к клиенту, которая сохранятся при сбросе устройства и при установке чистого образа. Для дополнительной безопасности можно запретить создание учетных записей; такой запрет включается в процессе подготовки. 

Устройства HoloLens 2 с программным обеспечением Windows Holographic версии 20H2 или более поздней поддерживают поставщик служб шифрования (CSP) TenantLockdown. 

Поставщик служб шифрования [TenantLockdown](https://docs.microsoft.com/windows/client-management/mdm/tenantlockdown-csp) дает возможность связать HoloLens 2 с регистрацией MDM с использованием только Autopilot. Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" или "false" (установлено изначально) для HoloLens 2, это значение сохранится на устройстве при установке чистого образа, обновлении ОС и т. д. 

Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" для HoloLens 2, OOBE будет ждать успешной загрузки и применения профиля Autopilot в течение неограниченного времени после установки подключения к сети. 

Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" для HoloLens 2, следующие операции в OOBE будут запрещены: 
- Создание локального пользователя с помощью подготовки среды выполнения 
- Выполнение операции присоединения к AAD с помощью подготовки среды выполнения 
- Выбор владельца устройства в интерфейсе OOBE 

### Как выполнить настройку с помощью Intune? 
1. Создайте пользовательский профиль конфигурации устройства OMA URI и укажите значение "true" для узла RequireNetworkInOOBE, как показано ниже.
У параметра OMA-URI должно быть значение ./Vendor/MSFT/TenantLockdown/RequireNetworkInOOBE

   > [!div class="mx-imgBorder"]
   > ![Настройка блокировки клиента через OMA-URI](images/hololens-tenant-lockdown.png)

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте устройство HoloLens 2 участником группы, созданной на предыдущем шаге, и запустите синхронизацию.  

Убедитесь на портале Intune, что конфигурация устройства применена. После применения этой конфигурации устройства к устройству HoloLens 2 будут действовать параметры политики TenantLockdown.

### Как отменить применение RequireNetworkInOOBE в TenantLockdown для HoloLens 2 с помощью Intune? 
1. Удалите HoloLens 2 из группы устройств, которой ранее была назначена конфигурация устройства, созданная выше. 

1. Создайте пользовательский профиль конфигурации устройства OMA URI и укажите значение "false" для RequireNetworkInOOBE, как показано ниже. У параметра OMA-URI должно быть значение ./Vendor/MSFT/TenantLockdown/RequireNetworkInOOBE

   > [!div class="mx-imgBorder"]
   > ![Снимок экрана: установка для параметра RequireNetworkInOOBE значения "false" через OMA URI в Intune](images/hololens-tenant-lockdown-false.png)

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте устройство HoloLens 2 участником группы, созданной на предыдущем шаге, и запустите синхронизацию.

Убедитесь на портале Intune, что конфигурация устройства применена. После применения этой конфигурации устройства к устройству HoloLens 2 ограничения TenantLockdown перестанут действовать. 

### Что произойдет в процессе OOBE, если отменить назначение профиля Autopilot устройству HoloLens после установки значения "true" для параметра TenantLockdown? 
Интерфейс OOBE будет ждать загрузки профиля Autopilot в течение неограниченного времени. Появится следующее диалоговое окно. Чтобы удалить ограничения TenantLockdown, необходимо сначала зарегистрировать устройство с первоначальным клиентам, используя только Autopilot. Также необходимо отменить настройку RequireNetworkInOOBE, как описано на предыдущем шаге. После этого ограничения поставщика служб шифрования TenantLockdown будут удалены. 

![Представление на устройстве при применении политики на устройстве.](images/hololens-autopilot-lockdown.png)

## Известные проблемы

- Экземпляр приложения, основанный на контексте устройства и настроенный в Intune, не работает.
- При настройке Autopilot через Wi-Fi профиль Autopilot может не загрузиться при первом подключении к Интернету, когда отображается лицензионное соглашение и у пользователя есть возможность выполнить настройку без Autopilot. Чтобы повторно выполнить настройку с помощью Autopilot, либо переведите устройство в спящий режим и затем включите, либо перезагрузите устройство и попробуйте еще раз.

## Отзывы о программе Autopilot

Чтобы оставить отзыв или сообщить о проблемах, используйте один из следующих методов:

- Используйте приложение "Центр отзывов". Вы можете найти это приложение на компьютере, подключенном к HoloLens. В Центре отзывов выберите категорию **Управление предприятием** > **Устройство**. При предоставлении отзыва или сообщении о проблеме добавьте подробное описание. Если возможно, приложите снимки экрана и журналы.
- Если у вас возникли проблемы с Intune во время регистрации устройства или не назначается профиль Autopilot, оформите обращение в службу поддержки: [https://aka.ms/apsupport](https://aka.ms/apsupport).
- Если вы столкнулись с проблемами на устройстве HoloLens во время работы с программой Autopilot, оформите обращение в службу поддержки [https://aka.ms/hlsupport](https://aka.ms/hlsupport) с использованием [автономных журналов диагностики](hololens-diagnostic-logs.md#offline-diagnostics).

  В своем сообщение предоставьте подробное описание проблемы. Но на включайте такие данные, как снимки экрана или журналы, если сотрудники службы поддержки их не запрашивали. Такие данные могут содержать личные или персональные сведения (PII).
