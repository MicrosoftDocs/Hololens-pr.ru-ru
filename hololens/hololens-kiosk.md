# <a name="set-up-hololens-as-a-kiosk"></a>Настройка HoloLens в качестве терминала

Вы можете настроить устройство HoloLens для работы в качестве устройства ** фиксированного назначения, также называемого киоском, путем настройки устройства для работы в режиме киоска. Режим киоска ограничивает приложения (или пользователей), доступные на устройстве. Режим киоска — это удобная функция, которую можно использовать, чтобы посвятить устройство HoloLens бизнес-приложениям или использовать устройство HoloLens в демонстрационной версии приложения.

В этой статье данная статья содержит сведения об аспектах конфигурации киосков, которые являются специфическими для устройств HoloLens. Общие сведения о различных типах киосков на основе Windows и их настройке см. в рубке Настройка киосков и цифровых знаков в [настольных выпусках Windows.](https://docs.microsoft.com/windows/configuration/kiosk-methods)  

> [!IMPORTANT]  
> Режим киоска определяет, какие приложения доступны при в записи пользователя на устройство. Однако режим киоска не является методом безопасности. Это не останавливает "разрешенные" приложения от открытия другого приложения, которое не разрешено. Чтобы заблокировать открытие приложений или процессов, Защитник Windows службы управления [приложениями (WDAC)](https://docs.microsoft.com/windows/client-management/mdm/applicationcontrol-csp) для создания соответствующих политик.
>
> Дополнительные данные о службах Майкрософт, чтобы предоставить пользователям расширенный уровень безопасности, который использует HoloLens 2, подробнее о разделении и изоляции штата [— защитнике.](security-state-separation-isolation.md#defender-protections) Или узнайте, как использовать WDAC и Windows PowerShell, чтобы разрешить или заблокировать приложения на [устройствах HoloLens 2 с помощью Microsoft Intune.](https://docs.microsoft.com/mem/intune/configuration/custom-profile-hololens)

Режим киоска можно использовать в конфигурации одного или нескольких приложений, а для настройки и развертывания конфигурации киоска можно использовать один из трех процессов.

> [!IMPORTANT]  
> Удаление конфигурации нескольких приложений удаляет профили блокировки пользователя, созданные функцией назначенного доступа. Тем не менее, это не отвратит все изменения политики. Чтобы восстановить эти политики, необходимо сбросить устройство в заводские параметры.

## <a name="plan-the-kiosk-deployment"></a>Планирование развертывания киоска

При планировании киоска необходимо иметь возможность ответить на следующие вопросы. Вот некоторые решения, которые следует учитывать при чтении этой страницы, а также некоторые соображения по этим вопросам.
1. **Кто будет использовать ваш киоск, и какой тип учетной записи [(s)](hololens-identity.md) они будут использовать?** Это решение, которое, скорее всего, уже принято и не должно быть скорректировано для вашего киоска, но повлияет на то, как будет назначен киоск позже.
1. **Нужно ли иметь разные киоски на одного пользователя или группу или не включить их для некоторых?** Если это так, вы хотите создать свой киоск с помощью XML. 
1. **Сколько приложений будет в вашем киоске?** Если у вас больше 1 приложения, вам понадобится киоск с несколькими приложениями. 
1. **Какое приложение будет в вашем киоске?** Пожалуйста, используйте наш список AUMIDs ниже, чтобы добавить In-Box приложения в дополнение к вашим собственным.
1. **Как планируется развертывание киоска?** При регистрации устройства в MDM мы предлагаем использовать MDM для развертывания киоска. Если вы не используете MDM, развертывание с пакетом предварительной готовки доступно.  

### <a name="kiosk-mode-requirements"></a>Требования режима киоска

Вы можете настроить любое устройство HoloLens 2 для использования режима киоска.

> [!IMPORTANT]
> Режим киоска доступен только в том случае, если на устройстве есть Windows Holographic для бизнеса. Все устройства HoloLens 2 отгрузка с Windows Holographic для бизнеса и нет других выпусков. Все устройства HoloLens 2 могут запускать режим Киоска из окна.
>
> Устройства HoloLens (1-го поколения) необходимо обновить как с точки зрения сборки ОС, так и выпуска ОС. Дополнительные сведения об обновлении holoLens (1-го поколения) в [Windows Holographic для бизнеса.](hololens1-upgrade-enterprise.md) Чтобы обновить устройство HoloLens (1-го поколения) для использования режима киоска, сначала необходимо убедиться, что устройство работает с Windows 10, версией 1803 или более поздней версией. Если вы использовали средство восстановления устройств Windows для восстановления устройства HoloLens (1-го поколения) для сборки по умолчанию или установили последние обновления, устройство готово к настройке.

> [!IMPORTANT]  
> Чтобы защитить устройства, которые работают в режиме киоска, рассмотрите возможность добавления политик управления устройствами, которые отключили такие функции, как подключение USB. Кроме того, проверьте параметры кольца обновления, чтобы убедиться, что автоматические обновления не происходят в часы работы.

### <a name="decide-between-a-single-app-kiosk-or-a-multi-app-kiosk"></a>Решение между киоском с одним приложением или киоском с несколькими приложениями

Киоск с одним приложением запускает указанное приложение, когда пользователь вступает на устройство. Меню Пуск отключено, как и Кортана. Устройство HoloLens 2 не отвечает на жест [Начните.](hololens2-basic-usage.md#start-gesture) Устройство HoloLens (1-го поколения) не реагирует на жест [цветения.](hololens1-basic-usage.md) Так как только одно приложение может работать, пользователь не может разместить другие приложения.

В киоске с несколькими приложениями отображается меню "Пуск" при впуске пользователя на устройство. Конфигурация киоска определяет, какие приложения доступны в меню "Пуск". Вы можете использовать киоск с несколькими приложениями, чтобы предоставить пользователям простой для понимания опыт, преподнося им только то, что им нужно использовать, и удаляя то, что им не нужно использовать.

### <a name="plan-kiosk-profiles-for-users-or-groups"></a>Планирование профилей киосков для пользователей или групп

При создании xml-файла или использовании пользовательского интерфейса Intune для создания киоска необходимо учитывать, кто будет пользователем киоска. Конфигурация киоска может быть ограничена отдельной учетной записью или группами Azure AD. 

Обычно киоски включены для пользователя или группы пользователей. Однако если вы планируете написания собственного XML-киоска, вам может потребоваться рассмотреть глобальный назначенный доступ, в котором киоск применяется на уровне устройства независимо от identity. Если это обращается к вам, ознакомьтесь с дополнительными [данными о глобальных назначенных киосках доступа.](hololens-global-assigned-access-kiosk.md)

#### <a name="if-you-are-creating-an-xml-file"></a>При создании XML-файла:
-   Многие создают несколько профилей киосков и назначают каждый из них разным пользователям или группам. Например, киоск для вашей группы Azure AD с большим количеством приложений и посетитель с несколькими киосками приложений с особым приложением.
-   Конфигурация киоска будет называться **ID профиля и** иметь GUID.
-   Вы назначите этот профиль в разделе configs, указав тип пользователя и используя тот же GUID для **id DefaultProfile.**
- XML-файл можно создать, но при этом применить к устройству с помощью MDM, создав настраиваемый профиль конфигурации устройства OMA URI и применяя его к группе устройств HoloLens с использованием значения URI: ./Device/Vendor/MSFT/AssignedAccess/Configuration

#### <a name="if-you-are-creating-a-kiosk-in-intune"></a>Если вы создаете киоск в Intune.
-   Каждое устройство может получать только один профиль киоска, в противном случае это создаст конфликт и не получит конфигурации киоска вообще. 
    -   Другие типы профилей и политик, например ограничения устройств, не связанные с профилем конфигурации киоска, не конфликтуются с профилем конфигурации киоска.
-   Киоск будет включен для всех пользователей, которые являются частью типа пользовательского логотипа, это будет установлено с пользователем или группой Azure AD. 
-   После настройки киоска и **** выбора типа пользовательского логотипа (пользователи, которые могут войти в киоск) и приложения, конфигурация устройства по-прежнему должна быть назначена группе. Назначенная группа определяет, какие устройства получают конфигурацию устройства киоска, однако не взаимодействует с тем, включен ли киоск или нет. 
    - Полное обсуждение последствий назначения профилей конфигурации в Intune см. в странице Назначение профилей пользователей и устройств [в Microsoft Intune.](https://docs.microsoft.com/intune/configuration/device-profile-assign)

### <a name="mdm-step-5-single-app-ndash-deploy-a-single-app-kiosk"></a><a id="mdmsingledeploy"></a>MDM, шаг 5 (одно приложение) &ndash; Развертывание киоска с одним приложением

При использовании системы MDM можно записать устройство в MDM во время OOBE. После завершения OOBE вход на устройство будет простым.

1. Дождись завершения OOBE, загрузки и установки приложения магазина и применяйте политики. Затем перезапустите устройство.

Если вы не видите конфигурацию киоска на данном этапе, [проверьте состояние назначения.](https://docs.microsoft.com/intune/configuration/device-profile-monitor)

При следующем входе на устройство с помощью учетной **** записи, которая принадлежит типу пользовательского логотипа, приложение киоска должно автоматически запуститься.

#### <a name="optional-add-guest-access-to-the-kiosk-configuration"></a><a id="ppkioskguest"></a>Необязательный. Добавление гостевого доступа к конфигурации киоска

В разделе [ **Configs** XML-файла](https://docs.microsoft.com/windows/configuration/lock-down-windows-10-to-specific-apps#configs)можно настроить специальную группу с именем **Посетитель,** чтобы разрешить гостям использовать киоск. Когда киоск настроен на поддержку специальной группы **Посетитель,** на страницу входной страницы добавляется параметр****"Гость". **Учетная** запись Гостевой не требует пароля, и все данные, связанные с учетной записью, удаляются при выходе учетной записи.

Чтобы включить **учетную запись Guest,** добавьте следующий фрагмент в конфигурацию киоска XML:

```xml
<Configs>
  <Config>  
    <SpecialGroup Name="Visitor" />  
    <DefaultProfile Id="enter a profile ID"/>  
  </Config>  
</Configs>  
```

#### <a name="placeholder-start-layout-for-hololens"></a><a id="start-layout-for-hololens"></a>Макет старта placeholder для HoloLens

## <a name="use-the-windows-device-portal-to-set-up-a-single-app-kiosk"></a>Использование портала устройств Windows для настройка киоска с одним приложением

Чтобы настроить режим киоска с помощью портала устройств Windows, выполните следующие действия.

1. [Настройка устройства HoloLens для использования портала устройств Windows.](https://developer.microsoft.com/windows/mixed-reality/using_the_windows_device_portal#setting_up_hololens_to_use_windows_device_portal) Портал устройств — это веб-сервер на вашем устройстве HoloLens, к которому можно подключаться из веб-браузера на ПК.

    > [!CAUTION]
    > При настройках HoloLens для использования портала устройств необходимо включить режим разработчика на устройстве. Режим разработчика на устройстве с Windows Holographic для бизнеса позволяет загружать приложения боком. Однако этот параметр создает риск того, что пользователь может установить приложения, которые не были сертифицированы в Microsoft Store. Администраторы могут блокировать возможность включить режим разработчика с помощью параметра Разблокировки **ApplicationManagement/AllowDeveloper** в [CSP политики.](https://docs.microsoft.com/windows/client-management/mdm/policy-configuration-service-provider) [Подробнее о режиме разработчика.](https://docs.microsoft.com/windows/uwp/get-started/enable-your-device-for-development#developer-mode)
    
1. На компьютере подключайтесь к HoloLens с помощью [Wi-Fi](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#connecting_over_wi-fi) или [USB.](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#connecting_over_usb)

1. Выполните одно из следующих действий.
   - При первом подключении к порталу устройств Windows [создайте имя пользователя и пароль](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#creating_a_username_and_password)
   - Введите имя пользователя и пароль, которые были настроены ранее.

    > [!TIP]
    > Если вы видите в браузере ошибку сертификата, выполните [эти действия по устранению неполадок](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#security_certificate).

1. На портале устройств Windows выберите **режим киоска**.

1. Выберите **режим Включить режим киоска,** выберите приложение для запуска при запуске устройства, а затем выберите **Сохранить**.

    ![Режим полного экрана](images/kiosk.png)
1. Перезапустите HoloLens. Если страница портала устройств по-прежнему открыта, можно выбрать **перезапуск** в верхней части страницы.

> [!NOTE]
> Режим киоска можно задать с помощью API REST API портала устройств, делая POST в /api/holographic/kioskmode/settings с одним требуемым параметром строки запроса ("kioskModeEnabled" со значением "true" или "false") и одним необязательным параметром ("startupApp" со значением имени пакета). Обратите внимание, что портал устройств предназначен только для разработчиков и не должен быть включен на устройствах, не предназначенных для разработчиков. API REST может изменяться в будущих обновлениях и выпусках.

### <a name="global-assigned-access--kiosk-mode"></a>Глобальный назначен доступ — режим киоска
- Уменьшено управление удостоверениями для киоска, включив новый метод Киоск, который применяет режим Киоск на уровне системы.

Эта новая функция позволяет ИТ-администратору настраивать устройство HoloLens 2 для нескольких режимов киосков приложений, которые применимы на уровне системы, не имеют никакого отношения к удостоверению в системе и применяются ко всем, кто впишется в устройство. Дополнительные сведения об этой новой функции см. в глобальной документации по киоску доступа [HoloLens.](hololens-global-assigned-access-kiosk.md)

### <a name="automatic-launch-of-an-application-in-multiple-app-kiosk-mode"></a>Автоматический запуск приложения в режиме киоска с несколькими приложениями

### <a name="cache-azure-ad-group-membership-for-offline-kiosk"></a>Кэш Azure AD Group для автономного киоска
- Включено использование автономных киосков с группами Azure AD на срок до 60 дней.

Эта политика контролирует количество дней, когда кэш членства в группах Azure AD может использоваться для конфигураций назначенного доступа, нацеленных на группы Azure AD для подписывающихся пользователей. После того как это значение политики установлено для значения больше 0, кэш используется иначе.  

Имя: AADGroupMembershipCacheValidityInDays URI: ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays

Min — 0 дней  
Max — 60 дней 

Действия по правильному использованию этой политики: 
1. Создайте профиль конфигурации устройства для киосков, нацеленных на группы Azure AD, и назначьте его устройству HoloLens (s). 
1. Создайте настраиваемую конфигурацию устройства на основе URI OMA, которая задает это значение политики нужному числу дней (> 0) и назначьте его устройству HoloLens (s). 
    1. Значение URI должно быть вписано в текстовое поле OMA-URI как ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays
    1. Значение может быть между минутой и разрешенным максимумом.
1. Регистрация устройств HoloLens и проверка того, что обе конфигурации применяются к устройству. 
1. Позвольте пользователю Azure AD 1 войти, когда доступ к Интернету будет доступен, после успешного подтверждения членства пользователя в группе Azure AD будет создан кэш. 
1. Теперь пользователь Azure AD 1 может использовать HoloLens в автономном режиме и использовать его для режима киоска, если значение политики позволяет использовать X число дней. 
1. Шаги 4 и 5 можно повторить для любого другого пользователя Azure AD N. Ключевой момент здесь в том, что любой пользователь Azure AD должен войти на устройство с помощью Интернета, чтобы по крайней мере, как только мы могли определить, что он является членом группы Azure AD, для которой ориентирована конфигурация Киоска. 
 
> [!NOTE]
> До выполнения шага 4 для пользователя Azure AD будет выполняться сбой, указанный в "отключенных" средах.
