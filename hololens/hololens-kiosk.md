# Настройка HoloLens в качестве терминала

Устройство HoloLens можно настроить для работы в качестве устройства фиксированного назначения( терминала), настроив устройство для работы в режиме терминала. ** Режим терминала ограничивает приложения (или пользователей), доступные на устройстве. Режим терминала — это удобная функция, которую можно использовать для выделенных устройств HoloLens бизнес-приложениям или использования устройства HoloLens в демонстрационной версии приложения.

В этой статье данная статья содержит сведения об аспектах конфигурации терминала, характерных для устройств HoloLens. Общие сведения о различных типах киосков на основе Windows и их настройке см. в подстройке "Настройка киосков и цифровых вывесок" в классических выпусках [Windows.](https://docs.microsoft.com/windows/configuration/kiosk-methods)  

> [!IMPORTANT]  
> Режим терминала определяет, какие приложения доступны при под учетной записи пользователя на устройстве. Однако режим терминала не является методом обеспечения безопасности. Это не останавливает открытие "разрешенным" приложением другого приложения, которое не разрешено. Чтобы заблокировать открытие приложений или процессов, используйте CSP Защитник Windows управления приложениями [(WDAC)](https://docs.microsoft.com/windows/client-management/mdm/applicationcontrol-csp) для создания соответствующих политик.
>
> Узнайте больше о службах Майкрософт, чтобы предоставить пользователям расширенный уровень безопасности, который использует HoloLens 2, подробнее о разделении и изоляции состояния — защита [Защитника.](security-state-separation-isolation.md#defender-protections) Или узнайте, как использовать WDAC и Windows PowerShell, чтобы разрешить или заблокировать приложения на [устройствах HoloLens 2 с помощью Microsoft Intune.](https://docs.microsoft.com/mem/intune/configuration/custom-profile-hololens)

Режим терминала можно использовать в конфигурации с одним или несколькими приложениями, а также один из трех процессов для настройки и развертывания конфигурации терминала.

> [!IMPORTANT]  
> При удалении конфигурации с несколькими приложениями удаляются профили блокировки пользователей, созданные функцией назначенного доступа. Однако он не отвратит все изменения политики. Чтобы восстановить эти политики, необходимо сбросить устройство до заводских настроек.

## Планирование развертывания терминала

При планировании терминала вам потребуется ответить на следующие вопросы. Ниже учесть некоторые решения при чтении этой страницы, а также некоторые вопросы по этим вопросам.
1. **Кто будет использовать ваш терминал и какие [учетные](hololens-identity.md) записи будут использовать?** Это решение, скорее всего, уже принято и не должно быть скорректировано для вашего киоска, но повлияет на то, как терминал будет назначен позже.
1. **Требуется ли для некоторых пользователей или групп разные киоски или киоски, для некоторых из них не включены?** В этом случае вы захотите создать терминал с помощью XML. 
1. **Сколько приложений будет в вашем терминале?** Если у вас более 1 приложения, вам потребуется терминал с несколькими приложениями. 
1. **Какие приложения будут в вашем терминале?** Используйте наш список AUMID ниже, чтобы добавить любые In-Box приложения в дополнение к вашим собственным.
1. **Как вы планируете развернуть терминал?** При регистрации устройства в MDM мы рекомендуем использовать MDM для развертывания терминала. Если вы не используете MDM, развертывание с пакетом подготовка будет доступно.  

### Требования к режиму терминала

Можно настроить любое устройство HoloLens 2 для использования режима терминала.

> [!IMPORTANT]
> Режим терминала доступен, только если на устройстве есть Windows Holographic for Business. Все устройства HoloLens 2 погрузка с Windows Holographic for Business и не существует других выпусков. Все устройства HoloLens 2 могут запускать режим терминала из коробки.
>
> Устройства HoloLens (1-е поколения) необходимо обновить как с точки зрения сборки ОС, так и выпуска ОС. Ниже приводится дополнительные сведения об обновлении HoloLens (1-го поколения) до [выпуска Windows Holographic for Business.](hololens1-upgrade-enterprise.md) Чтобы обновить устройство HoloLens (1-е поколения) для использования режима терминала, сначала необходимо убедиться, что устройство работает под windows 10 версии 1803 или более поздней версии. Если вы использовали средство восстановления устройств с Windows для восстановления устройства HoloLens (1-го поколения) до сборки по умолчанию или если установлены последние обновления, устройство готово к настройке.

> [!IMPORTANT]  
> Чтобы защитить устройства, которые работают в режиме терминала, рассмотрите возможность добавления политик управления устройствами, которые отключят такие функции, как подключение ПО USB. Кроме того, проверьте параметры кругов обновления, чтобы убедиться, что автоматическое обновление не происходит в часы работы.

### Решение между терминалом с одним приложением или терминалом с несколькими приложениями

Терминал с одним приложением запускает указанное приложение, когда пользователь вошел на устройство. Меню "Пуск" отключено, как и Кортана. Устройство HoloLens 2 не реагирует на жест ["Начните".](hololens2-basic-usage.md#start-gesture) Устройство HoloLens (1-е поколения) не реагирует на жест [цветения.](hololens1-basic-usage.md) Так как может работать только одно приложение, пользователь не может разместить другие приложения.

Терминал с несколькими приложениями отображает меню "Пуск", когда пользователь входят на устройство. Конфигурация терминала определяет, какие приложения доступны в меню "Пуск". Вы можете использовать терминал с несколькими приложениями, чтобы предоставить пользователям понятные и понятные впечатления, выведя для них только необходимые им вещи и удаляя то, что им не нужно использовать.

### Планирование профилей киоска для пользователей или групп

При создании XML-файла или использовании пользовательского интерфейса Intune для создания терминала необходимо учитывать, кто будет пользователем терминала. Конфигурация терминала может быть ограничена отдельной учетной записью или группами Azure AD. 

Обычно киоски включены для пользователя или группы пользователей. Однако если вы планируете написать собственный терминал XML, можно рассмотреть возможность глобального назначенного доступа, в котором терминал применяется на уровне устройства независимо от удостоверения. Если это не так, ознакомьтесь с дополнительными [данными о глобальных терминалах с назначенным доступом.](hololens-global-assigned-access-kiosk.md)

#### При создании XML-файла:
-   Многие создают несколько профилей терминала и назначают их разным пользователям или группам. Например, киоск для группы Azure AD с большим количеством приложений и посетитель с несколькими киосками приложений с единым приложением.
-   Конфигурация терминала будет называться **ИД профиля** и иметь GUID.
-   Вы назначите этот профиль в разделе "Configs", указав тип пользователя и используя тот же GUID для **ИД DefaultProfile.**
- XML-файл можно создать, но применить к устройству с помощью MDM, создав пользовательский профиль конфигурации устройства OMA URI и применив его к группе устройств HoloLens, используя значение URI: ./Device/Vendor/MSFT/AssignedAccess/Configuration

#### Если вы создаете терминал в Intune.
-   Каждое устройство может получить только один профиль терминала, в противном случае это создаст конфликт и вообще не получит конфигураций терминала. 
    -   Другие типы профилей и политик, например ограничения устройств, не связанные с профилем конфигурации терминала, не конфликтуют с профилем конфигурации терминала.
-   Терминал будет включен для всех пользователей, которые являются частью типа входа пользователя, он будет установлен с пользователем или группой Azure AD. 
-   После настройки терминала и **** выбора типа входа пользователя (пользователи, которые могут войти в терминал) и выбора приложений конфигурация устройства все равно должна быть назначена группе. Группа "Назначенная" определяет, какие устройства получают конфигурацию устройства киоска, но не взаимодействуют с ними, если терминал включен или нет. 
    - Полное обсуждение последствий назначения профилей конфигурации в Intune см. в подразтроек "Назначение профилей пользователей и [устройств" в Microsoft Intune.](https://docs.microsoft.com/intune/configuration/device-profile-assign)

### <a id="mdmsingledeploy"></a>MDM, шаг 5 (одно приложение) &ndash; Развертывание терминала с одним приложением

При использовании системы MDM устройство можно зарегистрировать в MDM во время OOBE. После завершения OOBE вход на устройство будет простым.

Во время OOBE выполните следующие действия:

1. Во sign in by using the account that you specified in the kiosk configuration profile.
1. Зарегистрив устройство. Убедитесь, что устройство добавлено в группу, для которую назначен профиль конфигурации терминала.
1. Дождись завершения OOBE, скачивания и установки приложения Магазина и применяемых политик. Затем перезапустите устройство.

При следующем входе на устройство приложение терминала должно автоматически запуститься.

Если вы не видите конфигурацию терминала на этом этапе, проверьте [состояние назначения.](https://docs.microsoft.com/intune/configuration/device-profile-monitor)

### <a id="mdmmultideploy"></a>MDM, шаг 5 (с несколькими приложениями) &ndash; Развертывание терминала с несколькими приложениями

При использовании системы MDM можно присоединить устройство к вашему клиенту Azure AD и зарегистрировать устройство в MDM во время OOBE. При необходимости предоплату сведений о регистрации для пользователей, чтобы они могли их получить во время процесса OOBE.

> [!NOTE]  
> Если профиль конфигурации терминала назначен группе, в которую входят пользователи, убедитесь, что одна из этих учетных записей пользователей является первой учетной записью, которая входит в устройство.

Во время OOBE выполните следующие действия:

1. Во sign in by using the account that belongs to the **User logon type** group.
1. Зарегистрив устройство.
1. Дождись скачивания и установки всех приложений, которые являются частью профиля конфигурации терминала. Кроме того, подождите, пока политики будут применены.  
1. После завершения запуска при ветвях можно установить дополнительные приложения из Microsoft Store или путем загрузки нео том же самого приложения. [Необходимые приложения](https://docs.microsoft.com/mem/intune/apps/apps-deploy#assign-an-app) для группы, к которой принадлежит устройство, устанавливаются автоматически.
1. После завершения установки перезапустите устройство.

При следующем входе в устройство с использованием учетной записи, которая принадлежит к типу **пользователя,** приложение терминала должно автоматически запуститься.

Если вы не видите конфигурацию терминала на этом этапе, проверьте [состояние назначения.](https://docs.microsoft.com/intune/configuration/device-profile-monitor)

### <a id="ppkioskconfig"></a>Пакет предоставления, шаг &ndash; 1. Создание XML-файла конфигурации терминала

Следуйте общим инструкциям по созданию XML-файла конфигурации терминала для [настольного компьютера с Windows,](https://docs.microsoft.com/windows/configuration/lock-down-windows-10-to-specific-apps#create-xml-file)за исключением следующих случаев:

- Не включайте классические приложения для Windows (Win32). HoloLens не поддерживает эти приложения.
- Используйте [XML-разметку макета "Начните"](#start-layout-for-hololens) для HoloLens.
- Необязательно: добавление гостевого доступа к конфигурации терминала

#### <a id="ppkioskguest"></a>Необязательно: добавление гостевого доступа к конфигурации терминала

В разделе [ **"Конфигурации"** XML-файла](https://docs.microsoft.com/windows/configuration/lock-down-windows-10-to-specific-apps#configs)можно настроить специальную группу с именем **"Посетитель",** чтобы разрешить гостям использовать терминал. Если в терминале настроена **** поддержка специальной группы "Посетитель", на страницу входов добавляется параметр****"Гость". **Гостевой** учетной записи не требуется пароль, и все данные, связанные с этой учетной записью, удаляются при выходе учетной записи.

Чтобы включить **гостевую** учетную запись, добавьте следующий фрагмент кода в XML конфигурации терминала:

```xml
<Configs>
  <Config>  
    <SpecialGroup Name="Visitor" />  
    <DefaultProfile Id="enter a profile ID"/>  
  </Config>  
</Configs>  
```

#### <a id="start-layout-for-hololens"></a>Макет замещего начала для HoloLens

Если для настройки [терминала](#use-a-provisioning-package-to-set-up-a-single-app-or-multi-app-kiosk) с несколькими приложениями используется пакет предоставления, для этой процедуры требуется макет "Начните". Настройка макета начала не поддерживается в Windows Holographic for Business. Поэтому необходимо использовать макет-заметель "Начните".

> [!NOTE]  
> Так как киоск с одним приложением запускает приложение киоска, когда пользователь вступает в приложение, оно не использует меню "Пуск" и не требует макета меню "Пуск".

> [!NOTE]  
> Если вы используете [MDM](#use-microsoft-intune-or-other-mdm-to-set-up-a-single-app-or-multi-app-kiosk) для настроить терминал с несколькими приложениями, при желании можно использовать макет "Начните". Дополнительные сведения см. в файле макета "Начните с места" для [MDM (Intune и других)](#start-layout-file-for-mdm-intune-and-others).

Для макета "Начните" добавьте следующий раздел **StartLayout** в XML-файл подготовка терминала:

```xml
<!-- This section is required for parity with Desktop Assigned Access. It is not currently used on HoloLens -->
            <StartLayout>
                <![CDATA[<LayoutModificationTemplate xmlns:defaultlayout="http://schemas.microsoft.com/Start/2014/FullDefaultLayout" xmlns:start="http://schemas.microsoft.com/Start/2014/StartLayout" Version="1" xmlns="http://schemas.microsoft.com/Start/2014/LayoutModification">
                      <LayoutOptions StartTileGroupCellWidth="6" />
                      <DefaultLayoutOverride>
                        <StartLayoutCollection>
                          <defaultlayout:StartLayout GroupCellWidth="6">
                            <start:Group Name="">
                              <start:Tile Size="2x2" Column="0" Row="0" AppUserModelID="placeholderpackagename_kzf8qxf38zg5c!App" />
                            </start:Group>
                          </defaultlayout:StartLayout>
                        </StartLayoutCollection>
                      </DefaultLayoutOverride>
                    </LayoutModificationTemplate>
                ]]>
            </StartLayout>
            <!-- This section is required for parity with Desktop Assigned Access. It is not currently used on HoloLens -->
```

#### <a id="start-layout-file-for-mdm-intune-and-others"></a>Файл макета "Начните" для MDM (Intune и другие)

Сохраните следующий пример в XML-файле. Этот файл можно использовать при настройке терминала с несколькими приложениями в Microsoft Intune (или в другой службе MDM, которая предоставляет профиль терминала).

> [!NOTE]
> Если для настройки терминала в службе MDM необходимо использовать настраиваемый параметр и полную конфигурацию XML, воспользуйтесь инструкциями по макету для пакета [наполнения.](#start-layout-for-hololens)

```xml
<LayoutModificationTemplate
    xmlns="http://schemas.microsoft.com/Start/2014/LayoutModification"
    xmlns:defaultlayout="http://schemas.microsoft.com/Start/2014/FullDefaultLayout"
    xmlns:start="http://schemas.microsoft.com/Start/2014/StartLayout"
    Version="1">
  <RequiredStartGroupsCollection>
    <RequiredStartGroups>
      <AppendGroup Name="">
        <start:Tile Size="2x2" Column="0" Row="0" AppUserModelID="placeholderpackagename_kzf8qxf38zg5c!App" />
      </AppendGroup>
    </RequiredStartGroups>
  </RequiredStartGroupsCollection>
 </LayoutModificationTemplate>
```

## Настройка терминала с одним приложением с помощью портала устройств с Windows

Чтобы настроить режим терминала с помощью портала устройств с Windows, выполните следующие действия.

1. [Настройка устройства HoloLens для использования портала устройств Windows.](https://developer.microsoft.com/windows/mixed-reality/using_the_windows_device_portal#setting_up_hololens_to_use_windows_device_portal) Портал устройств — это веб-сервер на вашем устройстве HoloLens, к которому можно подключаться из веб-браузера на ПК.

    > [!CAUTION]
    > При настройках HoloLens для использования портала устройств необходимо включить режим разработчика на устройстве. Режим разработчика на устройстве с Windows Holographic for Business позволяет загружать неоконтружители приложений. Однако этот параметр создает риск того, что пользователь сможет установить приложения, не сертифицированные Microsoft Store. Администраторы могут заблокировать возможность включить режим разработчика с помощью параметра **ApplicationManagement/AllowDeveloper Unlock** в [CSP политики.](https://docs.microsoft.com/windows/client-management/mdm/policy-configuration-service-provider) [Подробнее о режиме разработчика.](https://docs.microsoft.com/windows/uwp/get-started/enable-your-device-for-development#developer-mode)
    
1. На компьютере подключите Устройство HoloLens с помощью [Wi-Fi](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#connecting_over_wi-fi) или [USB.](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#connecting_over_usb)

1. Выполните одно из следующих действий.
   - При первом подключении к порталу устройств Windows создайте имя пользователя [и пароль](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#creating_a_username_and_password)
   - Введите имя пользователя и пароль, которые вы настроили ранее.

    > [!TIP]
    > Если вы видите в браузере ошибку сертификата, выполните [эти действия по устранению неполадок](https://developer.microsoft.com/windows/mixed-reality/Using_the_Windows_Device_Portal#security_certificate).

1. На портале устройств с Windows выберите **режим терминала.**

1. Выберите **"Включить режим**терминала", выберите приложение, которое будет запускаться при запуске устройства, а затем выберите **"Сохранить".**

    ![Режим полного экрана](images/kiosk.png)
1. Перезапустите HoloLens. Если страница портала устройств по-прежнему **** открыта, можно выбрать "Перезапустить" в верхней части страницы.

> [!NOTE]
> Режим терминала можно настроить с помощью REST API портала устройств, выдав запрос POST для /api/holographic/kioskmode/settings с одним обязательным параметром строки запроса ("kioskModeEnabled" со значением "true" или "false") и одним необязательным параметром ("startupApp" со значением имени пакета). Помните, что портал устройств предназначен только для разработчиков и не должен быть включен на устройствах, не предназначенных для разработчиков. API REST может изменяться в будущих обновлениях и выпусках.

### Кэшное членство в группе Azure AD для автономной терминала
- Включены автономные киоски для использования с группами Azure AD в течение 60 дней.

Эта политика контролирует, сколько дней можно использовать кэш членства в группах Azure AD для конфигураций с assigned Access, нацеленных на группы Azure AD для во входе пользователя. Если для этого значения политики установлено значение больше 0, то в противном случае используется кэш.  

Имя: Значение URI AADGroupMembershipCacheValidityInDays: ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays

Min — 0 дней  
Максимальное — 60 дней 

Действия для правильного использования этой политики: 
1. Создайте профиль конфигурации устройства для терминала, предназначенного для групп Azure AD, и назначьте его устройствам HoloLens. 
1. Создайте настраиваемую конфигурацию устройства на основе OMA URI, которая задает для этого значения политики необходимое количество дней (> 0) и назначьте его устройствам HoloLens. 
    1. Значение URI должно быть введено в текстовом поле OMA-URI как ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays
    1. Допустимые значения: min/max.
1. Зарегистрировать устройства HoloLens и убедиться, что обе конфигурации применяются к устройству. 
1. Разрешив вход пользователю Azure AD 1, если интернет-доступ доступен, после успешного подтверждения членства пользователя в группе Azure AD кэш будет создан. 
1. Теперь пользователь Azure AD 1 может отсоедидить HoloLens и использовать его для режима терминала, если значение политики допускает X число дней. 
1. Шаги 4 и 5 могут повторяться для любого другого пользователя Azure AD N. Ключевой момент здесь в том, что любой пользователь Azure AD должен войти на устройство с помощью Интернета, чтобы хотя бы один раз определить, что он является членом группы Azure AD, для которой предназначена конфигурация терминала. 
 
> [!NOTE]
> До выполнения шага 4 для пользователя Azure AD будет выполняться сбой, упомянутый в "отключенных" средах.
