# Настройка HoloLens с помощью пакета подготовка

[Подготовка Windows упрощает](https://docs.microsoft.com/windows/configuration/provisioning-packages/provisioning-packages) настройку устройств конечных пользователей без создания образа для ИТ-администраторов. Конструктор конфигураций Windows — это инструмент для настройки образов и параметров времени работы, которые затем встроены в пакеты подготовка.

Некоторые конфигурации HoloLens, которые можно применить в пакете предоставления, включают следующие:

- Обновление до [Windows Holographic for Business](hololens1-upgrade-enterprise.md)
- Настройка локальной учетной записи
- Настройка подключения к сети Wi-Fi
- Применить сертификаты к устройству
- Включить режим разработчика
- Настройте режим терминала, следуя нашим [подробным инструкциям.](hololens-kiosk.md#use-a-provisioning-package-to-set-up-a-single-app-or-multi-app-kiosk)

## Мастер пакета подготовка HoloLens

Мастер HoloLens помогает настроить следующие параметры в пакете предоставления:

- Обновление до корпоративного выпуска

    > [!NOTE]
    > Этот способ следует использовать только для устройств HoloLens 1-го поколения. Параметры в пакете предоставления применяются только в том случае, если пакет включает лицензию на обновление выпуска до Windows Holographic for Business или если устройство уже обновлено до [Windows Holographic for Business.](hololens1-upgrade-enterprise.md)

- Настройка первого опыта работы HoloLens (OOBE)
- Настройка сети Wi-Fi сети
- Регистрация устройства в Azure Active Directory или создание локальной учетной записи
- Добавление сертификатов
- Включить режим разработчика
- Настройте режим терминала, следуя [подробным инструкциям).](hololens-kiosk.md#use-a-provisioning-package-to-set-up-a-single-app-or-multi-app-kiosk)

> [!WARNING]
> Необходимо запустить конструктор конфигураций Windows в Windows 10 для настройки регистрации Azure Active Directory с помощью любого из мастеров.

Пакеты предоставления могут включать инструкции и политики управления, настраиваемые сетевые подключения и политики и другие.

> [!TIP]
> Используйте мастер настольных компьютеров, чтобы создать пакет с общими параметрами, затем переключитесь в расширенный редактор, чтобы добавить другие параметры, приложения, политики и т. д.

### 2. Создание пакета подготовка

Для создания пакета подготовки используйте конструктор конфигураций Windows.

1. Откройте конструктор конфигураций Windows (по умолчанию %windir%\Program Files (x86)\Windows Kits\10\Assessment and Deployment Kit\Imaging and Configuration Designer\x86\ICD.exe).

2. Выберите **"Подготовка устройств HoloLens".**

   ![Параметры запуска ICD](images/icd-create-options-1703.png)

3. Назовите проект и выберите **"Готово".**

4. Прочитайте инструкции на странице **"Начало работы"** и выберите **"Далее".** На страницах для предоставления компьютеров вы можете пройти следующие этапы.
  
> [!IMPORTANT]
> При создании пакета подготовки вы можете включить конфиденциальную информацию в файлы проекта и в файл пакета подготовки (PPKG). Несмотря на то что у вас есть возможность шифрования PPKG-файла, файлы проекта не шифруются. Файлы проекта следует хранить в надежном месте, если они больше не требуются, их необходимо удалять.

### 3. Создание пакета подготовка для HoloLens с помощью расширенных настойки

> [!NOTE]
> Пакеты предоставления, которые вы создаете в **advanced provisioning,** не должны включать лицензию на обновление выпуска до Windows Holographic for Business, чтобы она была применима к HoloLens (1-е поколения). [Дополнительные см. в Windows Holographic for Business для HoloLens (1-е поколения).](hololens1-upgrade-enterprise.md)

1. На начальной странице конструктора конфигураций Windows нажмите **Расширенная подготовка**.
2. В окне **Введите сведения о проекте** укажите имя и расположение своего проекта. При желании вы также можете ввести краткое описание своего проекта.

3. Выберите **Далее**.

4. В **окне "Выбор параметров** для просмотра и настройки" выберите **Windows 10 Holographic**и затем выберите **"Далее".**

5. Select **Finish**.

6. **Развертывите** параметры времени работы и настройте пакет с помощью любого из параметров, описанных далее [в этой статье.](#what-you-can-configure)

    > [!IMPORTANT]
    > (Только для Windows 10 версии 1607) При создании локальной учетной записи в пакете предоставления **** необходимо изменить пароль с помощью приложения "Параметры" каждые 42 дня. Если пароль не меняется в течение этого периода, учетная запись может быть заблокирована и стать недоступной для входа. Если учетная запись пользователя заблокирована, необходимо [выполнить полное восстановление устройства](https://developer.microsoft.com/windows/mixed-reality/reset_or_recover_your_hololens#perform_a_full_device_recovery).

7. Выберите **"Сохранить**  >  **файл"**.

8. Прочитайте предупреждение о том, что файлы проекта могут содержать конфиденциальную информацию, и выберите **"ОК".**

    > [!IMPORTANT]
    > При создании пакета подготовки вы можете включить конфиденциальную информацию в файлы проекта и в файл пакета подготовки (PPKG). Несмотря на то что у вас есть возможность шифрования PPKG-файла, файлы проекта не шифруются. Файлы проекта следует хранить в надежном месте, если они больше не требуются, их необходимо удалять.
    
9. Выберите **пакет для**  >  **экспорта.**

10. Смена **владельца** на **ИТ-администратора.** Это устанавливает приоритет этого пакета подготовка выше, чем пакеты, примененные к этому устройству из других источников. Выберите **Далее**.

11. Задайте значение параметра **Версия пакета**.

    > [!TIP]
    > Можно внести изменения в существующие пакеты и изменить номер версии для обновления ранее примененных пакетов.

12. On the **Select security details for the provisioning package,** select **Next**.

    > [!WARNING]
    > Если зашифровать пакет подготовки, при подготовке устройства HoloLens возникнет ошибка.  

13. Выберите пункт "Далее", чтобы указать выходное расположение, куда должен перейти пакет подготовка после его построения. **** По умолчанию конструктор конфигураций Windows использует в качестве расположения выходных данных папку проекта.

    При желании можно выбрать **"Обзор",** чтобы изменить расположение выходных данных по умолчанию.

14. Выберите **Далее**.

15. Выберите **"Сборка",** чтобы начать сборку пакета. Сведения о проекте отображаются на странице сборки, а индикатор выполнения указывает состояние сборки.

16. После завершения сборки выберите **"Готово".**

## Применение пакета подготовка к HoloLens во время установки

Устройства HoloLens 2 в Windows Holographic версии 2004 или сборки [19041.1103](hololens-release-notes.md#windows-holographic-version-2004) или более поздней версии могут использовать USB-накопитель для применения пакета подготовка. Просто скопируйте PPKG-файл в корневой файл USB-накопителя. Пакеты подготовка будут применяться только в том случае, если они есть в корне USB-накопителя. Несколько пакетов предоставления будут применены последовательно.

Устройства HoloLens 2 в [Windows Holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2) или более поздней версии имеют более новые функции, которые упрощают этот процесс, делая его автоматическим. Просмотрите следующие разделы:

- [Подготовка к автоматическому запуску с USB](hololens-provisioning.md#auto-launch-provisioning-from-usb)
- [Пакеты автоматической подтверждения в OOBE](hololens-provisioning.md#auto-confirm-provisioning-packages-in-oobe)
- [Автоматическая подготовка без использования пользовательского интерфейса](hololens-provisioning.md#automatic-provisioning-without-using-ui)

1. С помощью USB-кабеля подключите устройство к компьютеру (или USB-накопителю для HoloLens 2, как упоминалось выше), а затем запустите устройство. Не пережимай страницу **"Первый взаимодействуемый момент"** при первом внося в OOBE.   
    - На HoloLens (1-е поколения) эта страница содержит синее поле. 
    - На HoloLens 2 эта страница содержит hummingbird.

2. Одновременно и кратковременно нажмите и отпустите кнопки **уменьшения громкости** и **питания**. 

3. HoloLens показан как устройство в проводнике на компьютере.

4. В проводнике перетащите пакет подготовки (.ppkg) в раздел хранения данных на устройстве.

5. Кратковременно нажмите **** и отпустите кнопки "Отключение громкости" и **"Питание"** одновременно на странице "Первый **взаимодействуемый момент** запуска при первом ветве при первом ветви при первом вставок".

6. Устройство спросит вас, доверяете ли вы пакету, и хотите ли вы применить его. Подтвердите, что доверяете пакету.

7. Вы увидите, успешно ли прошло применение пакета. Если произойдет сбой, можно исправить пакет и повторить попытку. Если применение выполнено успешно, продолжите процедуру запуска при первом включении устройства.

> [!NOTE]
> Если устройство было приобретено до августа 2016 г., вам потребуется войти на устройство с помощью учетной записи Майкрософт, получить последнее обновление операционной системы, а затем сбросить операционную систему, чтобы применить пакет предоставления.

### Подготовка к автоматическому запуску с USB

- Автоматизированные процессы, позволяющие уменьшить взаимодействие с пользователем, когда USB-накопители с пакетами предоставления используются во время OOBE.

Перед этим выпуском пользователям приходилось вручную запускать экран подготовка во время запуска при запуске при запуске, чтобы сделать это с помощью сочетания кнопок. Теперь пользователи могут пропустить сочетание кнопок с помощью пакета подготовка на USB-накопителе. 

1. Подключите USB-накопитель с пакетом подготовка во время первого взаимодействия при первом включении при первом включении.
1. Когда устройство будет готово к подготовию, оно автоматически откроет запрос со страницей предоставления. 

Примечание. Если USB-накопитель остается подключенным во время загрузки устройства, при OOBE будет выводиться существующее устройство USB-хранилища, а также будут наблюдать за подключением дополнительных usb-накопителей.

Прочитайте о [применении пакетов подготовка во время OOBE.](hololens-provisioning.md#apply-a-provisioning-package-to-hololens-during-setup)

### Пакеты автоматической подтверждения в OOBE
- Автоматический процесс, позволяющий уменьшить взаимодействие с пользователем, при отобрализации страницы пакета подготовка автоматически применяет все перечисленные пакеты.

После запуска главного экрана OOBE отсчитывает 10 секунд до автоматического начала применения всех пакетов. Пользователи по-прежнему могут подтверждать или отменять их в течение этих 10 секунд после проверки ожидаемых пакетов.

### Автоматическая подготовка без использования пользовательского интерфейса
- Объединенные автоматические процессы для уменьшения взаимодействия с устройством для предоставления. 

Объединяя автоматический запуск для предоставления с USB-устройств и автоматическое подтверждение пакетов, пользователь может автоматически подчищать устройства HoloLens 2, не используя пользовательский интерфейс устройства или даже не пользуясь устройством. Вы можете продолжать использовать один и тот же USB-накопитель и пакеты подготовка для нескольких устройств. Это полезно для одновременного развертывания нескольких устройств в одной области. 

1. [Создание пакета подготовка с](hololens-provisioning.md) помощью [конструктора конфигураций Windows](https://www.microsoft.com/store/productId/9NBLGGH4TX22). 
1. Скопируйте пакет на USB-накопитель.
1. [Flash your HoloLens 2](hololens-insider.md#ffu-download-and-flash-directions) to [19041.1361 or newer build](https://aka.ms/hololens2previewdownload). 
1. После [завершения работы устройства Advanced Recovery Companion](https://www.microsoft.com/store/productId/9P74Z35SFRS8) устройство отключает кабель USB-C. 
1. Подключите USB-накопитель к устройству.
1. При загрузке устройства HoloLens 2 в режиме запуска при запуске при впуске устройство автоматически обнаружит пакет подготовка на USB-накопителе и запустит страницу.
1. Через 10 секунд устройство автоматически примеает пакет предоставления. 

Теперь устройство настроено и отобразит экран "Подготовка успешно".

## Применение пакета подготовка к HoloLens после установки

> [!NOTE]
> Эти действия применимы только к Windows 10 версии 1809.

На компьютере выполните следующие действия.
1. Создайте пакет предоставления, как описано в описании создания пакета подготовка для HoloLens с помощью мастера [HoloLens.](hololens-provisioning.md)
2. Подключите устройство HoloLens к компьютеру с помощью USB-кабеля. HoloLens показан как устройство в проводнике на компьютере.
3. Перетащите пакет предоставления в папку Documents на HoloLens.

На holoLens выполните следующие действия:
1. Go to **Settings**  >  **Accounts**  >  **Access work or school**. 
2. В **связанных параметрах**выберите **"Добавить или удалить пакет предоставления".**
3. На следующей странице выберите "Добавить **пакет",** чтобы запустить выбор файлов, и выберите пакет предоставления. Если папка пуста, убедитесь, что выбрано это **устройство** и выбраны **документы.**

После установки пакета он появляется в списке **установленных пакетов.** Чтобы просмотреть сведения о пакете или удалить пакет с устройства, выберите указанный пакет.

## Что можно настроить

Пакеты подготовки используют поставщики услуг конфигурации (CSP). Если вы не знакомы с понятием «поставщик службы конфигурации» (CSP), сначала ознакомьтесь с разделом [Вводные сведения о поставщиках службы конфигурации (CSP) для ИТ-профессионалов](https://docs.microsoft.com/windows/configuration/provisioning-packages/how-it-pros-can-use-configuration-service-providers).

В конструкторе конфигураций Windows при создании пакета подготовки для Windows Holographic параметры в разделе **Доступные настройки** основаны на [поставщиках CSP, которые поддерживаются в среде Windows Holographic](https://docs.microsoft.com/windows/client-management/mdm/configuration-service-provider-reference#csps-supported-in-hololens-devices). В следующей таблице рассматриваются параметры, которые можно настроить для HoloLens.

![Общие параметры среды выполнения для HoloLens](images/icd-settings.png)

| Параметр | Описание |
| --- | --- |
| **Сертификаты** | Развертывание сертификата на HoloLens.  |
| **ConnectivityProfiles** | Развертывание профиля Wi-Fi на HoloLens.   |
| **EditionUpgrade** | [Обновление до Windows Holographic for Business.](hololens1-upgrade-enterprise.md)  |
| **Политики** | Разрешение или запрет режима разработчика на устройстве HoloLens. [Политики, поддерживаемые Windows Holographic for Business](https://docs.microsoft.com/windows/client-management/mdm/policy-configuration-service-provider#hololenspolicies) |

## Установка приложения с помощью пакета подготовка

Приложения можно устанавливать с помощью пакетов подготовка на устройствах HoloLens 2. Это позволяет легко повторно использовать пакет, который можно использовать для распространения приложений. Ознакомьтесь с полными инструкциями по [развертыванию приложений с помощью пакетов наполнения.](app-deploy-provisioning-package.md)  

> [!NOTE]
> HoloLens (1-е поколения) имеет ограниченную поддержку установки приложений **(UniversalAppInstall)** с помощью пакета подготовка. Устройства HoloLens (1-е поколения) поддерживают установку приложения через PPKG только во время OOBE и только при установке контекста пользователя.
