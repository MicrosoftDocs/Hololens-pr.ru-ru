# Сбор и использование сведений диагностики с устройств HoloLens

Пользователи HoloLens и администраторы могут выбрать один из четырех различных способов сбора диагностических данных от HoloLens.

- Приложение центра отзывов
- Поставщик служб конфигурации DiagnosticLog
- Приложение «Параметры»

> [!IMPORTANT]  
> Журналы диагностики устройств содержат личные сведения (PII), такие как сведения о процессах и приложениях, запускаемых пользователем во время обычных операций. Если несколько пользователей совместно используют устройство HoloLens (например, пользователи входят в одно и то же устройство, используя разные учетные записи Microsoft Azure Active Directory (AAD)), журналы диагностики могут содержать сведения о личных данных, которые относятся к нескольким пользователям. Дополнительные сведения можно найти в [заявлении о конфиденциальности Microsoft](https://privacy.microsoft.com/privacystatement).

## Центр отзывов

Пользователь HoloLens может использовать классическое приложение центра отзывов Microsoft для отправки диагностических сведений в службу поддержки Майкрософт. Подробные сведения и инструкции можно найти в разделе [Поделитесь отзывом](hololens-feedback.md).  

> [!NOTE]  
> **Коммерческие и корпоративные пользователи:** Если вы используете приложение центра обратной связи, чтобы сообщить о проблеме, связанной с MDM, предоставлением или любым другим аспектом управления устройствами, измените категорию приложения на категорию устройства **управления предприятием**  >  **Device category**.

### Предварительные требования

- Устройство подключено к сети.
- Приложение центра отзывов доступно на настольном компьютере пользователя, и пользователь может отправлять файлы в Microsoft Cloud.

### Расположение, доступ и хранение данных

Благодаря согласию за использование центра отзывов пользователь явным образом отправляется на хранение и использование данных (как определено этим соглашением).

Центр отзывов предоставляет пользователю два места для хранения диагностических сведений.

- **Microsoft Cloud**. Данные, которые пользователь отправляет с помощью приложения центра обратной связи, хранятся в течение количества дней, которые соответствуют требованиям к конфиденциальности нового поколения (NGP). Сотрудники Майкрософт могут использовать для доступа к данным в течение этого периода средство просмотра, совместимое с NGP.
   > [!NOTE]  
   > Эти требования применяются к данным во всех категориях центра отзывов.

- **Устройство HoloLens**. При архивации отчета в центре отзывов пользователь может выбрать параметр **сохранить локальную копию диагностики и вложений, созданных при обратном отзыве**. Если пользователь выбирает этот параметр, центр обратной связи сохраняет копию диагностической информации на устройстве HoloLens. Эти сведения остаются доступными для пользователя (или любого, кто использует эту учетную запись для входа в HoloLens). Для удаления этих данных пользователю должны быть предоставлены разрешения **владельца устройства** или **администратора** на устройстве. Пользователь с необходимыми разрешениями может войти в центр отзывов, выбрать **Параметры**  >  **Просмотреть журналы диагностики**и удалить данные.

## Средство устранения неполадок с параметрами

Пользователь HoloLens может использовать приложение "Параметры" на устройстве для устранения неполадок и сбора диагностических данных. Для этого выполните следующие действия:

1. Откройте приложение параметры и выберите **Обновление &**  >  Страница**устранения неполадок** с безопасностью.
1. Выберите нужную область и нажмите кнопку **начать**.
1. Воспроизводить проблемы.
1. После воспроизведения проблемы вернитесь в раздел Параметры и нажмите кнопку **стоп**.

### Предварительные требования

- Приложение "Параметры" установлено на устройстве и доступно пользователю.

### Расположение, доступ и хранение данных

Так как пользователь запускает сбор данных, пользователь неявно отправляется в хранилище диагностической информации. Получить доступ к данным смогут только пользователи, у которых есть учетные данные.

Диагностическая информация хранится на устройстве. Если устройство подключено к компьютеру пользователя, сведения о нем также находятся на компьютере в следующем файле:

> Этот PC\\ \<*HoloLens device name*> \\Internal Storage\\Documents\\Trace \<*ddmmyyhhmmss*> . ETL

> [!NOTE]  
> В этом пути и имени файла \<*HoloLens device name*> представляет имя устройства HoloLens и \<*ddmmyyhhmmss*> представляет дату и время создания файла.

Диагностические данные остаются в этих папках до тех пор, пока пользователь не удалит их.

## Поставщик служб конфигурации DiagnosticLog

В среде управления мобильными устройствами (MDM) ИТ-администратор может использовать [поставщик услуг настройки DiagnosticLog (CSP)](https://docs.microsoft.com/windows/client-management/mdm/diagnosticlog-csp) для настройки параметров диагностики на зарегистрированных устройствах HoloLens. Администратор может настроить эти параметры для сбора журналов с зарегистрированных устройств.

### Предварительные требования

- Устройство подключено к сети.
- Устройство регистрируется в среде MDM, которая поддерживает поставщика службы криптографии DiagnosticLog.

### Расположение, доступ и хранение данных

Так как устройство является частью управляемой среды, пользователь неявно получает доступ к диагностическим сведениям для администратора.

ИТ – администратор использует поставщика службы криптографии DiagnosticLog для настройки политик хранения, хранения и доступа к данным, в том числе политик, которые определяют следующие параметры:

- Облачная инфраструктура, в которой хранятся диагностические данные.
- Срок хранения диагностических сведений.
- Разрешения, контролирующие доступ к диагностическим сведениям.

## Диагностика в автономном режиме
В ситуациях, когда устройство не может собирать диагностику через центр отзывов или средство устранения неполадок, вы можете собирать диагностику вручную. Одна из ситуаций, в которой это необходимо, заключается в том, что устройство не может подключаться к Wi-Fi. Диагностические дампы и журналы на устройстве, которые помогают инженеру службы поддержки Майкрософт изолировать проблемы.

Это происходит, когда устройство появляется в проводнике после его подключения к компьютеру с помощью USB-кабеля. 


> [!VIDEO https://channel9.msdn.com/Shows/Docs-Mixed-Reality/Gathering-Diagnostic-Files-on-HoloLens2/player]

Выполните указанные ниже действия, чтобы собрать диагностические данные.
1.  Подключите устройство к компьютеру с помощью USB-кабеля.
2.  В проводнике на компьютере перейдите к разделу **"Этот компьютер \<hololens-device> \Internal Storage"**.
3.  Если **Внутренняя папка хранилища** не отображается, устройство ждет, пока пользователь не выйдет из системы. Войдите в службу или включите питание устройства, держа нажатой кнопку ВЫКЛЮЧЕНия на 10 секунд.
4.  Нажмите и сразу же отпустите кнопки **Power + Volume вниз** .
5.  Подождите минуту, пока устройство не подготовит ZIP-архивы. (Временный файл с именем HololensDiagnostics. temp может стать видимым, пока устройство генерирует ZIP-архивы. Не получайте к ним доступ и не сохраняйте этот файл. После завершения процесса оно будет заменено архивами ZIP.
6.  Обновите проводник и перейдите в папку **"\Documents"** .
7.  Скопируйте ZIP-файлы диагностики и делитесь ими с группой поддержки Майкрософт.

Обратите внимание, что некоторые из ZIP-файлов диагностики могут содержать личные данные.
