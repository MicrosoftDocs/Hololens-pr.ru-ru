# <a name="collect-and-use-diagnostic-information-from-hololens-devices"></a>Сбор и использование сведений диагностики с устройств HoloLens

HoloLens пользователи и администраторы могут выбрать один из четырех различных методов для получения диагностических сведений от HoloLens:

- Приложение концентратора обратной связи
- Поставщик служб конфигурации DiagnosticLog
- Приложение «Параметры»
- Автономная диагностика

> [!IMPORTANT]  
> В журналах диагностики устройств содержатся персональные данные (PII), например сведения о том, какие процессы или приложения пользователь запускает во время стандартных операций. если несколько пользователей совместно используют устройство HoloLens (например, пользователи входят на одно устройство, используя разные учетные записи Microsoft Azure Active Directory (Azure AD)), журналы диагностики могут содержать персональные данные, относящиеся к нескольким пользователям. Дополнительные сведения см. в [заявлении о конфиденциальности Майкрософт](https://privacy.microsoft.com/privacystatement).

## <a name="feedback-hub"></a>Центр отзывов

HoloLens пользователь может использовать классическое приложение центра обратной связи майкрософт для отправки диагностических сведений в служба поддержки Майкрософт. Дополнительные сведения и инструкции см. в статье [обратная связь](hololens-feedback.md).  

> [!NOTE]  
> **Коммерческие и корпоративные пользователи:** если вы используете приложение центра обратной связи для сообщения о проблеме, связанной с MDM, подготовкой или любым другим аспектом управления устройствами, измените категорию приложения на **Enterprise**  >  **категории устройств** управления.

>[!IMPORTANT]
> Чтобы предоставить лучшие данные для устранения проблем, настоятельно рекомендуется задать для телеметрии устройства значение **необязательно**. это значение можно задать при первом запуске (OOBE) или с помощью **Параметры** приложения. чтобы сделать это с помощью Параметры, выберите **пуск > Параметры > конфиденциальность > диагностика приложений > в**.

### <a name="prerequisites"></a>Предварительные требования

- Устройство подключено к сети.
- Приложение центра обратной связи доступно на настольном компьютере пользователя, и пользователь может отправлять файлы в облако Майкрософт.

### <a name="data-locations-access-and-retention"></a>Расположение, доступ и хранение данных

Если принять условия использования центра обратной связи, пользователь явно отправил запрос на хранение и использование данных (как определено в этом соглашении).

Центр отзывов предоставляет пользователю два места для хранения диагностических сведений:

- **Microsoft Cloud**. Данные, которые пользователь отправляет с помощью приложения концентратора обратной связи, хранятся в течение числа дней, которые соответствуют требованиям к конфиденциальности следующего поколения (NGP). Сотрудники корпорации Майкрософт могут использовать для доступа к информации в течение этого периода средство просмотра, совместимое с NGP.

   > [!NOTE]  
   > Эти требования применяются к данным во всех категориях центра отзывов.

- **устройство HoloLens**. При сохранении отчета в центре обратной связи пользователь может выбрать вариант **сохранить локальную копию диагностики и вложений, созданных при** отправке отзыва. если пользователь выбирает этот параметр, центр обратной связи сохраняет копию диагностической информации на HoloLens устройстве. Эти сведения остаются доступными для пользователя (или любого, который использует эту учетную запись для входа в HoloLens). Чтобы удалить эти сведения, пользователь должен иметь права **владельца устройства** или **администратора** на устройстве. пользователь с соответствующими разрешениями может войти в центр обратной связи, выбрать **Параметры**  >  **просмотреть журналы диагностики** и удалить эти сведения.

## <a name="settings-troubleshooter"></a>Параметры Средство устранения неполадок

HoloLens пользователь может использовать приложение **Параметры** на устройстве для устранения неполадок и сбор диагностических данных. Для этого выполните следующие действия:

1. откройте приложение Параметры и выберите **Update &**  >  страница **устранение неполадок** безопасности.
1. Выберите соответствующую область и нажмите кнопку **запустить**.
1. Воспроизведите проблему.
1. после воспроизведения проблемы вернитесь в Параметры и нажмите кнопку " **закрыть**".

пользователь также может настроить поведение резервной диагностики из приложения **Параметры** . Перейдите к странице " **Конфиденциальность" — > устранении неполадок** , чтобы настроить этот параметр.
> [!NOTE]
> Если для устройства настроена политика MDM, пользователь не сможет переопределить это поведение.

### <a name="os-update-troubleshooter"></a>Средство устранения неполадок обновления ОС

в сборках [Windows holographic, version 21H1](hololens-release-notes.md#windows-holographic-version-21h1) и далее:
- в дополнение к предыдущим средствам устранения неполадок в приложении Параметры был добавлен новый средство устранения неполадок с добавлением нового приложения Параметры для обновления операционной системы. перейдите в раздел **Параметры-> обновление & безопасность — > устранение неполадок — > Центр обновления Windows** и нажмите кнопку **запустить**. Это позволяет выполнять накопление трассировок при воспроизведении проблемы с обновлениями ОС, чтобы помочь вам лучше устранять неполадки в работе ИТ или поддержки.

### <a name="prerequisites"></a>Предварительные требования

- приложение **Параметры** установлено на устройстве и доступно пользователю.

### <a name="data-locations-access-and-retention"></a>Расположение, доступ и хранение данных

Поскольку пользователь запускает сбор данных, пользователь неявно отправляет данные диагностики в хранилище. Получить доступ к данным могут только пользователи и пользователи, которым он совместно использует данные.

Диагностические данные хранятся на устройстве. Если устройство подключено к компьютеру пользователя, информация также находится на компьютере в следующем файле:

> на этом компьютере \\ \<*HoloLens device name*> \\ внутренняя служба хранилища \\ документы \\ Trace \<*ddmmyyhhmmss*> . etl

> [!NOTE]  
> в этом пути и имени файла \<*HoloLens device name*> представляет имя устройства HoloLens и \<*ddmmyyhhmmss*> представляет дату и время создания файла.

Диагностические сведения остаются в этих расположениях до тех пор, пока пользователь не удалит их.

### <a name="view-diagnostic-report"></a>Просмотр диагностического отчета

чтобы просмотреть диагностику MDM на HoloLens 2, щелкните значок wi-fi, перейдите в раздел **Параметры**  ->  **учетные записи**  >  **доступ к работе или учебы** и выберите **экспорт журналов управления**. HoloLens отправляет файлы журналов в учетную запись и отображает их расположение на настольном компьютере.

### <a name="data-locations-access-and-retention"></a>Расположение, доступ и хранение данных

Так как устройство является частью управляемой среды, пользователь неявно предоставляет пользователю административный доступ к диагностическим сведениям.

ИТ – администратор использует Диагностиклог CSP для настройки политик хранения данных, хранения и доступа, включая политики, которые управляют следующими:

- Облачная инфраструктура, в которой хранятся диагностические данные.
- Срок хранения диагностических данных.
- Разрешения, управляющие доступом к диагностическим сведениям.

## <a name="offline-diagnostics"></a>Автономная диагностика

в ситуациях, когда устройство не может выполнять сбор диагностических сведений с помощью центра обратной связи или средства устранения неполадок Параметры, можно выполнить сбор диагностических сведений вручную. Один из сценариев, в котором это необходимо, заключается в том, что устройство не может подключиться к Wi-Fi или вы не можете получить доступ к другим методам, упомянутым выше. Диагностика собираются аварийные дампы и журналы с устройства, которое помогает инженеру службы поддержки Майкрософт изолировать проблемы.

Это работает, когда устройство отображается в проводнике после его подключения к компьютеру через USB-кабель.

> [!NOTE]
> Создание автономной системы диагностики и управление ими управляются по-разному в зависимости от версии ОС. Ранее он управлялся параметром телеметрии, но теперь напрямую контролируется с помощью политики MDM. Если параметр отключен с помощью параметра или политики MDM, сбор журналов диагностики с помощью этого механизма невозможен.

поведение до [Windows holographic, версия 20H2](hololens-release-notes.md#windows-holographic-version-20h2):
 - Автономная диагностика включена только в том случае, если пользователь переходит через режим OOBE или [систем\алловтелеметри](/windows/client-management/mdm/policy-csp-system#system-allowtelemetry) значение политики Full (значение по умолчанию — Basic для HoloLens). 
- чтобы отключить диагностику автономной работы, перейдите на страницу **Параметры приложение > конфиденциальность** и выберите **базовый** в **диагностических данных**. В сборках, в которых автономная диагностика зависит от настройки телеметрии, она влияет только на сбор журналов. Это не влияет на собираемые файлы.
- Если устройство заблокировано, журналы не отображаются.

в сборках [Windows holographic, version 20H2](hololens-release-notes.md#windows-holographic-version-20h2) и далее:
- Если резервная диагностика включена, она будет управляться конкретной политикой MDM с соответствующим параметром [микседреалити/фаллбаккдиагностикс](/windows/client-management/mdm/policy-csp-mixedreality#mixedreality-fallbackdiagnostics) .
- Если устройство заблокировано, журналы не отображаются.

Просмотрите это видео для получения дополнительных сведений.

> [!VIDEO https://channel9.msdn.com/Shows/Docs-Mixed-Reality/Gathering-Diagnostic-Files-on-HoloLens2/player]

Для получения сведений о диагностике выполните следующие действия.

1.  Подключение устройство с USB-кабелем на компьютер.

2.  в проводнике на компьютере перейдите к папке **"This pc \<hololens-device> \интернал служба хранилища"**.

3.  если **внутренняя служба хранилищаная** папка не отображается, устройство ожидает входа пользователя. Войдите в систему или выключите устройство, удерживая кнопку питания на 10 секунд.

4.  Нажмите и немедленно отпустите кнопки **питания + громкость** .

5.  Подождите минуту, пока устройство не подготовит ZIP-архивы. (Временный файл с именем Хололенсдиагностикс. temp может стать видимым, пока устройство создает ZIP-архивы. Не отменяйте доступ к этому файлу и не сохраняйте его. После завершения процесса он будет заменен zip-архивом.)

6.  Обновите обозреватель файлов и перейдите в папку **"\Documents"** .

7.  Скопируйте ZIP-файлы диагностики и поделитесь ими с группой поддержки Майкрософт.

> [!NOTE]
> Некоторые ZIP-файлы диагностики могут содержать персональные данные.

### <a name="offline-diagnostics-notifications"></a>Автономные уведомления диагностики

- введено в [Windows holographic, version 21H2](hololens-release-notes.md#windows-holographic-version-21h2).

Это обновление для существующей функции, называемой [автономной диагностикой](hololens-diagnostic-logs.md#offline-diagnostics). Ранее не было четкого индикатора для пользователей о том, что они активировали сбор диагностических сведений или завершились.
теперь в сборки предварительной версии Windows есть две формы отзывов аудиовизуального для автономной диагностики. Первое уведомление, отображаемое при запуске и завершении сбора. Они будут отображаться, когда пользователь вошел в систему и имеет визуальные элементы.

![Всплывающее уведомление для сбора журналов.](./images/logcollection1.jpg)

![Всплывающее уведомление после завершения сбора журналов.](./images/logcollection2.jpg)

Так как пользователи часто используют автономную диагностику в качестве резервного механизма для сбора журналов, когда они не имеют доступа к дисплею, не могут войти в систему или по-прежнему находятся в OOBE. при сборе журналов также будет использоваться звуковая подсказка. Этот звук будет воспроизводиться в дополнение к всплывающему уведомлению.

Эта новая функция будет включена при обновлении устройства и не требует включения или управления. В любом случае, если этот новый отзыв не может быть отображен или слышен, Автономная диагностика будет по-прежнему создана.

Мы надеемся, что с этим новым дополнением аудиовизуального Feedback проще собирать диагностические данные и быстрее выполнять устранение неполадок.

### <a name="low-storage-log-collection-improvements"></a>Усовершенствования сбора журналов нехватки памяти

- введено в [Windows holographic, version 21H2](hololens-release-notes.md#windows-holographic-version-21h2).

В сценариях, в которых при сборе журналов диагностики на устройстве не хватает места, будет создан дополнительный отчет с именем **StorageDiagnostics.zip** . пороговое значение низкого уровня хранилища определяется автоматически Windowsным [датчиком хранилища](https://support.microsoft.com/office/use-onedrive-and-storage-sense-in-windows-10-to-manage-disk-space-de5faa9a-6108-4be1-87a6-d90688d08a48).

## <a name="view-advanced-diagnostic-report-in-settings-on-hololens"></a>просмотр расширенного диагностического отчета в Параметры на HoloLens

- введено в [Windows holographic, version 21H2](hololens-release-notes.md#windows-holographic-version-21h2).

Для управляемых устройств при устранении неполадок Проверка применения ожидаемой конфигурации политики является важным шагом. ранее для этой новой функции это было необходимо сделать на устройстве с помощью MDM или вблизи устройства после экспорта журналов диагностики mdm, собранных с помощью **Параметры**  ->  **учетных записей**  >  , в **рабочую или учебную** систему, а также выбрать **экспорт журналов управления** и просмотр на ближайшем компьютере.

Теперь можно просмотреть диагностику MDM на устройстве с помощью браузера пограничной сети. Чтобы упростить просмотр диагностического отчета MDM, перейдите на страницу доступа к рабочей или учебной системе и выберите **просмотреть расширенный диагностический отчет**. При этом отчет будет создан и открыт в новом пограничном окне.
