# <a name="collect-and-use-diagnostic-information-from-hololens-devices"></a>Сбор и использование сведений диагностики с устройств HoloLens

Пользователи и администраторы HoloLens могут выбрать один из четырех различных методов сбора диагностических сведений из HoloLens:

- Приложение Концентратор отзывов
- Поставщик служб конфигурации DiagnosticLog
- Приложение «Параметры»
- Диагностика в автономном режиме

> [!IMPORTANT]  
> Журналы диагностики устройств содержат персональные идентифицируемые сведения (PII), например о том, какие процессы или приложения пользователь запускает во время типичных операций. Если несколько пользователей используют устройство HoloLens (например, пользователи вписываются в одно устройство с помощью различных учетных записей Microsoft Azure Active Directory (Azure AD), диагностические журналы могут содержать сведения о PII, применимые к нескольким пользователям. Дополнительные сведения см. в [заявлении Microsoft Privacy.](https://privacy.microsoft.com/privacystatement)

## <a name="feedback-hub"></a>Центр отзывов

Пользователь HoloLens может использовать настольное приложение Microsoft Feedback Hub для отправки диагностических сведений в службу поддержки Майкрософт. Подробные сведения и инструкции см. в [материале Give us feedback](hololens-feedback.md).  

> [!NOTE]  
> **Коммерческие или корпоративные пользователи:** Если вы используете приложение Концентратор отзывов, чтобы сообщить о проблеме, которая связана с MDM, **** подготовка или любой другой аспект управления устройствами, измените категорию приложения на категорию устройства управления  >  **предприятием.**

### <a name="prerequisites"></a>Предварительные условия

- Устройство подключено к сети.
- Приложение Концентратор отзывов доступно на настольном компьютере пользователя, и пользователь может загружать файлы в облако Майкрософт.

### <a name="data-locations-access-and-retention"></a>Расположение данных, доступ и хранение данных

Соглашаясь на условия использования концентратора отзывов, пользователь явно соглашается на хранение и использование данных (как это определено в этом соглашении).

Центр обратной связи предоставляет пользователю два места для хранения диагностических сведений:

- **Облако Microsoft**. Данные, которые пользователь загружает с помощью приложения Концентратор отзывов, хранятся в течение нескольких дней, соответствующего требованиям конфиденциальности следующего поколения (NGP). Сотрудники Корпорации Майкрософт могут использовать viewer, совместимый с NGP, для доступа к данным в течение этого периода.
   > [!NOTE]  
   > Эти требования применяются к данным во всех категориях Концентратор отзывов.

- **Устройство HoloLens**. При подаче отчета в центре отзывов пользователь может выбрать сохранить локализованную копию диагностики и вложений, созданных **при предоставлении обратной связи.** Если пользователь выбирает этот параметр, центр отзывов сохраняет копию диагностических сведений на устройстве HoloLens. Эти сведения остаются доступными пользователю (или любому пользователю, использующему эту учетную запись для входов в HoloLens). Чтобы удалить эту информацию, пользователь должен **** иметь разрешения владельца **устройства** или администратора на устройстве. Пользователь с соответствующими разрешениями может войти в Центр отзывов, выбрать журналы диагностики **"Просмотр**параметров"  >  **** и удалить сведения.

## <a name="settings-troubleshooter"></a>Устранение неполадок параметров

Пользователь HoloLens может использовать приложение Параметры на устройстве для устранения неполадок и сбора диагностических сведений. Для этого выполните следующие действия:

1. Откройте приложение Параметры и выберите **страницу Обновление &**  >  **безопасности.**
1. Выберите соответствующую область и выберите **Начните**.
1. Воспроизвести проблему.
1. После воспроизведения проблемы вернись в Параметры и выберите **Стоп.**

### <a name="prerequisites"></a>Предварительные условия

- Приложение Параметры установлено на устройстве и доступно пользователю.

### <a name="data-locations-access-and-retention"></a>Расположение данных, доступ и хранение данных

Так как пользователь запускает сбор данных, пользователь неявно соглашается на хранение диагностических сведений. Доступ к данным может получить только пользователь или любой пользователь, с которым пользователь делится данными.

Диагностические сведения хранятся на устройстве. Если устройство подключено к компьютеру пользователя, информация также находится на компьютере в следующем файле:

> Этот КОМПЬЮТЕР\\ \<*HoloLens device name*> \\Внутреннее хранилище\\Документы\\Трассировка \<*ddmmyyhhmmss*> .etl

> [!NOTE]  
> В этом пути и имени файла представляет имя устройства HoloLens и представляет дату и время создания \<*HoloLens device name*> \<*ddmmyyhhmmss*> файла.

Диагностические сведения остаются в этих местах до удаления пользователем.

## <a name="diagnosticlog-csp"></a>Поставщик служб конфигурации DiagnosticLog

В среде управления мобильными устройствами (MDM) ИТ-администратор может использовать поставщика служб конфигурации [DiagnosticLog (CSP)](https://docs.microsoft.com/windows/client-management/mdm/diagnosticlog-csp) для настройки параметров диагностики на зарегистрированных устройствах HoloLens. ИТ-администратор может настроить эти параметры для сбора журналов с зарегистрированных устройств.

### <a name="prerequisites"></a>Предварительные условия

- Устройство подключено к сети.
- Устройство зарегистрировано в среде MDM, которая поддерживает CSP DiagnosticLog.

### <a name="data-locations-access-and-retention"></a>Расположение данных, доступ и хранение данных

Поскольку устройство является частью управляемой среды, пользователь неявно соглашается на административный доступ к диагностическим данным.

ИТ-администратор использует CSP DiagnosticLog для настройки политик хранения, хранения и доступа к данным, в том числе политик, управляющих следующими политиками:

- Облачная инфраструктура, которая хранит диагностические сведения.
- Период хранения диагностических сведений.
- Разрешения, которые контролируют доступ к диагностическим данным.

## <a name="offline-diagnostics"></a>Диагностика в автономном режиме
В ситуациях, когда устройство не может собирать диагностику с помощью центра обратной связи или средства устранения неполадок параметра, можно собирать диагностику вручную. Один из сценариев, когда это необходимо, это когда устройство не может подключиться к Wi-Fi или вы не можете получить доступ к другим методам, упомянутым выше. Диагностика собирает сбои и журналы с устройства, которые помогают инженеру службы поддержки Майкрософт изолировать проблемы.

Это работает, когда устройство появляется в Проводнике файлов после подключения его к компьютеру с помощью USB-кабеля.

> [!NOTE]
> Автономное генерация и управление диагностикой контролируется по-разному в зависимости от версии ОС. Раньше он контролировался параметром телеметрии, но теперь непосредственно контролируется с помощью политики MDM. Если параметр или политика MDM отключены, то журналы диагностики не могут быть собраны с помощью этого механизма.

Поведение до [Windows Holographic, verison 20H2](hololens-release-notes.md#windows-holographic-version-20h2):
 - Диагностика в автономном режиме включена только в том случае, если пользователь проходит через OOBE или [System\AllowTelemetry](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-system#system-allowtelemetry) значение Full (Basic — это значение по умолчанию для HoloLens). 
- Чтобы отключить диагностику в автономном режиме, перейдите на страницу **Параметры приложения > конфиденциальности** и выберите **Basic** в **диагностических данных.** При сборках, где диагностика в автономном режиме зависит от параметра телеметрии, это влияет только на то, собираются журналы или нет. Это не влияет на то, какие файлы собираются.
- Если устройство заблокировано, журналы не отображаются.

При сборке [Windows Holographic verison 20H2](hololens-release-notes.md#windows-holographic-version-20h2) и далее:
- При включенной диагностике fallback будет управляться определенной политикой MDM с соответствующими настройками [MixedReality/FallbackDiagnostics](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-mixedreality#mixedreality-fallbackdiagnostics)
- Если устройство заблокировано, журналы не отображаются.

Просмотрите это видео, чтобы узнать больше.

> [!VIDEO https://channel9.msdn.com/Shows/Docs-Mixed-Reality/Gathering-Diagnostic-Files-on-HoloLens2/player]

Выполните следующие действия для сбора диагностики:
1.  Подключите устройство с помощью USB-кабеля к компьютеру.
2.  В Обозревателе файлов на компьютере перейдите на **"Этот компьютер \<hololens-device> \Внутреннее хранилище".**
3.  Если **папка внутреннего** хранения не показывается, устройство ожидает, когда пользователь войдет. Вход или цикл питания устройства, удерживая кнопку POWER вниз в течение 10 секунд.
4.  Нажмите и немедленно отпустите **кнопки POWER + VOLUME DOWN** вместе.
5.  Подождите минуту, пока устройство подготовит архивы почтовых индексов. (Временный файл с именем HololensDiagnostics.temp может быть виден, пока устройство создает архивы почтовых индексов. Не нужно получать доступ к этому файлу или сохранять его. По завершению процесса его заменят архивы почтовых индексов.)
6.  Обновите проводник файлов и перейдите в **папку "\Documents".**
7.  Скопируйте файлы ZIP диагностики и поделитесь ими с командой поддержки Майкрософт.

> [!NOTE]
> Некоторые из диагностических почтовых файлов могут содержать персональные идентифицируемые сведения.
