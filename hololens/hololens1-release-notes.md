# <a name="hololens-1st-gen-release-notes"></a>Заметки о выпуске для HoloLens 1 (Gen)

## <a name="hololens-1st-gen-long-term-servicing"></a>Долгосрочное обслуживание HoloLens (1-го поколения)
HoloLens (1-й общий) перешел в состояние долгосрочного обслуживания (LTS). В будущих обновлениях основное внимание уделяется устранению проблем и исправлений безопасности, а также обеспечению соответствия функций в выпуске Windows 10 holographic, версии 1809 для HoloLens (1-го поколения).

Для разработчиков это означает, что приложения HoloLens (1-го поколения) не будут поддерживать API Опенкср.  Эти гарнитуры по-прежнему поддерживаются в Unity 2019 LTS с серверной частью API WinRT для полного жизненного цикла Unity 2019 LTS до середины 2022.

### <a name="windows-10-version-1803-for-microsoft-hololens"></a>Windows 10, версия 1803 для Microsoft HoloLens

> **Применимо к:** HoloLens (1-й общий)

Windows 10, версия 1803, является первым обновлением компонентов Windows holographic для бизнеса с момента выпуска в Windows 10 версии 1607. В этом обновлении представлены следующие изменения.

- Ранее вы могли только убедиться, что лицензия на обновление для коммерческого набора была применена к устройству HoloLens, проверив, был ли VPN доступен на устройстве. Теперь   >  после применения лицензии на обновление **система** параметров будет отображать **Windows holographic для бизнеса** . [Узнайте, как разблокировать функции Windows holographic для бизнеса](hololens1-upgrade-enterprise.md).

- Номер сборки операционной системы можно просмотреть в свойствах устройства в приложении проводника и в [средстве восстановления устройств Windows (вдрт)](https://support.microsoft.com/help/12379/windows-10-mobile-device-recovery-tool-faq).
- Подготовка устройства HoloLens теперь упрощается с помощью нового мастера **подготовки устройств hololens** в средстве "конструктор конфигураций Windows". В мастере можно настроить интерфейс установки и сетевые подключения, задать режим разработчика и получить негрупповые маркеры Azure AD. [Узнайте, как использовать мастер простой подготовки для HoloLens](hololens-provisioning.md#provisioning-package-hololens-wizard).

- При создании локальной учетной записи в пакете подготовки пароль больше не будет истечь каждые 42 дней.

- HoloLens можно [настроить как киоск с одним приложением или с несколькими приложениями](hololens-kiosk.md). Режим киоска с несколькими приложениями позволяет настроить HoloLens для запуска только указанных вами приложений и запрещает пользователям вносить изменения.

- Протокол передачи мультимедиа (MTP) включен, поэтому вы можете подключить устройство HoloLens к компьютеру по USB и передавать файлы между HoloLens и компьютером. Для перемещения и удаления файлов в HoloLens можно также использовать приложение проводника.

- Ранее после входа на устройство с помощью учетной записи Azure Active Directory (Azure AD) вам пришлось **Добавить рабочий доступ** в **Параметры** , чтобы получить доступ к корпоративным ресурсам. Теперь вы входите с учетной записью Azure AD, и регистрация выполняется автоматически.

- Перед входом в систему можно выбрать значок сети под полем "пароль", чтобы выбрать другую Wi-Fi сеть для подключения. Вы также можете подключиться к гостевой сети, например в отеле, в центре конференц-связи или в компании.

- Теперь можно легко [предоставить доступ к HoloLens нескольким людям](hololens-multiple-users.md) с помощью учетных записей Azure AD.

- При сбое установки или входа выберите новый параметр **сбор сведений** , чтобы получить журналы диагностики для устранения неполадок.

- Отдельные пользователи могут синхронизировать корпоративную электронную почту без регистрации устройства в системе управления мобильными устройствами (MDM). Вы можете использовать устройство с учетной записью Майкрософт, скачать и установить почтовое приложение и добавить учетную запись электронной почты напрямую.

- Проверить состояние синхронизации MDM для устройства можно в **настройках**  >  **учетные записи**  >  **доступ к рабочей или учебной**  >  **информации**. В разделе **состояние синхронизации устройства** можно запустить синхронизацию, Просмотреть области, управляемые MDM, а также создать и экспортировать Расширенный отчет о диагностике.
