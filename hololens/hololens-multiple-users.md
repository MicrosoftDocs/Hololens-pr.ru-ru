# <a name="share-your-hololens-with-multiple-people"></a>совместное использование HoloLens с несколькими людьми

## <a name="overview"></a>Обзор

предприятия часто вкладываются во многие общие HoloLens устройства. способ использования HoloLens является гибким по отношению к доске в зависимости от конкретных требований. Ниже приведен пример некоторых возможностей для нескольких пользователей.

- парк HoloLens 2 устройств настраивается через Windows автопилот для HoloLens 2, с единым портфелем приложений компании на каждом устройстве. Вы настроили несколько профилей киосков, предназначенных для разных групп Azure AD. каждый пользователь входит в HoloLens с помощью ключей FIDO2 и выполняет вход в свою учетную запись Azure AD и предоставляется с помощью специализированного интерфейса.
- независимый поставщик программного обеспечения (ISV) сдается HoloLens 2 устройства с помощью удаленного помощника D365 и бизнес-приложения в компанию клиента. Эти устройства настроены для киосков, которые включают только свои бизнес-приложения и удаленный помощник, и являются общими для нескольких конечных пользователей. WDAC используется для сохранения Параметры приложения и Microsoft Edge запуска. В комплект поставки, входящий в аренду, входит USB-C аккумуляторный пакет для полной оплаты за несколько смен.
- пользователь на предприятии пытается внести изменения в Bluetooth на устройстве, чтобы они могли подключаться к новому устройству, но политика видимости на странице Параметры включена, чтобы ограничить просмотр страницы устройств. Они по-прежнему разрешают доступ к другим страницам по мере необходимости, например Wi-Fi, чтобы они могли использовать удаленный помощник в нескольких расположениях с тем же HoloLens.

## <a name="best-practices"></a>Рекомендации

При планировании совместного использования устройств существует несколько рекомендаций по оптимизации среды устройства в соответствии с потребностями бизнеса.

- [Идентификация и проверка подлинности](#identity-and-authentication)
- [Управление устройствами](#device-management)
- [Расширенное управление устройствами — киоск и WDAC](#advanced-device-management---kiosk-and-wdac)
- [Физическое управление](#physical-management)

### <a name="identity-and-authentication"></a>[Идентификация и проверка подлинности](hololens-identity.md)

Если вы планируете использовать несколько учетных записей на устройстве, то у вас будут учетные записи Azure AD со всеми режимами проверки подлинности. эти методы проверки подлинности будут основываться на [Windows Hello](/windows-hardware/design/device-experiences/windows-hello), включая ключи iri и FIDO2.

- FIDO 2 ключи безопасности отлично подходят, если у вас несколько устройств, много пользователей или постоянно используются новые устройства.
- Если число пользователей не превышает 10, IRI — это быстрое решение для входа пользователей, которые ранее выполнили вход на одно и то же устройство.

### <a name="device-management"></a>[Управление устройствами](hololens-csp-policy-overview.md)

Если устройства совместно используются пользователями, то, скорее всего, потребуется использовать ограничения для устройств. С помощью управления устройствами можно задать некоторые политики, чтобы лучше разрешить пользователям использовать устройство, управлять обновлениями или ограничить возможности устройства. Рекомендуется ознакомиться с нашими [общими ограничениями на устройства](hololens-common-device-restrictions.md)и узнать, соответствуют ли эти рекомендации вашей организации. когда вы узнаете, какие политики вы хотите использовать, их можно применить с помощью [Microsoft Endpoint Manager (MDM)](hololens-mdm-configure.md) или подготовки пакетов.

### <a name="advanced-device-management---kiosk-and-wdac"></a>Расширенное управление устройствами — [киоск](hololens-kiosk.md) и [WDAC](windows-defender-application-control-wdac.md)

В некоторых случаях может потребоваться ограничить доступ к приложениям для конечных пользователей. Вы можете ограничить, какие приложения отображаются в меню "Пуск", используя [режим киоска](hololens-kiosk.md). Киоск можно настроить для отображения различных начальных меню на основе пользователей, групп Azure или специальных типов пользователей. такой посетитель или за исключением владельцев устройств. Можно выбрать несколько приложений или только одно приложение. Приложение с несколькими приложениями не останавливает запуск другого приложения, поэтому, если хранилище или другое приложение доступны, пользователи по-прежнему могут запускать другое приложение.

также может потребоваться полностью завершить запуск приложений или служб с помощью [Защитник Windows управления приложениями (WDAC)](windows-defender-application-control-wdac.md) для ограничения приложений. WDAC отличается от киоска, так как он не изменяет пользовательский интерфейс HoloLens, а не позволяет запускать заблокированное приложение.

[видимость Параметры страницы](settings-uri-list.md) — еще один способ добавления ограничений на устройство. в случае, когда необходимо предоставить пользователям доступ к некоторым страницам в Параметры приложении, но не все можно использовать видимость страницы Параметры для ограничения доступа. Это полезно, например, если пользователям необходимо изменить Wi-Fi, но вы не хотите, чтобы они были доступны на странице учетные записи.

### <a name="physical-management"></a>Физическое управление

При совместном использовании устройства несколькими пользователями существует ряд физических факторов.

- Убедитесь, что между сменами производится оплата устройств.
- Если устройство требуется для сдвига и при этом требуется выполнить несколько смен, рассмотрите возможность использования внешней батареи в начале смены, когда на устройстве по-прежнему взимается значительная плата за [Управление теплоотводами](hololens2-charging.md#managing-heat).
- При хранении устройства должны быть подключены к сети и подключены к ней. Это лучший способ обеспечить обновление операционной системы и приложений.
- Подумайте, как вы планируете [очистить устройство](hololens2-maintenance.md) между пользователями.
- Для устройства с одним общим пользователем, если для одного пользователя используется общий ПИН-код или пароль, не следует вводить ПИН-код и пароль на стороне устройства.
- Для нескольких устройств с одним общим пользователем Используйте различные ПИН-коды и пароли.

## <a name="share-with-multiple-people-each-using-their-own-account"></a>Предоставление общего доступа нескольким людям с использованием собственной учетной записи

учетные записи отдельных Azure Active Directory (Azure AD) являются предпочтительным и наиболее безопасным вариантом использования удостоверений для HoloLens 2 пользователей. При использовании собственных учетных записей Azure AD несколько пользователей могут иметь собственные параметры пользователя и данные пользователя на устройстве. Только один пользователь может войти в каждый момент времени. когда пользователь входит в систему, HoloLens входит в систему предыдущего пользователя.

чтобы убедиться, что несколько пользователей могут использовать собственные учетные записи на HoloLens, выполните следующие действия, чтобы настроить ее.

1. Когда вы [настроите устройство](hololens2-start.md), выберите **Моя работа или учебный заведение** , а затем выполните вход с помощью учетной записи Azure AD.
1. после завершения установки убедитесь, что параметры учетной записи (Параметры > учетные записи) включают **других пользователей**.

> [!NOTE]
> Если устройство не настроено с учетной записью Azure AD, оно должно быть либо [сброшено, либо](hololens-recovery.md) переустановлено правильно.

чтобы использовать HoloLens, каждый пользователь выполняет следующие действия:

1. Если устройство используется другим пользователем, выберите один из следующих вариантов.
   - Нажмите кнопку питания один раз, чтобы вернуться в режим ожидания, а затем снова нажмите кнопку питания, чтобы вернуться на экран блокировки.
   - выберите плитку пользователя в **меню** или выберите выйти из **меню питания** , чтобы выйти из текущего пользователя.

1. Используйте учетные данные учетной записи Azure AD для входа на устройство.  
    - если вы используете устройство впервые, вам будет предложено [откалибровать](hololens-calibration.md) HoloLens своими глазами.
    - Если вы ранее использовали устройство:
        - [Windows holographic, version 20H2, build 19041,1128](hololens-release-notes.md#windows-holographic-version-20h2) или более поздней версии, экран легко корректирует качество и удобство просмотра.
        - В предыдущих сборках потребуется ручная калибровка для изменения ваших глаз.

> [!TIP]
> Если пользователь еще не вошел в устройство, попробуйте использовать один из этих двух методов для ускорения входа:
>
> - **Ключ безопасности Fido 2** . ваш ключ безопасности FIDO2 будет автоматически распознан, и пользователю не нужно будет вводить учетные данные пользователя или использовать mfa. Это самый быстрый способ входа на новое устройство.
> - **Веб-аутентификация** . при входе на новое устройство можно выбрать ссылку **Вход с другого устройства** , которая создаст код 9 символов, который можно использовать в [AKA.MS/devicelogin](https://login.microsoftonline.com/common/oauth2/deviceauth) для входа в качестве пользователя на устройстве, или ввести имя пользователя и пароль с помощью клавиатуры для удобства.

чтобы просмотреть список пользователей устройства или удалить пользователя с устройства, перейдите в раздел **Параметры**  >  **учетные записи**  >  **других пользователей**.

## <a name="share-with-multiple-people-all-using-the-same-account"></a>Поделиться с несколькими людьми, используя одну и ту же учетную запись

при использовании одной учетной записи пользователя несколько пользователей также могут совместно использовать устройство HoloLens. хотя пользователям рекомендуется HoloLens входить на устройство с индивидуальными удостоверениями (учетные записи Azure AD), это не является вариантом в некоторых организациях.

Доступны два общих метода устройства:

- **несколько конечных пользователей, совместно использующих 1 устройство** . устройство HoloLens выделяется для выделенного пространства, где любой сотрудник может использовать устройство. Примерами могут быть чистые комнаты или сургикал Suite.

- **несколько конечных пользователей, совместно использующих несколько устройств** . HoloLens устройства находятся в общем дисковом пространстве, где сотрудники могут использовать любое устройство. Примерами могут быть многопрограммные платформы, автоматические дилеры и необычное.

Когда новый пользователь помещает устройство в первый раз при сохранении той же учетной записи, на которой выполняется вход, устройство предлагает пользователю быстро выполнить калибровку и персонализацию процесса просмотра. Устройство будет хранить сведения о калибровке, чтобы автоматически оптимизировать качество и удобство просмотра каждого пользователя. Пользователям больше не потребуется откалибровать устройство.
