## <a name="eap-troubleshooting"></a>Устранение неполадок с EAP
1. Необходимо внимательно проверить следующие настройки профиля Wi-Fi:
    - Правильная настройка типа EAP, распространенные типы EAP: EAP-TLS (13), EAP-TTLS (21) и PEAP (25).
    - Правильность идентификатора SSID Wi-Fi и его совпадение с шестнадцатеричной строкой.
    - Для EAP-TLS: наличие в TrustedRootCA хэша SHA-1 сертификата доверенного корневого центра сертификации сервера. На ПК с Windows строку хэша SHA-1 для сертификата можно получить с помощью команды certutil.exe -dump cert_file_name.
2. Чтобы узнать, в каких случаях происходит сбой сеанса EAP, запишите сетевые пакеты в журнале точки доступа, контроллера или сервера AAA.
    - Если удостоверение EAP, предоставленное службой HoloLens, не соответствует ожидаемому, проверьте правильность удостоверения, предоставленного с помощью профиля Wi-Fi или клиентского сертификата.
    - Если сервер отклонил сертификат клиента HoloLens, убедитесь, что на устройстве установлен требуемый сертификат клиента.
    - Если HoloLens отклоняет сертификат сервера, убедитесь, что на HoloLens установлен сертификат корневого ЦС сервера.
3. Если профиль предприятия предоставлен с помощью пакета подготовки Wi-Fi, рассмотрите возможность установки пакета подготовки на компьютер с Windows 10. Если при этом на ПК с Windows 10 также возникает сбой, выполните инструкции по устранению неполадок с аутентификацией 802.1X для клиента Windows.
4. Отправьте нам отзыв через Центр отзывов.

## <a name="wi-fi-troubleshooting"></a>Устранение неполадок с Wi-Fi

Ниже приведены некоторые действия, которые рекомендуется выполнить, если вы не можете подключить HoloLens к сети Wi-Fi:

1. Убедитесь, что интерфейс Wi-Fi включен. Для этого выполните жест "Пуск", а затем выберите "Параметры" > "Сеть и Интернет" > "Wi-Fi". Если интерфейс Wi-Fi включен, попробуйте отключить его и снова включить.
2. Переместитесь ближе к маршрутизатору или точке доступа.
3. Перезапустите маршрутизатор Wi-Fi, а затем перезапустите HoloLens. Повторите попытку подключения.
4. Если ни одно из этих действий не помогло, убедитесь, что на маршрутизаторе используется последняя версия встроенного ПО. Эти сведения можно найти на веб-сайте изготовителя.

Когда выполняется вход в учетную запись предприятия или организации на устройстве, также может применяться политика управления мобильными устройствами (MDM), если она настроена ИТ-администратором.

## <a name="network-troubleshooting"></a>Устранение неполадок с сетью
Если проблемы с сетью препятствуют развертыванию и успешному использованию HoloLens 2 в организации, настройте сбор пактов и анализ трафика HTTP/HTTPS с помощью Fiddler и/или Wireshark. 

### <a name="configure-fiddler-to-capture-http-traffic"></a>Настройка Fiddler для сбора пактов трафика HTTP
Fiddler — это прокси-сервер отладки. Он используется для устранения проблем с HTTP/HTTPS. Fiddler фиксирует каждый HTTP-запрос от компьютера и записывает все данные, связанные с этими запросами. Обнаружение проблем с проверкой подлинности пользователей в ваших приложениям, использующих протокол HTTPS, позволяет повысить производительность и эффективность для целевых вариантов использования HoloLens 2. 

#### <a name="prerequisites"></a>Предварительные требования
 
- Устройства HoloLens 2 и ваш компьютер должны работать в одной сети.
- Запишите IP-адрес своего компьютера.

#### <a name="install-and-configure-fiddler"></a>Установка и настройка Fiddler

1. На своем компьютере [установите](https://docs.telerik.com/fiddler-everywhere/get-started/installation-procedure) и запустите Fiddler.  
1. На своем компьютере настройте Fiddler, разрешив подключаться удаленным компьютерам.
    1. В Fiddler выберите "Settings (Параметры)" -> "Connections (Подключения)".
    1. Запишите порт прослушивания для Fiddler (по умолчанию — 8866).
    1. Установите флажок Allow remote computers to connect (Разрешить подключение удаленным компьютерам).
    1. Щелкните Сохранить
3. В HoloLens 2 настройте Fiddler в качестве прокси-сервера<sup>1</sup>:
    1. Откройте меню "Пуск" и выберите раздел "Параметры".
    1. Откройте раздел "Сеть и Интернет", а затем выберите пункт "Прокси-сервер" в меню слева.
    1. Прокрутите окно вниз до раздела "Настройка прокси-сервера вручную" и установите переключатель "Использовать прокси-сервер" в положение "Вкл.".
    1. Введите IP-адрес компьютера, на котором установлено средство Fiddler.
    1. Введите вышеупомянутый номер порта (по умолчанию — 8866).
    1. Щелкните Сохранить

<sup>1</sup> Для сборки 20279.1006 и более поздних версий (в рамках программы предварительной оценки и для будущих выпусков), прокси-сервер необходимо настраивать следующим образом:
1. Откройте меню "Пуск" и перейдите на страницу свойств сети Wi-Fi. 
1. Прокрутите вниз до раздела "Прокси-сервер".
1. Выберите "Настройка вручную".
1. Введите IP-адрес компьютера, на котором установлено средство Fiddler.
1. Введите вышеупомянутый номер порта (по умолчанию — 8866).
1. Нажмите кнопку Apply (Применить).
    
#### <a name="decrypt-https-traffic-from-hololens-2"></a>Расшифровка трафика HTTPS из HoloLens 2

1. На своем компьютере экспортируйте сертификат Fiddler.
    1. В Fiddler выберите "Settings (Параметры)" -> "HTTPS" и разверните раздел Advanced Settings (Дополнительные параметры).
    2. Нажмите Export Fiddler certificate (Экспорт сертификата Fiddler). Сертификат сохранится на компьютере.
    3. Переместите сертификат в папку "Загрузки" в HoloLens 2.

2.  В HoloLens 2 импортируйте сертификат Fiddler.
    1. Выберите "Параметры -> Обновление и безопасность -> Сертификаты".
    2. Нажмите "Установить сертификат", перейдите в папку "Загрузки" и выберите сертификат Fiddler.
    3. Измените значение параметра "Расположение хранилища" на "Локальный компьютер".
    4. Установите корневое хранилище для параметра "Хранилище сертификата".
    5. Нажмите "Установить".
    6. Проверьте, есть ли сертификат в списке сертификатов. Если он отсутствует, выполните шаги выше снова.

#### <a name="inspect-https-sessions"></a>Проверка сеансов HTTP/HTTPS

Fiddler на вашем компьютере будет показывать активные сеансы HTTP/HTTPS HoloLens 2. HTTP/HTTPS-запросы и ответы на панели Inspectors (Инспекторы) в Fiddler могут отображаться в разных представлениях. Например, в представлении Raw (Необработанные) будет показан необработанный запрос или ответ в виде обычного текста. 

### <a name="configure-wireshark-to-capture-network-traffic"></a>Настройка Wireshark для сбора пакетов сетевого трафика
Wireshark — это анализатор сетевых протоколов, который используется для проверки трафика TCP/UDP из устройств HoloLens 2 и к ним. Он позволяет с легкостью определить, какой трафик проходит через вашу сеть HoloLens 2, каковы его объем, частота, задержки между определенными прыжками и т. п.

#### <a name="prerequisites"></a>Предварительные требования:
- Компьютер должен иметь доступ к Интернету и поддерживать использование Общего Интернета в сети Wi-Fi.

#### <a name="install-and-configure-wireshark"></a>Установка и настройка Wireshark
1. На своем компьютере установите [Wireshark](https://www.wireshark.org/#download). 
1. На своем компьютере включите мобильный хот-спот для совместного использования подключения к Интернету в сети Wi-Fi.
1. На своем компьютере запустите Wireshark и настройте сбор пакетов трафика из интерфейса мобильного хот-спота. 
1. В HoloLens 2 измените сеть Wi-Fi на мобильный хот-спот компьютера. IP-трафик HoloLens 2 будет отображаться в Wireshark.

#### <a name="analyze-wireshark-logs"></a>Анализ журналов Wireshark
Фильтры Wireshark позволяют просматривать интересующие пакеты. 

Ознакомьтесь с исходной [записью в блоге](https://techcommunity.microsoft.com/t5/windows-it-pro-blog/diagnose-hololens-2-network-issues-with-fiddler-and-wireshark/ba-p/2322458).

## <a name="cant-sign-in-to-a-previously-setup-hololens-device"></a>Не удается войти в настроенное ранее устройство HoloLens

Если устройство было ранее настроено для другого пользователя (клиента или бывшего сотрудника) и у вас нет пароля для разблокировки устройства, с помощью Intune вы можете выполнить удаленную [очистку](/intune/remote-actions/devices-wipe) устройства. После этого устройство самостоятельно установит нужный образ.  
> [!IMPORTANT]
> При очистке устройства не устанавливайте флажок **Сохранить состояние регистрации и учетную запись пользователя**.

## <a name="cant-login-after-updating-to-windows-holographic-21h1"></a>Не удается войти после обновления до Windows Holographic 21H1

### <a name="symptoms"></a>Симптомы
- При вводе правильного PIN-кода для входа возникает ошибка.
- При использовании метода входа через Интернет возникает ошибка после успешного входа на веб-странице.
- Устройство не имеет статуса "Присоединено к Azure AD" на [портале Azure](https://portal.azure.com/) (Azure Active Directory -> Устройства).

### <a name="cause"></a>Причина
Возможно, устройство с ошибкой удалено из арендатора Azure AD. Например, это могло произойти по следующей причине:

- Администратор или пользователь удалил устройство на портале Azure или с помощью PowerShell.
- Устройство было удалено из арендатора Azure AD из-за неактивности. Для эффективного управления средой мы обычно рекомендуем ИТ-администраторам [удалять бездействующие, неактивные устройства из арендатора Azure AD](/azure/active-directory/devices/manage-stale-devices).

Когда устройство с ошибкой попытается повторно обратиться к арендатору Azure AD после его удаления, его аутентификация в Azure AD завершится сбоем. Пользователь устройства часто остается в неведении о такой проблеме, так как кэшируемый вход с помощью PIN-кода будет разрешать пользователю вход.

### <a name="mitigation"></a>Меры по снижению риска
Пока нет способа добавить удаленное устройство HoloLens обратно в Azure AD. Для затронутых устройств необходимо будет выполнить чистую установку образа с помощью [соответствующих инструкций](hololens-recovery.md#clean-reflash-the-device).

## <a name="managed-hololens-devices-faqs"></a>Часто задаваемые вопросы об управляемых устройствах HoloLens

### <a name="can-i-use-system-center-configuration-manager-sccm-to-manage-hololens-devices"></a>Можно ли использовать System Center Configuration Manager (SCCM) для управления устройствами HoloLens?

Нет. Для управления устройствами HoloLens необходимо использовать систему MDM.

### <a name="can-i-use-active-directory-domain-services-ad-ds-to-manage-hololens-user-accounts"></a>Можно ли использовать доменные службы Active Directory (AD DS) для управления учетными записями пользователей HoloLens?

Нет. Для управления учетными записями пользователей устройств HoloLens необходимо использовать Azure Active Directory (Azure AD).

### <a name="is-hololens-capable-of-automated-data-capture-systems-adcs-auto-enrollment"></a>Может ли HoloLens выполнять автоматическую регистрацию в системах автоматического сбора данных (ADCS)?

Нет.

### <a name="can-hololens-participate-in-integrated-windows-authentication"></a>Может ли HoloLens выполнять встроенную аутентификацию Windows?

Нет.

### <a name="does-hololens-support-branding"></a>Поддерживает ли HoloLens фирменную символику?

Нет. Но для решения этой проблемы можно воспользоваться одним из следующих подходов:

- Создайте пользовательское приложение, а затем [включите режим киоска](hololens-kiosk.md). Пользовательское приложение может иметь фирменную символику, а также запускать другие приложения (например, Remote Assist).  
- Измените все изображения профилей пользователей в Azure AD на логотип вашей компании. Но в некоторых сценариях это может быть нежелательным.

### <a name="what-logging-capabilities-does-hololens-2-offer"></a>Какие возможности ведения журналов доступны в HoloLens 2?

Ведение журналов ограничено трассировками, которые могут собираться в сценариях разработки или устранения неполадок, или данными телеметрии, которые устройства отправляют на серверы Майкрософт.
