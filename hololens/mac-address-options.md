# <a name="enterprise-enrollment-of-hololens-devices-in-mac-address-restricted-wi-fi-environment"></a>Корпоративная регистрация устройств HoloLens в среде Wi-Fi с ограниченным количеством MAC-адресов

В этом документе будет описан общий сценарий, который был определен в средах клиента, где Wi-Fi ограничен MAC-адресами, или где для подключения к беспроводным сетям требуются сертификаты.

## <a name="example-scenario"></a>Пример сценария

Многие клиенты в безопасных средах имеют ограничения на беспроводные или проводные сети, которые позволяют успешно подключаться только утвержденным устройствам (на основе MAC-адресов). Это может быть реализовано с помощью фильтрации MAC-адресов в точке беспроводного доступа или через DHCP-сервер. Кроме того, в некоторых беспроводных сетях можно использовать для защиты протокол PEAP, при котором для успешной проверки подлинности беспроводной сети необходимо применить сертификат к устройству.

В этом сценарии при подключении устройств HoloLens к сети два ключевых требования могут привести к задержкам или потребовать ручного вмешательства.

- Сертификат Wireless PEAP должен быть применен к устройству до того, как устройство успешно подключится к беспроводной сети.
- MAC-адрес адаптера Wi-Fi HoloLens должен быть зарегистрирован.

Основные проблемы, связанные с приведенными выше требованиями:

1. В настоящее время MAC-адрес можно определить только из приложения "Параметры" на устройстве или из Intune после успешной регистрации.

2. Без MAC-адреса устройство не может присоединиться к сети Wi-Fi для начала регистрации.

3. Для решения этих задач вручную требуется, чтобы с устройством взаимодействовал технический специалист.

## <a name="solutions"></a>Решения

Существует множество способов улучшить эту ситуацию в зависимости от инфраструктуры, доступной в среде.

| Решение | Преимущества | Требования |
| --- | --- | --- |
| Пакет подготовки с адаптером Ethernet | Улучшает интерфейс запуска при первом включении и ускоряет взаимодействие с техническим специалистом. | Совместимый с HoloLens адаптер USB-C Hub + Ethernet. Для записи MAC-адресов и финализации запуска при первом включении по-прежнему потребуется взаимодействие технического специалиста с устройством. |
| Автопилот с регистрацией в Intune через Ethernet | Это одношаговое подключение и регистрация устройства в клиентской среде. Запись MAC-адреса может быть завершена без взаимодействия с устройством | Включение Intune для клиента AAD и совместимого с HoloLens адаптера USB-C Ethernet |
| Автоматизированная отчетность о MAC-адресах | Когда устройства зарегистрированы в клиенте Intune, сценарий может сообщить MAC-адрес техническому специалисту. | Командлеты Intune PowerShell |

## <a name="provisioning-package-with-ethernet-adaptor"></a>Пакет подготовки с адаптером Ethernet

> [!NOTE] 
> Если на проводную сеть также распространяются ограничения MAC, то MAC-адрес адаптера USB-C Hub + Ethernet также должен быть предварительно утвержден. Будьте внимательны при использовании этого адаптера, так как он позволяет получить доступ к сети с других устройств.

### <a name="requirements"></a>Требования

- Порт проводной сети с доступом к клиентской сети
- Совместимый с HoloLens концентратор USB-C с адаптером Ethernet — должен подойти любой адаптер, не требующий дополнительных драйверов или установки приложений.
- Пакет подготовки:
  - Содержащий информацию и сертификат беспроводной сети
  - Содержащий информацию о регистрации для Azure AD организации (необязательно)
  - Содержащий любые другие необходимые параметры подготовки

### <a name="process"></a>Process

Процесс может отличаться в зависимости от уровня программного обеспечения устройства. Если на устройстве установлено [обновление от мая 2004 г.](hololens-release-notes.md#windows-holographic-version-2004), выполните следующие действия.

1. Поместите пакет подготовки в корень USB-накопителя и подключите к концентратору.
2. Подключите кабель Ethernet к адаптеру Hub + Ethernet.
3. Подключите USB-C Hub к устройству HoloLens.
4. Включите HoloLens и наденьте устройство.
5. Нажмите **кнопку уменьшения громкости и кнопку питания**, чтобы применить пакет подготовки.
6. Теперь технический специалист может следить за OOBE, а по завершении открыть приложение "Параметры" и извлечь MAC-адрес устройства.

Если на устройстве установлена сборка ОС до [обновления от мая 2004 г.](hololens-release-notes.md#windows-holographic-version-2004), выполните следующие действия.

1. Включите устройство HoloLens и подключите его к компьютеру.
2. Устройство должно отображаться на компьютере как запоминающее устройство файлов.
3. Скопируйте пакет подготовки на устройство
4. Подключите кабель Ethernet к концентратору.
5. Подключите USB-C Hub к устройству HoloLens.
6. Наденьте HoloLens
7. Нажмите кнопку **уменьшения громкости и кнопку питания**, чтобы применить пакет подготовки.
8. Теперь технический специалист может следить за OOBE, а по завершении открыть приложение "Параметры" и извлечь MAC-адрес устройства.

### <a name="benefits"></a>Преимущества

Это позволит традиционным сенсорным вводом на устройстве применить правильный пакет подготовки и собрать MAC-адрес устройства. [Пакеты подготовки можно создавать с помощью приведенных здесь инструкций.](https://docs.microsoft.com/hololens/hololens-provisioning)

## <a name="autopilot-with-intune-enrollment"></a>Автопилот с регистрацией в Intune

### <a name="requirements"></a>Требования

- Порт проводной сети с доступом к сети клиента
- Устройства HoloLens под управлением Windows Holographic 2004
- Совместимый c HoloLens адаптер USB-C Ethernet
- Настройка и включение Intune для клиента
- Устройство зарегистрировано для автопилота и импортировано в клиент
- Политики Intune, определенные для устройства:
   - Содержащий информацию и сертификат о беспроводной сети
   - Содержащий любые другие необходимые параметры подготовки

Это позволит клиентам с расширенными требованиями к сети регистрировать устройства в рамках более свободного, масштабируемого подхода

Требуются дополнительные предварительные условия, как указано ниже:
1. [Включите клиент для предварительного просмотра автопилота](https://docs.microsoft.com/hololens/hololens2-autopilot).
1. Создайте политики HoloLens, чтобы заменить пакет подготовки в Intune.
1. Создайте политики HoloLens Intune.
1. Назначьте устройства в нужную группу.

### <a name="process"></a>Процесс

1. Подключите кабель Ethernet к адаптеру и подключите адаптер к порту USB-C на устройстве HoloLens 2.

2. Включите HoloLens.

3. Устройство должно автоматически подключаться к Интернету во время запуска при первом включении с помощью адаптера Ethernet. Он должен обнаруживать конфигурацию автопилота и автоматически регистрироваться в Azure AD и Intune.

4. Устройство будет применять необходимые сертификаты Wi-Fi и другие настройки по мере необходимости с помощью Intune.

5. По завершении технический специалист сможет загрузить портал Intune (Endpoint Manager) и перейти на страницу свойств устройства в разделе **Главная -> Устройства -> Имя устройства -> Оборудование**.

6. MAC-адрес Wi-Fi будет виден на портале Intune.

   ![MAC-адрес через Intune](images/mac-address-intune.jpg)

7. Технический специалист добавит этот MAC-адрес в качестве разрешенного устройства.

### <a name="benefits"></a>Преимущества

Это позволит техническому специалисту выполнить развертывание обходным путем, при котором он может зарегистрировать устройство в Azure AD и Intune прямо из коробки, без необходимости носить его или вручную взаимодействовать со средой HoloLens.

## <a name="reporting-of-mac-addresses-to-the-technician"></a>Предоставление MAC-адресов техническому специалисту

### <a name="requirements"></a>Требования

- Авторизация Intune Graph PowerShell для клиента пользователя
- Установка Intune Graph PowerShell на компьютере технического специалиста.
- [https://www.powershellgallery.com/packages/Microsoft.Graph.Intune/6.1907.1.0](https://www.powershellgallery.com/packages/Microsoft.Graph.Intune/6.1907.1.0)
- Доступ для чтения элементам "Управляемые устройства" Intune. (Оператор службы поддержки или более высокого уровня либо пользовательская роль)

В настоящее время не существует "простого" способа запустить команду автоматизации на основе регистрации нового устройства в Intune. Следовательно, эта команда предоставит техническому специалисту простой способ получить MAC-адрес без необходимости входа на портал и извлечения его вручную.

```powershell
Import-Module Microsoft.Graph.Intune

Connect-MSGraph

Get-IntuneManagedDevice -Filter "model eq 'Hololens 2'" | where {$_.enrolledDateTime -gt (get-date).AddDays(-30)}  | select deviceName, wiFiMacAddress 
```

Это позволит вернуть имя и MAC-адрес любых устройств HoloLens, зарегистрированных за последние 30 дней.

![MAC-адрес через PowerShell](images/mac-address-powershell.jpg)

### <a name="process"></a>Процесс

После завершения регистрации Intune технический специалист запустит указанный выше сценарий для извлечения MAC-адреса.
