# <a name="manage-user-identity-and-sign-in-for-hololens"></a>Управление удостоверениями пользователей и входом для HoloLens

> [!NOTE]
> Эта статья является технической ссылкой для ИТ-специалистов и энтузиастов технологий. Если вы ищете инструкции по настройке HoloLens, читайте в публикации "Настройка[holoLens (1-го поколения)](hololens1-start.md)" или["Настройка holoLens 2".](hololens2-start.md)

Как и другие устройства Windows, HoloLens всегда работает в пользовательском контексте. Всегда существует идентификатор пользователя. HoloLens обрабатывает удостоверения почти так же, как и другие устройства с Windows 10. Эта статья является ссылкой на удостоверение в HoloLens и посвящена отличиям HoloLens от других устройств с Windows 10.

HoloLens поддерживает несколько типов удостоверений пользователей. Для регистрации можно использовать одну или несколько учетных записей пользователей. Ниже представлен обзор типов удостоверений и параметров проверки подлинности в HoloLens:

| Тип удостоверения | Учетные записи на устройство | Параметры проверки подлинности |
| --- | --- | --- |
| [Azure Active Directory](https://docs.microsoft.com/azure/active-directory/) (требуется Azure AD Premium) | 64 | <ul><li>Поставщик веб-учетных данных Azure</li><li>Приложение Azure Authenticator</li><li>Биометрический (Iris) &ndash; HoloLens 2 только <sup> 1</sup> </li><li>&ndash;PIN-код Необязательно для HoloLens (1-го поколения), необходимый для HoloLens 2</li><li>Пароль</li></ul> |
| [Учетная запись Майкрософт (MSA)](https://docs.microsoft.com/windows/security/identity-protection/access-control/microsoft-accounts) | 1 | <ul><li>Только для биометрических (Iris) &ndash; HoloLens 2</li><li>&ndash;PIN-код Необязательно для HoloLens (1-го поколения), необходимый для HoloLens 2</li><li>Пароль</li></ul> |
| [Локализованная учетная запись](https://docs.microsoft.com/windows/security/identity-protection/access-control/local-accounts) | 1 | Пароль |

Облачные учетные записи (Azure AD и MSA) предоставляют больше возможностей, так как они могут использовать службы Azure.  

> [!NOTE]
> 1. Хотя устройство HoloLens 2 может поддерживать до 64 учетных записей Azure AD, только 10 из этих учетных записей могут зарегистрироваться в проверке подлинности Iris. Это согласуется с другими [вариантами биометрической](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-faq#how-many-users-can-enroll-for-windows-hello-for-business-on-a-single-windows-10-computer)проверки подлинности для Windows Hello для бизнеса.

## <a name="setting-up-users"></a>Настройка пользователей

Наиболее распространенный способ настроить нового пользователя — это во время работы holoLens вне окна (OOBE). Во время настройки HoloLens подсказывают пользователю войти с помощью учетной записи, которую он хочет использовать на устройстве. Эта учетная запись может быть учетной записью Майкрософт или корпоративной учетной записью, настроенной в Azure. См. [параметры Настройка HoloLens (1-го поколения)](hololens1-start.md) или [HoloLens 2](hololens2-start.md).

Как и Windows на других устройствах, вход во время настройки создает профиль пользователя на устройстве. Профиль пользователя хранит приложения и данные. Эта же учетная запись также предоставляет один вход для таких приложений, как Edge или Skype с помощью API-API диспетчера учетной записи Windows.  

Если для регистрации в HoloLens используется корпоративная или организационная учетная запись, HoloLens зачислена в ИТ-инфраструктуру организации. Эта регистрация позволяет ИТ-администратору настроить управление мобильными устройствами (MDM) для отправки групповых политик в HoloLens.

По умолчанию, как и для других устройств с Windows 10, вам придется снова войти, когда HoloLens перезапустит или возобновит работу из ожидания. Вы можете использовать приложение Параметры, чтобы изменить это поведение, или поведение можно контролировать групповой политикой.

### <a name="linked-accounts"></a>Связанные учетные записи

Как и в настольной версии Windows, дополнительные учетные данные веб-учетной записи можно связать с учетной записью HoloLens. Такая связь упрощает доступ к ресурсам в приложениях или в приложениях (например, в Магазине) или к доступу к личным и work ресурсам. После подключения учетной записи к устройству можно предоставить разрешение на использование устройства приложениями, чтобы не входил в каждое приложение по отдельности.

Привязка учетных записей не отделяет пользовательские данные, созданные на устройстве, например изображения или скачивания.  

### <a name="setting-up-multi-user-support-azure-ad-only"></a>Настройка поддержки нескольких пользователей (только Azure AD)

HoloLens поддерживает несколько пользователей из одного клиента Azure AD. Чтобы использовать эту функцию, необходимо использовать учетную запись, которая принадлежит вашей организации, чтобы настроить устройство. Впоследствии другие пользователи из того же клиента могут войти на устройство с экрана входной записи или нажав на плитку пользователя на панели Начните. Только один пользователь может быть подписан одновременно. При вступлении пользователя HoloLens подписывает предыдущего пользователя. Первый пользователь на устройстве считается владельцем устройства, за исключением случая с Azure AD Join, дополнительные данные [о владельцах устройств.](security-adminless-os.md#device-owner)

Все пользователи могут использовать приложения, установленные на устройстве. Однако у каждого пользователя есть свои данные и предпочтения приложения. Удаление приложения с устройства удаляет его для всех пользователей.  

Устройства, установленные с учетными записями Azure AD, не разрешят вход на устройство с учетной записью Microsoft. Все последующие учетные записи, используемые, должны быть учетными записями Azure AD из того же клиента, что и устройство. Вы по-прежнему [можете войти с помощью учетной записи Майкрософт](hololens-identity.md#setting-up-multi-user-support-azure-ad-only) в поддерживаемые приложения (например, в Microsoft Store). Чтобы изменить использование учетных записей Azure AD на учетные записи Microsoft Accounts для регистрации на устройстве, необходимо переименовывать [устройство.](hololens-recovery.md#clean-reflash-the-device)

> [!NOTE]
> **HoloLens (1-й ген)** начал поддерживать несколько пользователей Azure AD в [обновлении Windows 10 апреля 2018](https://docs.microsoft.com/windows/mixed-reality/release-notes-april-2018) г. в рамках [Windows Holographic для бизнеса.](hololens-upgrade-enterprise.md)

## <a name="removing-users"></a>Удаление пользователей

Вы можете удалить пользователя с устройства, переехав в **Параметры**  >  **учетных записей**  >  **Других людей.** Это действие также возвращает пространство, удаляя все данные приложения этого пользователя с устройства.  

## <a name="using-single-sign-on-within-an-app"></a>Использование единого входного знака в приложении

Как разработчик приложений, вы можете использовать связанные удостоверения в HoloLens с помощью [API диспетчера](https://docs.microsoft.com/uwp/api/Windows.Security.Authentication.Web.Core)учетных записей Windows, как и на других устройствах Windows. Некоторые примеры кода для этих API доступны в GitHub: [пример управления веб-учетной записью.](https://go.microsoft.com/fwlink/p/?LinkId=620621)

Любые прерывания учетной записи, например запрос согласия пользователя на сведения об учетной записи, двух-факторная проверка подлинности и так далее, должны обрабатываться, когда приложение запрашивает маркер проверки подлинности.

Если вашему приложению требуется определенный тип учетной записи, который ранее не был связан, приложение может попросить систему попросить пользователя добавить его. Этот запрос запускает области параметров учетной записи для запуска в качестве модального ребенка вашего приложения. Для 2D-приложений это окно отрисовка непосредственно по центру приложения. Для приложений Unity этот запрос ненадолго вынимает пользователя из голографического приложения для отображения окна ребенка. Сведения о настройке команд и действий на этой области см. в [webAccountCommand Class.](https://docs.microsoft.com/uwp/api/Windows.UI.ApplicationSettings.WebAccountCommand)

## <a name="enterprise-and-other-authentication"></a>Корпоративная и другая проверка подлинности

Если ваше приложение использует другие типы проверки подлинности, такие как NTLM, Basic или Kerberos, вы можете использовать пользовательский интерфейс [учетных](https://docs.microsoft.com/uwp/api/Windows.Security.Credentials.UI) данных Windows для сбора, обработки и хранения учетных данных пользователя. Пользовательский интерфейс для сбора этих учетных данных очень похож на другие прерывания учетной записи, управляемые облаком, и отображается как детское приложение в верхней части вашего 2D-приложения или на короткое время приостанавливать приложение Unity для показа пользовательского интерфейса.

## <a name="deprecated-apis"></a>Неподготовленные API

Один из способов, при котором разработка для HoloLens отличается от разработки для Desktop, — это то, что API [OnlineIDAuthenticator](https://docs.microsoft.com/uwp/api/Windows.Security.Authentication.OnlineId.OnlineIdAuthenticator) не поддерживается полностью. Хотя API возвращает маркер, если основная учетная запись находится в исправном положении, прерывания, такие как описанные в этой статье, не отображают пользовательского интерфейса пользователя и не удается правильно проверить подлинность учетной записи.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

### <a name="is-windows-hello-for-business-supported-on-hololens-1st-gen"></a>Поддерживается ли Windows Hello для бизнеса в HoloLens (1-й gen)?

Windows Hello для бизнеса (которая поддерживает использование ПИН-кода для входов) поддерживается для HoloLens (1-й Gen). Чтобы разрешить вход ПИН-кода Windows Hello для бизнеса в HoloLens:

1. Устройство HoloLens должно [управляться MDM.](hololens-enroll-mdm.md)
1. Для устройства необходимо включить Windows Hello для бизнеса. ([См. инструкции для Microsoft Intune.](https://docs.microsoft.com/intune/windows-hello))
1. В HoloLens пользователь может использовать **параметры параметров**параметров для добавления  >  **ПИН-кода**для  >  **** настройки ПИН-кода.

> [!NOTE]
> Пользователи, которые въехают с помощью учетной записи Майкрософт, также могут настроить ПИН-код в **параметрах Параметры**входного параметров Добавить  >  ****  >  **ПИН-код.** Этот ПИН-код связан с [Windows Hello,](https://support.microsoft.com/help/17215/windows-10-what-is-hello)а не [с Windows Hello для бизнеса.](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-overview)

### <a name="how-is-iris-biometric-authentication-implemented-on-hololens-2"></a>Как реализуется биометрическая проверка подлинности Iris на HoloLens 2?

HoloLens 2 поддерживает проверку подлинности Iris. Iris основан на технологии Windows Hello и поддерживается для использования как Azure Active Directory, так и учетных записей Майкрософт. Iris реализуется так же, как и другие технологии Windows Hello, и достигает биометрической безопасности FAR 1/100K.

Дополнительные сведения см. в биометрических требованиях и [спецификациях для Windows Hello.](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello-biometric-requirements) Дополнительные информацию о [Windows Hello](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello) и Windows Hello [для бизнеса](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-identity-verification). 

### <a name="how-does-the-type-of-account-affect-sign-in-behavior"></a>Как тип учетной записи влияет на поведение входных записей?

При применении политик для входа в систему политики всегда соблюдаются. Если политика для входной записи не применяется, это поведение по умолчанию для каждого типа учетной записи:

- **Azure AD:** просит проверить подлинность по умолчанию и настроить **параметры,** чтобы больше не просить проверку подлинности.
- **Учетная**запись Майкрософт. Поведение блокировки отличается от автоматической разблокировки, однако при перезагрузке по-прежнему требуется вход в проверку подлинности.
- **Локализованная**учетная запись: всегда просит проверить подлинность в виде пароля, не настраиваемого в **Параметры**

> [!NOTE]
> В настоящее время не поддерживаются неактивные хронометверы, что означает, что политика **AllowIdleReturnWithoutPassword** соблюдается только при переходе устройства в режим StandBy.
