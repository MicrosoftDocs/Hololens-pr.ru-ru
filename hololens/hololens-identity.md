# Управление удостоверениями пользователей и входом для HoloLens

> [!NOTE]
> Эта статья является техническим справочником для ИТ-специалистов и технических энтузиастов. Если вы ищете инструкции по настройке HoloLens, прочитайте "Настройка[HoloLens (1-е поколения)](hololens1-start.md)" или["Настройка HoloLens 2".](hololens2-start.md)

Как и другие устройства с Windows, HoloLens всегда работает в контексте пользователя. Всегда существует удостоверение пользователя. HoloLens обрабатывает удостоверения почти так же, как и другие устройства с Windows 10. Эта статья является справочной статьей по удостоверениям на HoloLens и посвящена отличию HoloLens от других устройств с Windows 10.

HoloLens поддерживает несколько типов удостоверений пользователей. Для этого можно использовать одну или несколько учетных записей пользователей. Ниже представлен обзор типов удостоверений и параметров проверки подлинности на HoloLens.

| Тип удостоверения | Учетные записи на устройство | Параметры проверки подлинности |
| --- | --- | --- |
| [Azure Active Directory (Azure AD)](https://docs.microsoft.com/azure/active-directory/) | 64 | <ul><li>Поставщик веб-учетных данных Azure</li><li>Приложение Azure Authenticator</li><li>Биометрические (Iris) &ndash; HoloLens 2 только <sup> 1</sup> </li><li>Pin &ndash; Optional for HoloLens (1-е gen), required for HoloLens 2</li><li>Пароль</li></ul> |
| [Учетная запись Майкрософт (MSA)](https://docs.microsoft.com/windows/security/identity-protection/access-control/microsoft-accounts) | 1 | <ul><li>Только &ndash; holoLens 2 для биометрических данных (Iris)</li><li>Pin &ndash; Optional for HoloLens (1-е gen), required for HoloLens 2</li><li>Пароль</li></ul> |
| [Локализованная учетная запись](https://docs.microsoft.com/windows/security/identity-protection/access-control/local-accounts) | 1 | Пароль |

Облачные учетные записи (Azure AD и MSA) предоставляют больше возможностей, так как они могут использовать службы Azure.  

> [!NOTE]
> 1 — хотя устройство HoloLens 2 может поддерживать до 64 учетных записей Azure AD, только 10 из них могут зарегистрироваться в проверке подлинности Iris. Это согласуется с другими вариантами биометрической проверки подлинности для Windows Hello для бизнеса. [Дополнительные ознакомьтесь с этой темой.](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-faq#how-many-users-can-enroll-for-windows-hello-for-business-on-a-single-windows-10-computer)

## Настройка пользователей

Наиболее распространенный способ настроить нового пользователя — это во время работы holoLens при при вступлении в OOBE. Во время настройки HoloLens запросит у пользователя вход с использованием учетной записи, которую он хочет использовать на устройстве. Эта учетная запись может быть учетной записью Майкрософт потребителя или корпоративной учетной записью, настроенной в Azure. См. настройку [HoloLens (1-е поколения)](hololens1-start.md) или [HoloLens 2.](hololens2-start.md)

Как и Windows на других устройствах, при входе во время установки создается профиль пользователя на устройстве. В профиле пользователя хранится приложение и данные. Та же учетная запись также предоставляет единый вход для таких приложений, как Edge или Skype, с помощью API диспетчера учетных записей Windows.  

Если для вход в HoloLens используется корпоративная или организационная учетная запись, HoloLens регистрется в ИТ-инфраструктуре организации. Эта регистрация позволяет ИТ-администратору настроить управление мобильными устройствами (MDM) для отправки групповых политик на устройство HoloLens.

По умолчанию, как и для других устройств с Windows 10, вам придется снова войти, когда HoloLens перезагрузится или возобновит работу из режимов ожидания. Вы можете использовать приложение "Параметры" для изменения этого поведения или управлять поведением с помощью групповой политики.

### Связанные учетные записи

Как и в настольной версии Windows, вы можете связать дополнительные учетные данные веб-учетной записи с учетной записью HoloLens. Такое связывание упрощает доступ к ресурсам в приложениях (например, в Магазине) или объединение доступа к личным и work ресурсам. После подключения учетной записи к устройству можно предоставить разрешение на использование устройства в приложениях, чтобы вам не нужно было в индивидуальном входе в каждое приложение.

Связывание учетных записей не разделяет пользовательские данные, созданные на устройстве, например изображения или загрузки.  

### Настройка поддержки нескольких пользователей (только в Azure AD)

HoloLens поддерживает нескольких пользователей из одного клиента Azure AD. Чтобы использовать эту функцию, необходимо использовать учетную запись, которая принадлежит вашей организации, чтобы настроить устройство. Впоследствии другие пользователи из того же клиента могут войти на устройство с экрана регистрации или нажав плитку пользователя на панели "Начните". Только один пользователь может быть одновременно вписаться в учетную записи. Когда пользователь войду, HoloLens выпишет предыдущего пользователя. Первый пользователь на устройстве считается владельцем устройства, за исключением случаев присоединить Azure AD, узнайте больше [о владельцах устройств.](security-adminless-os.md#device-owner)

Все пользователи могут использовать установленные на устройстве приложения. Однако каждый пользователь имеет свои собственные данные и предпочтения приложения. При удалении приложения с устройства оно удаляется для всех пользователей.  

Устройства с учетными записями Azure AD не разрешят вход на устройство с помощью учетной записи Майкрософт. Все последующие используемые учетные записи должны быть учетными записями Azure AD из того же клиента, что и устройство. Вы по-прежнему можете войти с помощью учетной [записи Майкрософт в приложения, которые](hololens-identity.md#setting-up-multi-user-support-azure-ad-only) ее поддерживают (например, Microsoft Store). Чтобы изменить использование учетных записей Azure AD на учетные записи Майкрософт для входов на устройство, необходимо отменить [его.](hololens-recovery.md#clean-reflash-the-device)

> [!NOTE]
> **HoloLens (1-е поколения)** начал поддерживать нескольких пользователей Azure AD в обновлении Windows 10 за апрель [2018](https://docs.microsoft.com/windows/mixed-reality/release-notes-april-2018) г. в составе [Windows Holographic for Business.](hololens-upgrade-enterprise.md)

## Использование единого вход в приложении

Как разработчик приложений, вы можете использовать связанные удостоверения на HoloLens с помощью API диспетчера учетных записей [Windows,](https://docs.microsoft.com/uwp/api/Windows.Security.Authentication.Web.Core)как и на других устройствах с Windows. Некоторые примеры кода для этих API доступны на сайте GitHub: пример управления [веб-учетной записью.](https://go.microsoft.com/fwlink/p/?LinkId=620621)

Любые прерывания учетной записи, такие как запрос согласия пользователя на информацию об учетной записи, двух-факторная проверка подлинности и так далее, должны обрабатываться, когда приложение запрашивает маркер проверки подлинности.

Если вашему приложению требуется определенный тип учетной записи, который ранее не был связан, оно может попросить систему попросить пользователя добавить ее. Этот запрос инициирует запуск области параметров учетной записи в качестве модального child вашего приложения. Для двухd-приложений это окно отрисовка непосредственно через центр приложения. Для приложений Unity этот запрос кратковременно отбирает пользователя из голографического приложения для отрисовки окна. Сведения о настройке команд и действий на этой области см. в классе [WebAccountCommand.](https://docs.microsoft.com/uwp/api/Windows.UI.ApplicationSettings.WebAccountCommand)

## Корпоративная и другая проверка подлинности

Если ваше приложение использует другие типы проверки подлинности, такие как NTLM, Basic или Kerberos, вы можете использовать пользовательский интерфейс учетных данных [Windows](https://docs.microsoft.com/uwp/api/Windows.Security.Credentials.UI) для сбора, обработки и хранения учетных данных пользователя. Пользовательский интерфейс для сбора этих учетных данных очень похож на прерывание других облачных учетных записей и отображается в качестве приложения-потомка поверх двухдного приложения или на короткое время приостанавливать приложение Unity для показа пользовательского интерфейса.

## Неподготовленные API

Один из способов, при котором разработка для HoloLens отличается от разработки для настольного компьютера, — это то, что API [OnlineIDAuthenticator](https://docs.microsoft.com/uwp/api/Windows.Security.Authentication.OnlineId.OnlineIdAuthenticator) не полностью поддерживается. Хотя API возвращает маркер, если основная учетная запись находится в корректном положении, прерывания, описанные в этой статье, не отображают пользовательский интерфейс пользователя и не могут правильно проверить подлинность учетной записи.

## Вопросы и ответы

### Поддерживается ли Windows Hello для бизнеса на HoloLens (1-е поколения)?

Для HoloLens (1-го поколения) поддерживается Windows Hello для бизнеса (поддерживает использование ПИН-кода для входов). Чтобы разрешить вход с pin-кода Windows Hello для бизнеса на HoloLens:

1. Устройство HoloLens должно управляться [MDM.](hololens-enroll-mdm.md)
1. Необходимо включить Windows Hello для бизнеса для устройства. ([См. инструкции для Microsoft Intune.](https://docs.microsoft.com/intune/windows-hello))
1. На HoloLens пользователь может **** использовать параметры для добавления ПИН-кода для настройки  >  **ПИН-кода**с помощью параметров  >  **** входов.

> [!NOTE]
> Пользователи, вы входившие с помощью учетной **** записи Майкрософт, также могут настроить ПИН-код в параметрах для добавления  >  **ПИН-кода.**  >  **** Этот ПИН-код связан с [Windows Hello,](https://support.microsoft.com/help/17215/windows-10-what-is-hello)а [не с Windows Hello для бизнеса.](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-overview)

### Как биометрическая проверка подлинности Iris реализована на HoloLens 2?

HoloLens 2 поддерживает проверку подлинности радужной оболочки радужной оболочки. Iris основан на технологии Windows Hello и поддерживается как в Azure Active Directory, так и в учетных записях Майкрософт. Iris реализован так же, как другие технологии Windows Hello, и достигает биометрической безопасности FAR 1/100K.

Подробнее о биометрических требованиях и спецификациях для Windows Hello можно узнать [здесь.](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello-biometric-requirements) Узнайте больше о [Windows Hello](https://docs.microsoft.com/windows-hardware/design/device-experiences/windows-hello) и Windows Hello [для бизнеса.](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-identity-verification) 

### Как тип учетной записи влияет на поведение при входе?

При применении политик для входа в систему политики всегда соблюдаются. Если политика для входов не применяется, для каждого типа учетной записи применяются указанные по умолчанию действия.

- **Azure AD**: по умолчанию запросит проверку подлинности и настраивается с помощью параметров, чтобы больше не запросить проверку подлинности. ****
- **Учетная запись Майкрософт:** режим блокировки отличается от автоматической разблокировки, но при перезагрузке по-прежнему требуется проверка подлинности при входе.
- **Локализованная**учетная запись: всегда запросит проверку подлинности в виде пароля, не настраиваемую в **параметрах**

> [!NOTE]
> В настоящее время неактивные timers не поддерживаются, что означает, что политика **AllowIdleReturnWithoutPassword** соблюдается только при переходе устройства в режим ожидания.
