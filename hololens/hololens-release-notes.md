    - LoopbackGain (текущее значение "на странице" смешанная работа приложения "на вкладке" устройства "в Windows Device Portal)
    - MicrophoneGain (текущее значение "микрофона" на странице "смешанная аудиосистема" на портале устройств Windows)
- Это обновление включает исправление ошибки, повышающее качество звука в сценариях захвата в смешанной реальности. В частности, при отображении меню "Пуск" необходимо устранить сбой звука в записи.

- В этой статье описано решение проблемы, при которой в ходе смешанной реальности не удается записать видео после того, как устройство останется в состоянии ожидания в течение нескольких дней.

UserPresence обычно отключен для приложений Unity, чтобы избежать проблем, которые приводят к приостановке некоторых приложений при отражении гипервизора, даже если параметр для выполнения в фоновом режиме включен. Теперь API разрешены для версий Unity 2018.4.18 и выше, а также 2019.3.4 и выше.
- При доступе к порталу устройства через WiFi-соединение может препятствовать доступу к Интернету, если он не является допустимым сертификатом, сообщение об ошибке, например "ERR_SSL_PROTOCOL_ERROR", даже если сертификат устройства уже надежен.  В этом случае вы не сможете выполнить действия на портале устройств, так как параметры для игнорирования предупреждений системы безопасности недоступны.  Это обновление устраняет проблему.  Если сертификат устройства был ранее скачан и надежен на компьютере для удаления предупреждений системы безопасности браузера и обнаружена ошибка SSL, необходимо загрузить новый сертификат и получить доступ к предупреждениям системы безопасности для обозревателя адресов.
- Включена возможность создания пакета подготовки среды выполнения, который может установить приложение с помощью пакетов MSIX.

- Исправлена проблема, из-за которой приложения HoloLens изменяют формат пикселей для отображения черного в эмуляторе HoloLens.
- Исправлена ошибка, из-за которой при входе в IRI произошел сбой.
- Устранена проблема, связанная с повторением загрузок магазина для уже существующих приложений.
- Устранена ошибка, из-за которой не удается запустить иммерсивное приложение несколько вхождений.
- Устранена проблема, связанная с запуском приложения "фотографии" в начальной загрузке после обновления из выпуска 1903.
- Повышенная производительность и надежность.

### Поддержка Windows для автопилота 

Windows Pilot для HoloLens 2 позволяет каналу продаж устройства предварительно зарегистрировать HoloLens в клиенте Intune.  Когда устройства прибывают, они готовы к развертыванию как общие устройства в рамках вашего клиента. Чтобы воспользоваться преимуществами саморазвертывания, устройства должны подключаться к сети во время первого экрана программы установки, используя один из портов USB-C для Ethernet или USB-C на ключ LTE. 

Когда пользователь запускает саморазвертывание Autopilot, этот процесс выполняет следующие действия: 

1. Присоединение устройства к Azure Active Directory (Azure AD). 
1. Регистрация устройства в Microsoft Intune (или другой службе MDM) с помощью Azure AD. 
1. Скачивание политик для устройств, сертификатов и сетевых профилей. 
1. Подготовка устройства. 
1. Демонстрация экрана входа пользователю. 

Дополнительные сведения о [пробных руководствах по Windows "Автопилот" для HoloLens 2](https://docs.microsoft.com/hololens/hololens2-autopilot).

**Чтобы присоединиться к программе автопилота, обратитесь к своему менеджеру по учетным записям. Готовые устройства для автоматического пилотного развертывания скоро будут доставляться.**

### Поддержка ключа безопасности FIDO2 

Многие из вас представляют собой устройство HoloLens с большим количеством людей в рабочей или учебной среде. При совместном доступе к устройствам между студентами в аудитории или их извлеченными из него вы должны иметь возможность быстро и легко изменять пользователей, не вводя длинные имена пользователя и пароли. 

FIDO позволяет любому пользователю в вашей организации (в AAD-клиенте) легко войти в HoloLens, не вводя имя пользователя или пароль. 

Ключи безопасности FIDO2 — это способ проверки подлинности без фишинга, который может быть основан на любом конструктивном методе. Быстрая идентификация Online (FIDO) — это открытый стандарт для проверки подлинности с использованием пароля. FIDO позволяет пользователям и организациям использовать стандарт для входа в свои ресурсы без имени пользователя или пароля с помощью внешнего ключа безопасности или ключа платформы, встроенного в устройство. 

Чтобы приступить к работе, прочтите документ по безопасности, не защищенный [паролем](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-security-key) . 

### Улучшенная регистрация в MDM с помощью пакета подготовки 

Пакеты подготовки позволяют настроить конфигурацию HoloLens с помощью файла конфигурации, а не через поле "интерфейс HoloLens". Ранее предоставлялись пакеты, которые были скопированы в внутреннюю память HoloLens, теперь они могут быть на USB-накопителе, чтобы их было проще использовать на нескольких HoloLens, и другие пользователи могут подготавливать HoloLens параллельно.  Кроме того, пакеты подготовки поддерживают новое поле для регистрации в управлении устройствами, поэтому вы не можете вручную настроить ее после подготовки. 

1. Чтобы попробовать это, скачайте последнюю версию конструктора конфигурации Windows из Магазина Windows на компьютер. 
1. Выберите **подготовить устройства hololens** > выберите пункт **подготовить устройства hololens 2** 
1. Создайте профиль конфигурации и, когда все будет готово, скопируйте все файлы, созданные на устройстве хранения USB-C. 
1. Подключите его к любому обновленному флэш-названию и нажмите клавиши **вниз + Power** , чтобы применить свой пакет подготовки.

В этом выпуске мы добавляем поддержку дополнительных политик, что повышает эффективность администраторов системы управления, развернутых с помощью устройств HoloLens. Список поставщиков служб, поддерживаемых HoloLens, можно найти на этой [странице](https://docs.microsoft.com/windows/client-management/mdm/networkqospolicy-csp). Новая версия в этом выпуске:

[Здесь](https://docs.microsoft.com/windows/client-management/mdm/policies-supported-by-hololens2)вы можете узнать больше о поддерживаемых политиках.

### Расширенная поддержка USB Ethernet для устройств с модемом 5G/LTE

Добавлена поддержка некоторых мобильных устройств мобильной связи, таких как 5G/LTE телефоны и WiFi hotpots, когда модем подключается к HoloLens 2 через USB. Эти устройства будут отображаться в параметрах сети как другое подключение Ethernet. Устройства мобильной широкополосной связи, которым нужен внешний драйвер, не поддерживаются. Это обеспечивает подключение с высокой пропускной способностью в сценариях, в которых WiFi недоступен, и модем WiFi не работает достаточно производительно. Дополнительные сведения о поддерживаемых USB-устройствах можно найти [здесь](https://docs.microsoft.com/hololens/hololens-connect-devices).  

### Улучшенные функции отслеживания

В этом выпуске было получено несколько улучшений, которые отслеживают от руки. 

- **Стабильность — наведите указатель на пункт позы.** Теперь система будет превышена изгибует указатель пальца, когда он превратится в Загороженная с помощью Palm.  Это повышает точность при нажатии кнопок, вводе, прокрутке содержимого и т. д. 
- **Уменьшение случайного AirTaps:** Мы улучшили обнаружение жеста AirTap.  Теперь у вас есть меньше случайных активаций в некоторых распространенных случаях, например при перетаскивании руки на вашу сторону. 
- Повышенная **надежность пользовательского переключателя.** Система стала более быстрой и надежной при обновлении размера при совместном использовании и возврате устройства. 
- Перенос **уменьшенной руки:** Мы улучшили обработку ситуаций, в которых более 2 руки отображаются на датчиках.  Если несколько людей работают вместе, стало еще более низким шанс того, что отслеживаемая рука будет переходить от пользователя к ладони другого человека в сцене. 
- **Надежность системы:** Исправлена проблема, из-за которой отслеживание не работает в течение периода, если устройство находится в высоком загрузку.
