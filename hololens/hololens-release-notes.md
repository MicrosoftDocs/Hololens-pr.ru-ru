# Заметки о выпуске HoloLens 2

Чтобы обеспечить продуктивную работу с устройствами HoloLens, мы по-прежнему выпускаем обновления функций, ошибок и безопасности. На этой странице вы можете видеть новые возможности HoloLens каждый месяц. Чтобы получить последнее обновление HoloLens 2, вы можете либо проверить обновление, либо вручную обновить устройство, либо получить полное обновление флэш-накопителя (FFU) для вспышки устройства с помощью [устройства Advanced Recovery Companion.](hololens-recovery.md#clean-reflash-the-device) [](hololens-update-hololens.md#check-for-updates-and-manually-update) [Загружаемая](https://aka.ms/hololens2download) версия находится в курсе последних общедоступной сборки.

Мы рады снова начать перенаббовку новых функций для пользователей, которые уже являются еетелями. Мы будем перенаться по каналу Dev для последних обновлений. Мы продолжим работу с заметками для пользователей [HoloLens insider](hololens-insider.md) по мере добавления дополнительных функций и обновлений в сборки, внося в нее обновления. Будьте рады и подготовьтесь к тому, чтобы совместить эти обновления с вашей реальностью.

### Установка приложений на HoloLens 2 с помощью установщика приложений

Мы **добавляем новую** возможность (установщик приложений), которая позволяет легко устанавливать приложения на устройства HoloLens 2. Эта функция по умолчанию **будет работать для неугодных устройств.** Во избежание нарушения работы предприятий установщик приложений в настоящее время будет не доступен для **управляемых** устройств.  

Устройство считается управляемым, если **имеется** следующее:
- Регистрация в [](hololens-enroll-mdm.md) MDM
- Настроено с [помощью пакета предоставления](hololens-provisioning.md)
- Удостоверение [пользователя](hololens-identity.md) — Azure AD

Теперь вы можете устанавливать приложения, не влияя на режим разработчика и не используя портал устройств.  Просто скачайте (через USB или через Edge) пакет Appx на свое устройство и перейдите к пакету Appx в проводнике, чтобы получить запрос на начало установки.  Или же [инициировать установку с веб-страницы.](https://docs.microsoft.com/windows/msix/app-installer/installing-windows10-apps-web)  Как и в приложениях, устанавливаемых из Microsoft Store или загрузке неогрузки с помощью [](https://docs.microsoft.com/windows/win32/appxpkg/how-to-sign-a-package-using-signtool) возможности развертывания [](https://docs.microsoft.com/windows/win32/appxpkg/how-to-sign-a-package-using-signtool#security-considerations) LOB-приложений MDM, приложения должны иметь цифровую подпись с помощью средства подписи, а сертификат, используемый для подписи, должен быть доверенным для устройства HoloLens, прежде чем развертывать приложение.

**Инструкции по установке приложения.**

1.  Убедитесь, что устройство не считается управляемым
1.  Убедитесь, что устройство HoloLens 2 подключено и подключено к компьютеру
1.  Убедитесь, что вы вписались на устройство HoloLens 2
1.  На компьютере перейдите к пользовательскому приложению и скопируйте файл yourapp.appxbundle в файл yourdevicename\Internal Storage\Downloads.   Завершив копирование файла, вы можете отключить устройство
1.  На устройстве HoloLens 2 откройте меню "Пуск", выберите все приложения и запустите приложение проводника.
1.  Перейдите в папку "Загрузки". Возможно, вам потребуется сначала выбрать это устройство на левой панели приложения, а затем перейти к файлу "Загрузки".
1.  Выберите файл yourapp.appxbundle.
1.  Запустится установщик приложений. Выберите кнопку "Установить", чтобы установить приложение.
Установленное приложение автоматически запустится после завершения установки.

Примеры приложений можно найти на универсальном сайте [GitHub](https://github.com/microsoft/Windows-universal-samples/tree/master/Samples) для Windows, чтобы протестировать этот поток.

Ознакомьтесь с полным процессом установки приложений на [HoloLens 2 с помощью установщика приложений.](app-deploy-app-installer.md)

### Улучшения и исправления в обновлении:

- Отслеживание рук теперь поддерживает отслеживание во многих новых случаях, когда ранее была бы потеряна рука.  В некоторых из этих новых случаев только положение ладони продолжает обновляться в зависимости от реальной руки пользователя, в то время как другие соединения высмеяются на основе предыдущей позиции.  Это изменение помогает повысить согласованность отслеживания перемещений, таких как пометка, разметка, подкачка и прыжки.  Это также помогает в тех случаях, когда рука близко к поверхности или удерживает объект.  При выводе кисти рук [](https://docs.microsoft.com/uwp/api/windows.perception.people.jointposeaccuracy?view=winrt-19041&preserve-view=true) для каждого совместного соединения будет установлено значение "Approximate" (Приблизительно), а не High (Высокая).
- Исправлена проблема, из-за которой при сбросе ПИН-кода для учетных записей Azure AD была выдана ошибка "Что-то пошло не так.
- При запуске ET, радужной оболочки из приложения параметров, нового пользователя или всплывающее уведомление пользователи должны видеть гораздо меньше сбоя при запуске при загрузке.
- Пользователи должны иметь правильный часовой пояс, исходя из OOBE.

### Поддержка автоматического определения положения глаз

В HoloLens 2 определение положения глаз обеспечивает точное расположение голограмм, удобство просмотра и повышенное качество изображения. Определение положения глаз производится встроенным способом в рамках вычисления отслеживания взгляда. Тем не менее, каждому пользователю для этого нужно пройти калибровку отслеживания взгляда, даже если интерфейс может не требовать управление с помощью взгляда.

**Автоматическое определение положения глаз (AEP)** включает сценарии, предусматривающие вычисление положения глаз пользователя, не требующее его действий. Автоматическое определение положения глаз начинает работать в фоновом режиме с того момента, когда пользователь надевает устройство. Если пользователь ранее не выполнял калибровку для отслеживания взгляда, функция автоматического определения положения глаз начнет передавать информацию о положении глаз пользователя в систему вывода изображения после обработки в течение 20–30 секунд. Данные пользователя не сохраняются на устройстве и, следовательно, этот процесс повторяется, если пользователь снимает устройство и надевает его снова, если устройство перезапускается или выходит из спящего режима.

Если устройство надевает пользователь, не проходивший калибровку, в поведении функции автоматического определения положения глаз происходят некоторые системные изменения. В этом контексте термином "пользователь, не проходивший калибровку" обозначается человек, который ранее не выполнял процесс калибровки отслеживания взгляда на устройстве.

| Активное приложение | Предыдущее поведение | Поведение из обновления Windows Holographic версии 20H2 |
|:-------------------|:-----------------|:-----------------------------------|
| Приложение, не поддерживающее управление взглядом, или Holographic Shell |Отображается диалоговое окно с запросом на проведение калибровки отслеживания взгляда. | Запрос не отображается. |
| Приложение, поддерживающее управление взглядом | Отображается диалоговое окно с запросом на проведение калибровки отслеживания взгляда. | Запрос на проведение калибровки отслеживания взгляда отображается только в том случае, если приложение получает доступ к потоку данных о направлении взгляда. |

Если пользователь переходит из приложения, не поддерживающего управление взглядом, в приложение, которое имеет доступ к данным о направлении взгляда, появится запрос на проведение калибровки. 

Все другие поведения системы будут похожи на действия, когда у текущего пользователя нет активной калибровки отслеживания взгляда. Например, жест "Пуск" одной рукой не будет включен. В интерфейсе первого запуска для начальной настройки не будет изменений.

Для интерфейсов, требующих данных о направлении взгляда или очень точного расположения голограмм, пользователям, не проходившим калибровку, рекомендуется выполнить калибровку отслеживания взгляда. Она доступна из запроса на проведение такой калибровки или из приложения "Параметры" меню "Пуск", где нужно выбрать **Система > Калибровка > Калибровка взгляда > Запуск калибровки взгляда**.

Эти сведения можно найти позже с другими [сведениями калибровки.](hololens-calibration.md#auto-eye-position-support) 

### Диспетчер сертификатов

- Улучшены инструменты аудита, диагностики и проверки для обеспечения безопасности и соответствия устройств требованиям с помощью нового диспетчера сертификатов. Эта возможность позволяет развертывать, устранять неполадки и проверять сертификаты масштабно в коммерческих средах.

В Windows Holographic версии 20H2 мы добавляем диспетчер сертификатов в приложение "Параметры HoloLens 2". Перейдите **в > обновления & безопасности > сертификатов.** Эта функция предоставляет простой и удобным для пользователя способ просмотра, установки и удаления сертификатов на устройстве. С помощью нового диспетчера сертификатов администраторы и пользователи теперь усовершенствовали средства аудита, диагностики и проверки, чтобы обеспечить безопасность и соответствие устройств требованиям. 

-   **Аудит:** Возможность проверки правильности развертывания сертификата или подтверждения правильности его удаления. 
-   **Диагностика:** При возникающих проблемах проверка на существования соответствующих сертификатов на устройстве экономит время и помогает устранять неполадки. 
-   **Проверка:** Проверка того, что сертификат служит назначению и работает, может значительно сэкономить время, особенно в коммерческих средах, прежде чем развертывать сертификаты в крупных масштабах.

Чтобы быстро найти определенный сертификат в списке, можно отсортировать его по имени, даты хранения или срока действия. Пользователи также могут напрямую искать сертификат. Чтобы просмотреть свойства отдельных сертификатов, выберите сертификат и щелкните **"Сведения".** 

В настоящее время установка сертификатов поддерживает CER- и CRT-файлы. Владельцы устройств могут устанавливать сертификаты на локальном компьютере и текущем пользователе;  все остальные пользователи могут устанавливаться только в текущий пользователь. Пользователи могут удалять только сертификаты, установленные непосредственно из пользовательского интерфейса параметров. Если сертификат был установлен другими средствами, он также должен быть удален тем же механизмом.

#### Чтобы установить сертификат:

1.  Поместите файл сертификата, который необходимо установить, в расположение на holoLens 2.

1.  Нажмите кнопку **Установить**.

Сертификат должен быть установлен на устройстве.

### CSP Tenantlockdown и Autopilot
- Устройства сохраняются в клиенте организации за счет принудительной привязки к клиенту, которая сохранятся при сбросе устройства и при установке чистого образа. Для дополнительной безопасности можно запретить создание учетных записей; такой запрет включается в процессе подготовки. 

Устройства HoloLens 2 теперь поддерживают CSP TenantLockdown с [Windows Holographic версии 20H2.](hololens-release-notes.md#windows-holographic-version-20h2) 

Поставщик служб шифрования [TenantLockdown](https://docs.microsoft.com/windows/client-management/mdm/tenantlockdown-csp) дает возможность связать HoloLens 2 с регистрацией MDM с использованием только Autopilot. Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" или "false" (установлено изначально) для HoloLens 2, это значение сохранится на устройстве при установке чистого образа, обновлении ОС и т. д. 

Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" для HoloLens 2, OOBE будет ждать успешной загрузки и применения профиля Autopilot в течение неограниченного времени после установки подключения к сети. 

Если установить для узла RequireNetworkInOOBE поставщика служб шифрования TenantLockdown значение "true" для HoloLens 2, следующие операции в OOBE будут запрещены: 
- Создание локального пользователя с помощью подготовки среды выполнения 
- Выполнение операции присоединения к Azure AD с помощью подготовки среды выполнения 
- Выбор владельца устройства в интерфейсе OOBE 

#### Как выполнить настройку с помощью Intune? 
1. Создайте пользовательский профиль конфигурации устройства OMA URI и укажите значение "true" для узла RequireNetworkInOOBE, как показано ниже.
У параметра OMA-URI должно быть значение ./Vendor/MSFT/TenantLockdown/RequireNetworkInOOBE

   > [!div class="mx-imgBorder"]
   > ![Настройка блокировки клиента через OMA-URI](images/hololens-tenant-lockdown.png)

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте устройство HoloLens 2 участником группы, созданной на предыдущем шаге, и запустите синхронизацию.  

Убедитесь на портале Intune, что конфигурация устройства применена. После применения этой конфигурации устройства к устройству HoloLens 2 будут действовать параметры политики TenantLockdown.

#### Как отменить применение RequireNetworkInOOBE в TenantLockdown для HoloLens 2 с помощью Intune? 
1. Удалите HoloLens 2 из группы устройств, которой ранее была назначена конфигурация устройства, созданная выше. 

1. Создайте пользовательский профиль конфигурации устройства OMA URI и укажите значение "false" для RequireNetworkInOOBE, как показано ниже. У параметра OMA-URI должно быть значение ./Vendor/MSFT/TenantLockdown/RequireNetworkInOOBE

   > [!div class="mx-imgBorder"]
   > ![Снимок экрана: установка для параметра RequireNetworkInOOBE значения "false" через OMA URI в Intune](images/hololens-tenant-lockdown-false.png)

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте устройство HoloLens 2 участником группы, созданной на предыдущем шаге, и запустите синхронизацию.

Убедитесь на портале Intune, что конфигурация устройства применена. После применения этой конфигурации устройства к устройству HoloLens 2 ограничения TenantLockdown перестанут действовать. 

#### Что произойдет в процессе OOBE, если отменить назначение профиля Autopilot устройству HoloLens после установки значения "true" для параметра TenantLockdown? 
Интерфейс OOBE будет ждать загрузки профиля Autopilot в течение неограниченного времени. Появится следующее диалоговое окно. Чтобы удалить ограничения TenantLockdown, необходимо сначала зарегистрировать устройство с первоначальным клиентам, используя только Autopilot. Также необходимо отменить настройку RequireNetworkInOOBE, как описано на предыдущем шаге. После этого ограничения поставщика служб шифрования TenantLockdown будут удалены. 

![Представление на устройстве при применении политики на устройстве.](images/hololens-autopilot-lockdown.png)

Теперь эти сведения можно найти вместе с остальными службами Autopilot в [CSP Tenantlockdown и Autopilot.](hololens2-autopilot.md#tenantlockdown-csp-and-autopilot)

### Кэшное членство в группе Azure AD для автономной терминала
- Включены автономные киоски для использования с группами AAD в течение 60 дней.

Эта политика контролирует, сколько дней можно использовать кэш членства в группах Azure AD для конфигураций с assigned Access, нацеленных на группы Azure AD для во входе пользователя. Если для этого значения политики установлено значение больше 0, то в противном случае используется кэш.  

Имя: Значение URI AADGroupMembershipCacheValidityInDays: ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays

Min — 0 дней  
Максимальное — 60 дней 

Действия для правильного использования этой политики: 
1. Создайте профиль конфигурации устройства для терминала, предназначенного для групп Azure AD, и назначьте его устройствам HoloLens. 
1. Создайте настраиваемую конфигурацию устройства на основе OMA URI, которая задает для этого значения политики необходимое количество дней (> 0) и назначьте его устройствам HoloLens. 
    1. Значение URI должно быть введено в текстовом поле OMA-URI как ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays
    1. Допустимые значения: min/max.
1. Зарегистрировать устройства HoloLens и убедиться, что обе конфигурации применяются к устройству. 
1. Разрешив вход пользователю Azure AD 1, если интернет-доступ доступен, после успешного подтверждения членства пользователя в группе Azure AD кэш будет создан. 
1. Теперь пользователь Azure AD 1 может отсоедидить HoloLens и использовать его для режима терминала, если значение политики допускает X число дней. 
1. Шаги 4 и 5 могут повторяться для любого другого пользователя Azure AD N. Ключевой момент здесь в том, что любой пользователь Azure AD должен войти на устройство с помощью Интернета, чтобы хотя бы один раз определить, что он является членом группы Azure AD, для которой предназначена конфигурация терминала. 
 
> [!NOTE]
> До выполнения шага 4 для пользователя Azure AD будет выполняться сбой, упомянутый в "отключенных" средах.

### Режим исследования
В режиме исследования HoloLens 2 становится средством исследования компьютерного зрения. По сравнению с предыдущими выпусками, режим исследования для HoloLens 2 имеет следующие преимущества:
-   Помимо датчиков, доступных в режиме исследований HoloLens (1-го поколения), теперь мы предоставляем доступ к датчикам IMU, включая акселерометр, гироскоп и магнитометр.
-   HoloLens 2 предоставляет новые возможности, которые можно использовать вместе с режимом исследования. В частности, доступ к API отслеживания рук и отслеживания глаз, которые могут предоставлять более полный набор экспериментов.

Теперь у исследователей есть возможность включения режима исследования на устройствах HoloLens для доступа ко всем внешним потокам датчиков необработанных изображений. Режим исследования для HoloLens 2 также предоставляет доступ к показаниям акселерометра, гироскопа и магнитометра. Чтобы защитить конфиденциальность пользователей, необработанные изображения камеры отслеживания взгляда недоступны в режиме исследования, но направление взгляда доступно через существующие API.

Дополнительные технические [сведения можно узнать в](https://docs.microsoft.com/windows/mixed-reality/research-mode) документации по режиму исследований.

### Увеличена длина записи
Из-за отзывов клиентов мы повысили длину записи записей [смешанной реальности.](holographic-photos-and-videos.md) По умолчанию записи смешанной реальности не ограничиваются 5 минутами, а вычисляют максимальную длину записи на основе доступного дискового пространства. Устройство оценивает максимальную продолжительность видеозаписи на основе доступного места на диске до 80 % от общего дискового пространства.

> [!NOTE]
> HoloLens будет использовать длительность видеозаписи по умолчанию (5 минут), если происходит одно из следующих:
> - Предполагаемая максимальная продолжительность записи меньше 5 минут по умолчанию.
> - Доступное место на диске составляет менее 20 % от общего дискового пространства.

Все требования можно найти в документации по голографическим [фотографиям и видео.](holographic-photos-and-videos.md#maximum-recording-length)

### Поддержка Windows Autopilot

Windows Autopilot для HoloLens 2 позволяет каналу продаж устройств предварительно зарегистрировать HoloLens в клиенте Intune. Когда устройства поступают, они готовы к самостоятельному развертыванию в качестве общих устройств в клиенте. Чтобы воспользоваться преимуществами самостоятельного развертывания, устройство должно подключиться к сети на первом экране установки с помощью USB-C-to-Ethernet.

После того как пользователь запускает процесс самостоятельного развертывания Autopilot, этот процесс завершает следующие действия:

1. Присоединение устройства к Azure Active Directory (Azure AD).
1. Регистрация устройства в Microsoft Intune (или другой службе MDM) с помощью Azure AD.
1. Скачивание политик для устройств, сертификатов и сетевых профилей.
1. Подготовка устройства.
1. Демонстрация экрана входа пользователю.

Узнайте больше из руководства по оценке [Windows Autopilot для HoloLens 2.](https://docs.microsoft.com/hololens/hololens2-autopilot)

*Обратитесь к диспетчеру учетных записей, чтобы присоединиться к предварительной версии AutoPilot. Скоро начнется доставка устройств, готовых к autopilot.*

### Поддержка ключей безопасности FIDO2

Некоторые пользователи совместно работают на устройстве HoloLens с другими пользователями в среде работы или учебного заведения. Поэтому очень важно, чтобы пользователи могли без ввода длинных имен пользователей и паролей. Fast Identity Online (FIDO) позволяет всем пользователям вашей организации (клиент Azure AD) легко входить в HoloLens без ввода имени пользователя или пароля.

Ключи безопасности FIDO2 — это "нефишируемый" метод проверки подлинности без пароля на основе стандартов, который может быть в любом форм-факторе. FIDO — это открытый стандарт для проверки подлинности без пароля. Она позволяет пользователям и организациям вводить данные в свои ресурсы без имени пользователя или пароля. Вместо этого они используют внешний ключ безопасности или ключ платформы, встроенный в устройство.

To get started, see [Enable passwordless security key sign-in](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-security-key).

### Улучшенная регистрация в MDM с помощью пакета предоставления

Пакеты обеспечения можно настроить конфигурацию HoloLens с помощью файла конфигурации, а не с помощью системы HoloLens. Ранее пакеты подготовка необходимо было скопировать во внутреннюю память HoloLens. Пакеты подготовка теперь также поддерживают поле для регистрации в управлении устройствами, поэтому после настройки вручную не будет установлено.

Попробуйте сами:

1. Скачайте последнюю версию конструктора конфигураций Windows из Магазина Windows на компьютер.
1. Выберите **"Подготовка устройств HoloLens"**  >  **для предоставления устройств HoloLens 2.**
2. Создайте профиль конфигурации. Затем скопируйте все файлы, созданные на устройство хранения USB-C.
3. Подключите устройство USB-C к любому новому устройству HoloLens. Затем нажмите кнопки **питания для**отключения  +  **** громкости, чтобы применить пакет подготовка.

### Улучшения отслеживания рук

Этот выпуск включает несколько улучшений отслеживания рук:

- **Pointing pose stability:** Система теперь противядочивает висячий указательный палец, когда он закупоряется ладони. Это изменение повышает точность при нажатии кнопок, введите, прокрутите содержимое и много другое! 
- **Сокращены случайные касания при нажатии:** Мы улучшили обнаружение жеста касания. В ряде распространенных сценариев, например при перенажатии рук на стороны, теперь происходит меньше случайных активаций.
- **Надежность коммутатора пользователя:** Теперь система работает быстрее и надежнее при обновлении размера руки при совместном подсистеме.
- **Уменьшение кражи рук:** Мы улучшили обработку случаев, когда датчики просматривают более двух рук. Если несколько человек работают близко друг к другу, вероятность того, что отслежаемая рука будет "перейти" от пользователя к другому пользователю в сцене, значительно снижается.
- **Надежность системы:** Исправлена проблема, из-за чего отслеживание рук перестает работать, когда устройство находится в условиях высокой нагрузки.
