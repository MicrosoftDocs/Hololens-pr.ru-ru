#### <a name="what-to-expect-from-the-new-microsoft-edge-on-hololens-2"></a>Что должно происходить от нового Microsoft ребра в HoloLens 2

Поскольку новый Microsoft ребр является собственным приложением Win32 с новым уровнем адаптера UWP, позволяющим запускать его на устройствах, работающих только с UWP, таких как HoloLens 2, некоторые функции могут быть недоступны сразу же. Мы будем поддерживать новые сценарии и функции в ближайшие месяцы, поэтому проверяйте это пространство на актуальные сведения.

**Сценарии и функции, которые должны работать:**
- Первый запуск, вход в профиль и синхронизация
- Веб-сайты должны отображаться и работать правильно
- Большинство функций браузера (Избранное, журнал и т. д.) должны работать надлежащим образом
- Темный режим
- Установка веб-приложений на устройстве
- Установка расширений (сообщите нам, если вы используете расширения, которые не работают должным образом в HoloLens 2)
- Просмотр и Пометка документа PDF
- Пространственный звук из одного окна браузера
- Автоматическое и ручное обновление браузера
- Сохранение PDF-файла из меню «Печать» (с использованием параметра «сохранить в PDF»)
- Расширение средства просмотра Вебкср и 360
- Восстановление содержимого в нужное окно при просмотре нескольких окон, помещенных в вашу среду

**Сценарии и функции, которые не должны работать:**
- Пространственный звук из нескольких окон с одновременными потоками звука
- «Видите, скажите»
- Печать

**Известные проблемы с наиболее посещаемыми браузерами:**

- Предварительная версия экранной лупы в holographic была отключена для нового Microsoft ребра. Мы надеемся включить эту функцию в будущем обновлении после правильной работы увеличения.
- Звук может воспроизводиться из неправильного окна браузера, если открыто и активно другое окно браузера. Эту ошибку можно обойти, закрыв другое активное окно, которое не предполагает воспроизведение звука.
- При воспроизведении звука из окна браузера в [режиме "следовать](hololens2-basic-usage.md#follow-me-stop-following)" звук продолжит воспроизводиться при отключении режима "следовать мне". Эту проблему можно обойти, отключив воспроизведение звука перед отключением режима "следовать мне" или закрыв окно с кнопкой " **X** ".
- Взаимодействие с активными окнами Microsoft ребра может привести к непредвиденному неактивному переходу между другими окнами 2D-приложений. Вы можете повторно активировать эти окна, взаимодействуя с ними снова.

#### <a name="microsoft-edge-insider-channels"></a>Каналы предварительной оценки Microsoft ребра

Группа Microsoft ребра предоставляет доступ к трем предварительным каналам для участников программы предварительной оценки: бета-версия, dev и ранний. Установка канала предварительного просмотра не приводит к удалению выпущенной версии Microsoft ребра в HoloLens 2, и вы можете установить более одного компонента одновременно. 

Посетите [домашнюю страницу Microsoft ребра](https://www.microsoftedgeinsider.com) , чтобы узнать больше о сообществе участников программы предварительной оценки. Чтобы узнать больше о различных каналах предварительной оценки и приступить к работе, посетите [страницу загрузки для предварительной оценки](https://www.microsoftedgeinsider.com/download).

Для установки каналов предварительной оценки Microsoft ребра в HoloLens 2 можно использовать несколько методов.

**Непосредственная установка на устройстве (в настоящее время доступно только для неуправляемых устройств)**
  1. В HoloLens 2 посетите [страницу загрузки для предварительной оценки](https://www.microsoftedgeinsider.com/download).
  1. Нажмите кнопку **скачать для HoloLens 2** , чтобы получить канал Insider Preview, который вы хотите установить.
  1. Запустите скачанный файл. msix из очереди загрузки пограничных файлов или из папки "загрузки" на устройстве (с помощью проводника).
  1. Запускается [установщик приложения](app-deploy-app-installer.md) .
  1. Нажмите кнопку **Установить**.
  1. После успешной установки вы найдете Microsoft пограничной бета-версию, dev или ранний в виде отдельной записи в списке **все приложения** меню Пуск.

**Установка через компьютер с помощью портала устройств Windows (требуется включить [режим разработчика](https://docs.microsoft.com/windows/mixed-reality/develop/platform-capabilities-and-apis/using-the-windows-device-portal#setting-up-hololens-to-use-windows-device-portal) в HoloLens 2)**
  1. На своем ПК посетите [страницу загрузки для предварительной оценки](https://www.microsoftedgeinsider.com/download).
  1. Нажмите **кнопку со стрелкой** раскрывающегося списка рядом с кнопкой "скачать для Windows 10" для канала Insider Preview, который вы хотите установить.
  1. Выберите **HoloLens 2** в раскрывающемся меню.
  1. Сохраните файл. msix в папке "Downloads" на компьютере (или в другой папке, которую можно легко найти).
  1. Используйте [портал устройств Windows](https://docs.microsoft.com/windows/mixed-reality/develop/platform-capabilities-and-apis/using-the-windows-device-portal#installing-an-app) на своем ПК, чтобы установить скачанный файл. Msix в HoloLens 2.
  1. После успешной установки вы найдете Microsoft пограничной бета-версию, dev или ранний в виде отдельной записи в списке **все приложения** меню Пуск.

#### <a name="using-wdac-to-block-new-microsoft-edge"></a>Использование WDAC для блокировки нового Microsoft ребра

Для ИТ – администраторов, которым требуется обновить [политику WDAC](windows-defender-application-control-wdac.md) , чтобы заблокировать новое приложение Microsoft ребро, необходимо добавить в политику следующее:

``` <Deny ID="ID_DENY_D_3_0" FriendlyName="C:\Data\Programs FileRule" PackageVersion="65535.65535.65535.65535" FileName="msedge.exe" /> ```

### <a name="webxr-and-360-viewer"></a>Вебкср и средство просмотра 360

Новая Microsoft ребро включает поддержку Вебкср, которая является новым стандартом для создания впечатляющих веб-приложений (замена Вебвр). Многие впечатляющие веб-работы были разработаны с учетом версии VR (они заменяют поле зрения на виртуальную среду), но эти возможности также поддерживаются в HoloLens 2. Вебкср Standard также позволяет использовать улучшенные и смешанные веб-интерфейсы, которые используют вашу физическую среду. По мере того как разработчики тратят больше времени на Вебкср, мы предполагаем, что новые улучшенные и смешанные впечатления будут поступать для клиентов HoloLens 2.

Расширение средства просмотра 360 основано на Вебкср и автоматически устанавливается вместе с новым Microsoft ребром в HoloLens 2. Это веб-расширение дает возможность создавать видео в 360-градусах. YouTube предлагает самый широкий выбор видео 360, поэтому мы рекомендуем вам начать с него.

#### <a name="how-to-use-webxr"></a>Как использовать Вебкср

1. Перейдите на веб-сайт с поддержкой Вебкср.
1. Нажмите кнопку **Ввод VR** на веб-сайте. Расположение и визуальное представление этой кнопки может различаться для каждого веб-сайта, но оно может выглядеть примерно так:

    ![Пример кнопки "вход в VR"](images/75px-enter-vr.png)

#### <a name="display-color-calibration"></a>Калибровка цвета экрана



С помощью этого нового параметра можно выбрать альтернативный цветовой профиль для экрана HoloLens 2. Это может улучшить точность цветов, особенно при низких уровнях яркости экрана. Калибровку цвета экрана можно найти в приложении "Параметры" на странице калибровки системных >.

> [!NOTE]
> Поскольку этот параметр сохраняет новый цветовой профиль в встроенном программном обеспечении, это параметр для каждого устройства (и не уникален для каждой учетной записи пользователя).

##### <a name="how-to-use-display-color-calibration"></a>Использование калибровки цвета экрана

1. Запустите приложение **параметров** и перейдите в раздел **калибровка системных >**.
1. В разделе **калибровка цвета экрана** нажмите кнопку **запустить калибровку цвета экрана** .
1. Будет запущен интерфейс калибровки цвета экрана, который позволит убедиться, что гипервизор находится в правильном положении.
1. После прохождения в диалоговых окнах инструкций дисплей будет автоматически затемнен до 30% яркости.
    > [!TIP]
    > Если у вас возникли проблемы при просмотре затемненной сцены в вашей среде, можно вручную настроить уровень яркости HoloLens 2 с помощью кнопок яркости в левой части устройства.
1. Выберите кнопки 1-6, чтобы мгновенно испытать каждый цветовой профиль, и найдите один из них, который лучше всего подходит для глаз (обычно это означает, что профиль, который помогает кадру, выглядит наиболее нейтральным, с шаблоном оттенков серого и с заданными в нем тонами.)

    ![Сцена калибровки цвета экрана](images/color-cal-ui.png)
    
1. Когда вы довольны выбранным профилем, нажмите кнопку **сохранить & выйти** .
1. Если вы предпочитаете не вносить изменения, нажмите кнопку **отмена & выйти** , и изменения будут отменены.

> [!TIP]
> Ниже приведены некоторые полезные советы, которые следует учитывать при использовании параметра калибровки цвета экрана.
> - Вы можете повторно запустить калибровку цветов экрана из параметров, когда хотите
> - Если кто-то на устройстве ранее использовал этот параметр для изменения цветовых профилей, Дата и время последнего изменения будут отражены на странице Параметры.
> - При повторном запуске калибровки цветов отображения цветовой профиль, сохраненный ранее, будет выделен, а профиль 0 не появится (как профиль 0 представляет исходный цветовой профиль экрана).
> - Если вы хотите вернуться к исходному профилю цвета экрана, это можно сделать на странице "Параметры" (см. раздел [Сброс цветового профиля](#how-to-reset-color-profile)).

##### <a name="how-to-reset-color-profile"></a>Сброс цветового профиля 

Если вы не довольны тем, как пользовательский цветовой профиль сохранен в HoloLens 2, можно восстановить исходный цветовой профиль устройства:
1. Запустите приложение **параметров** и перейдите в раздел **калибровка системных >**.
1. В разделе **калибровка цвета экрана** нажмите кнопку **сбросить к цвету профиля по умолчанию** .
1. Когда откроется диалоговое окно, выберите **перезапустить** , если будете готовы перезапустить HoloLens 2 и применить изменения.

#### <a name="swipe-to-type"></a>Прокрутка до типа

Некоторые клиенты быстрее находят слово "тип" на виртуальных клавиатурах, прожимая форму слова, которое они должны ввести, и мы предварительно используем эту функцию для работы с Holographic. #### <a name="troubleshooting-usb-c-microphones"></a>Устранение неполадок микрофонов USB-C

Имейте в виду, что некоторые микротелефоны USB-C неправильно сообщают себя как к микрофону *, так и* к докладчику. Это проблема с микрофоном, а не с HoloLens. При подключении одного из этих микрофонов к HoloLens звук может быть потерян. К счастью, существует простое исправление.  

В окне **Параметры**  ->  **системный**  ->  **звук** явным образом установите встроенные динамики **(Аналоговый звуковой драйвер аналогового компонента)** в качестве **устройства по умолчанию**. HoloLens должен запомнить этот параметр, даже если микрофон удален и повторно подключен позже.

### <a name="visitor-auto-logon-for-kiosks"></a>Автоматический вход посетителя для киосков

Эта новая функция позволяет использовать учетные записи автоматического входа в систему посетителя для полноэкранного режима.

Для конфигурации без AAD для настройки устройства для автоматического входа посетителя выполните следующие действия.

1. Создайте пакет подготовки, который:
    1. Настраивает **Параметры среды выполнения/ассигнедакцесс** для разрешения учетных записей посетителей.
    1. При необходимости регистрирует устройство в MDM **(параметры среды выполнения/Рабочая область/регистрации)** , чтобы его можно было управлять позже.
    1. Не создавать локальную учетную запись
1. [Примените пакет подготовки](hololens-provisioning.md).

Для конфигурации AAD пользователи могут в настоящее время достигать примерно такого рода без этого изменения. Устройства, присоединенные к AAD, настроенные для полноэкранного режима, могут войти в учетную запись посетителя с помощью одной кнопки на экране входа. После входа в учетную запись посетителя устройство не будет запрашивать вход снова, пока посетитель не выйдет из меню "Пуск" или не перезапускает устройство.

Для управления автовходом посетителя можно использовать [настраиваемую политику OMA-URI](https://docs.microsoft.com/mem/intune/configuration/custom-settings-windows-10) :

- Значение URI:./девице/вендор/мсфт/микседреалити/виситораутологон

| Политика  | Описание   | Конфигурации  |
|---|---|---|
| Микседреалити/Виситораутологон  | Позволяет посетителю выполнять автоматический вход в киоск   | 1 (да), 0 (нет, по умолчанию).  |

### <a name="kiosk-mode-behavior-changes-for-handling-of-failures"></a>Изменения режима киоска для обработки сбоев

В более старых сборках, если устройство имело конфигурацию киоска, которая является сочетанием глобальных назначенных прав доступа и назначенного члена группы AAD, при неудачном определении членства в группе AAD пользователь увидит меню "[ничего не отображается в начале](https://docs.microsoft.com/hololens/hololens-kiosk#kiosk-mode-behavior-changes-for-handling-of-failures)".

### <a name="auto-eye-position-support"></a>Поддержка положения автоматического взгляда

В HoloLens 2 позиции глаз обеспечивают точное позиционирование голограмм, удобство просмотра и качество отображения. Положения глаз вычисляются внутренне как часть вычислений отслеживания взгляда. Тем не менее для этого требуется, чтобы каждый пользователь продвигаться с помощью калибровки отслеживания взгляда, даже если у вас не потребуются входные данные взгляда на глаза.

**Положение с Автовзглядом (АЕП)** позволяет этим сценариям без взаимодействия вычислять положения глаз для пользователя. Автоматическое определение положения автоматически начинает работать в фоновом режиме с момента, когда пользователь помещает устройство в. Если у пользователя нет более ранней калибровки отслеживания взгляда, то положение автовзгляда будет начинаться с позиции глаз пользователя до момента обработки в 20-30 секунд. Данные пользователя не сохраняются на устройстве, поэтому этот процесс повторяется, если пользователь выходит из системы и снова помещает его в устройство, или если устройство перезагружается или выходит из спящего режима.

Существует несколько изменений поведения системы с возможностью автоматического позиционирования, когда некалиброванный пользователь помещает на устройство. В этом контексте некалиброванный пользователь обращается к пользователю, который не прошел процесс калибровки отслеживания взгляда на устройстве ранее.

| Активное приложение | Прежнее поведение | Поведение Windows holographic, версия 20H2 Update |
|:-------------------|:-----------------|:-----------------------------------|
| Приложение, не поддерживающее взгляд, или оболочка holographic |Откроется диалоговое окно с запросом на калибровку отслеживания взгляда. | Запрос не отображается. |
| Приложение с поддержкой взгляда | Откроется диалоговое окно с запросом на калибровку отслеживания взгляда. | Запрос калибровки отслеживания взгляда отображается только в том случае, если приложение обращается к потоку взгляда взгляда. |

Если пользователь переходит из приложения, не поддерживающего взгляд, в приложение, которое обращается к данным взгляда, отобразится запрос на калибровку. 

Все остальные действия системы будут похожи на то, когда у текущего пользователя нет активной калибровки отслеживания взгляда. Например, ручной жест запуска не будет включен. Для первоначальной настройки не будет внесено никаких изменений в готовый интерфейс.

Для работы, которая требует данных взгляда или очень точного позиционирования на голограмму, мы рекомендуем некалиброванным пользователям выполнять калибровку отслеживания взгляда. Он доступен из запроса на калибровку отслеживания взгляда или путем запуска приложения "Параметры" из меню "Пуск", а также выбора **системных > калибровки > глаз >** калибровки глаз.

Эти сведения можно найти позже с [другими сведениями о калибровке](hololens-calibration.md#auto-eye-position-support). 

### <a name="certificate-manager"></a>Диспетчер сертификатов

- Улучшенные средства аудита, диагностики и проверки для обеспечения безопасности устройств и обеспечения соответствия требованиям с помощью нового диспетчера сертификатов. Эта возможность позволит вам развертывать, устранять неполадки и проверять сертификаты в коммерческих средах.

В Windows holographic версии 20H2 мы добавляем диспетчер сертификатов в приложение "Параметры" HoloLens 2. Последовательно выберите **параметры > обновить & безопасность > сертификаты**. Эта функция предоставляет простой и понятный пользователю способ просмотра, установки и удаления сертификатов на устройстве. Благодаря новому диспетчеру сертификатов администраторы и пользователи теперь имеют улучшенные средства аудита, диагностики и проверки, чтобы обеспечить безопасность и соответствие устройств. 

-   **Аудит:** Возможность проверить, правильно ли развернут сертификат, или подтвердить его удаление соответствующим образом. 
-   **Диагностика:** При возникновении проблем проверка наличия соответствующих сертификатов на устройстве экономит время и помогает устранять неполадки. 
-   **Проверка:** Проверка того, что сертификат служит целью и является функциональным, может сэкономить значительное время, особенно в коммерческих средах, прежде чем развертывать сертификаты в крупном масштабе.

Чтобы быстро найти конкретный сертификат в списке, можно выполнить сортировку по имени, магазину или дате окончания срока действия. Пользователи также могут напрямую искать сертификат. Чтобы просмотреть отдельные свойства сертификата, выберите сертификат и щелкните **сведения**. 

В настоящее время установка сертификата поддерживает CER-и CRT-файлы. Владельцы устройств могут устанавливать сертификаты на локальном компьютере и у текущего пользователя.  все остальные пользователи могут устанавливаться только в текущий пользователь. Пользователи могут удалять только сертификаты, установленные непосредственно из пользовательского интерфейса параметров. Если сертификат был установлен с помощью других средств, он также должен быть удален тем же механизмом.

### <a name="auto-launch-provisioning-from-usb"></a>Автоматическое запуск подготовки с USB

- Автоматизированные процессы, обеспечивающие меньшее взаимодействие с пользователем, когда USB-диски с подготовленными пакетами используются во время OOBE.

Прежде чем пользователям выпуска пришлось запускать экран подготовки вручную во время подготовки к просмотру с помощью сочетания кнопок. Теперь пользователи могут пропускать сочетание кнопок с помощью пакета подготовки на USB-накопителе. 

1. Подключите USB-накопитель к пакету подготовки во время первого взаимодействия с приветствием OOBE.
1. Когда устройство будет готово к подготовке, оно автоматически откроет запрос на странице Подготовка. 

Примечание. Если USB-накопитель не подключен к сети, а устройство загружается, OOBE выполнит перечисление существующего USB-устройства хранения данных, а также проследит за дополнительными устройствами, подключенными к сети.

Дополнительные сведения о применении пакетов подготовки во время OOBE см. в документации по [подготовке HoloLens](hololens-provisioning.md#apply-a-provisioning-package-to-hololens-during-setup) .

Дополнительные сведения об [автоматическом запуске подготовки с помощью USB](hololens-provisioning.md#auto-launch-provisioning-from-usb) можно найти в документации по подготовке HoloLens.

### <a name="auto-confirm-provisioning-packages-in-oobe"></a>Автоматическая подтверждение пакетов подготовки в OOBE
- Автоматический процесс, допускающий меньшее взаимодействие с пользователем, когда отображается страница подготовки пакета, автоматически применяет все перечисленные пакеты.

Когда появится основной экран подготовки, OOBE выполнит отсчет 10 секунд, прежде чем автоматически начать применение всех пакетов подготовки. Пользователи по-прежнему могут [подтвердить или отменить](hololens-provisioning.md#auto-confirm-provisioning-packages-in-oobe) изменения в течение 10 секунд после проверки ожидаемых пакетов.

### <a name="automatic-provisioning-without-using-ui"></a>Автоматическая подготовка без использования пользовательского интерфейса
- Объединенные автоматические процессы для сокращения взаимодействия устройств при подготовке. 

Путем объединения автоматического запуска подготовки с устройств USB и автоматического подтверждения пакетов подготовки пользователь может автоматически подготавливать устройства HoloLens 2 без использования пользовательского интерфейса устройства или даже людьми устройства. Вы можете продолжать использовать один и тот же USB-накопитель и пакет подготовки для нескольких устройств. Это полезно для одновременного развертывания нескольких устройств в одной области. 

1. [Создайте пакет подготовки](hololens-provisioning.md) с помощью [конструктора конфигураций Windows](https://www.microsoft.com/store/productId/9NBLGGH4TX22). 
1. Скопируйте пакет на USB-накопитель.
1. Выпускайте [HoloLens 2](hololens-insider.md#ffu-download-and-flash-directions) до [19041,1361 или более новой сборки](https://aka.ms/hololens2previewdownload). 
1. После завершения [расширенного помощника по восстановлению](https://www.microsoft.com/store/productId/9P74Z35SFRS8) устройство отключается от кабеля USB-C. 
1. Подключите USB-накопитель к устройству.
1. Когда устройство HoloLens 2 загружается в OOBE, оно автоматически обнаружит пакет подготовки на USB-накопителе и запустит страницу подготовки.
1. Через 10 секунд устройство будет автоматически применять пакет подготовки. 

Теперь устройство настроено и [в нем отображается экран подготовка успешно завершена](hololens-provisioning.md#automatic-provisioning-without-using-ui).

### <a name="using-autopilot-with-wi-fi-connection"></a>Использование автопилота с Wi-Fi подключением

### <a name="tenantlockdown-csp-and-autopilot"></a>Тенантлоккдовн CSP и автопилот
- Сохраняет устройства в клиенте организации, блокируя их в клиенте даже при сбросе или восстановлении устройства. С дополнительной безопасностью, запрещая создание учетной записи при помощи подготовки. 

Устройства HoloLens 2 теперь поддерживают Тенантлоккдовн CSP в [Windows holographic версии 20H2](hololens-release-notes.md#windows-holographic-version-20h2). 

[Тенантлоккдовн](https://docs.microsoft.com/windows/client-management/mdm/tenantlockdown-csp) CSP позволяет привязывать HoloLens 2 к регистрации MDM только с помощью автопилота. После того как узел Рекуиренетворкинубе поставщика Тенантлоккдовн CSP имеет значение true или false (изначально задано) в HoloLens 2, это значение остается на устройстве, несмотря на повторную мигание, обновления ОС и т. д. 

После того как узел Рекуиренетворкинубе Тенантлоккдовн CSP установлен в значение true в HoloLens 2, OOBE ожидает успешного скачивания и применения профиля автопилота после подключения к сети. 

После того как узел Рекуиренетворкинубе Тенантлоккдовн CSP установлен в значение true в HoloLens 2, следующие операции запрещены в OOBE: 
- Создание локального пользователя с помощью подготовки среды выполнения 
- Выполнение операции присоединение к Azure AD с помощью подготовки среды выполнения 
- Выбор владельца устройства в интерфейсе OOBE 

#### <a name="how-to-set-this-using-intune"></a>Как задать это с помощью Intune? 
1. Создайте настраиваемый профиль конфигурации устройства с URI OMA и укажите значение true для узла Рекуиренетворкинубе, как показано ниже.
Значение OMA-URI должно быть равно./вендор/мсфт/тенантлоккдовн/рекуиренетворкинубе

   > [!div class="mx-imgBorder"]
   > ![Настройка блокировки пользователи клиента через OMA-URI](images/hololens-tenant-lockdown.png)

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте член устройства HoloLens 2 группы, созданной на предыдущем шаге, и запустите синхронизацию.  

Убедитесь, что конфигурация устройства успешно применена на портале Intune. После успешного применения этой конфигурации устройства на устройстве HoloLens 2 эффекты Тенантлоккдовн будут активны.

1. Создайте группу и назначьте профиль конфигурации устройства этой группе устройств. 

1. Сделайте член устройства HoloLens 2 группы, созданной на предыдущем шаге, и запустите синхронизацию.

Убедитесь, что конфигурация устройства успешно применена на портале Intune. После успешного применения этой конфигурации устройства на устройстве HoloLens 2 эффекты Тенантлоккдовн будут неактивны. 

#### <a name="what-would-happen-during-oobe-if-autopilot-profile-is-unassigned-on-a-hololens-after-tenantlockdown-was-set-to-true"></a>Что произойдет во время OOBE, если профиль для автопилота не назначен в HoloLens после того, как Тенантлоккдовн был установлен в значение true?

### <a name="cache-azure-ad-group-membership-for-offline-kiosk"></a>Кэширование членства в группах Azure AD для автономного киоска

Эта политика определяет, сколько дней кэширование членства в группах Azure AD разрешено использовать для назначенных конфигураций доступа, нацеленных на группы Azure AD для пользователя, выполнившего вход. Если значение этого параметра политики больше 0, то в противном случае используется кэш.

Min – 0 дней  
Макс. — 60 дней 

Действия по правильной работе с этой политикой: 
1. Создайте профиль конфигурации устройства для киосков, предназначенных для групп Azure AD, и назначьте его устройствам HoloLens. 
1. Создайте настраиваемую конфигурацию устройства на основе URI OMA, которая устанавливает значение этой политики в нужное число дней (> 0) и назначает его устройствам HoloLens. 
    1. Значение URI должно быть указано в текстовом поле OMA-URI как./вендор/мсфт/Полици/конфиг/микседреалити/аадграупмембершипкачевалидитиндайс
    1. Значение может находиться в диапазоне от минимального до максимального допустимого значения.
1. Регистрация устройств HoloLens и проверка того, что обе конфигурации применяются к устройству. 
1. Разрешить вход пользователя Azure AD в случае доступности в Интернете. После подтверждения успешного входа пользователя в систему и членства в группе Azure AD будет создан кэш. 
1. Теперь пользователь Azure AD 1 может перевести HoloLens в автономный режим и использовать его для полноэкранного режима, если значение политики разрешено в течение X дней. 
1. Шаги 4 и 5 могут повторяться для любого другого пользователя Azure AD N. ключевой момент здесь заключается в том, что любой пользователь Azure AD должен войти на устройство с помощью Интернета, по крайней мере, после этого можно определить, что он является членом группы Azure AD, на которую нацелена конфигурация киоска.

### <a name="research-mode"></a>Режим исследования
В режиме опроса HoloLens 2 превращается в потент средство для исследования компьютерных концепций. По сравнению с предыдущими выпусками режим исследования для HoloLens 2 имеет следующие преимущества.
-   Помимо датчиков, доступных в режиме исследовательского исследования HoloLens (1-го поколения), теперь мы предоставляем доступ к датчику иму, включая акселерометр, гироскопом и магнитометр.
-   HoloLens 2 предоставляет новые возможности, которые можно использовать совместно с режимом справочных материалов. В частности, доступ к управляемым API отслеживания и отслеживания взгляда, который обеспечивает более широкий набор экспериментов.

Исследователи теперь имеют возможность включения режима исследований на своих устройствах HoloLens для доступа ко всем внешним потокам датчиков необработанных изображений. Режим исследования для HoloLens 2 также предоставляет доступ к акселерометр, гироскопом и магнитометр. Чтобы защитить конфиденциальность пользователей, изображения камеры с отслеживанием необработанных данных недоступны в режиме исследования, но направление взгляда на глаза доступно через существующие API-интерфейсы.

### <a name="recording-length-increased"></a>Увеличение длины записи
Из-за отзывов клиентов мы увеличили продолжительность записи [смешанных записей реальность](holographic-photos-and-videos.md). Записи смешанной реальности по умолчанию не будут ограничены 5 минутами, а вместо этого будут рассчитывать максимальную длину записи в зависимости от доступного места на диске. Устройство будет оценивать максимальную продолжительность записи видео в зависимости от свободного места на диске (до 80% от общего места на диске).

> [!NOTE]
> HoloLens будет использовать длину записи видео по умолчанию (5 минут) при выполнении одного из следующих условий:
> - Оценочная максимальная продолжительность записи меньше, чем значение по умолчанию 5 минут.
> - Объем доступного места на диске составляет менее 20% от общего места на диске.

Полные требования можно найти в документации по нашим [фотографиям и видеороликам](holographic-photos-and-videos.md#maximum-recording-length) .

### <a name="support-for-windows-autopilot"></a>Поддержка автопилота Windows

Windows Автопилот для HoloLens 2 позволяет каналу продаж устройства предварительно зарегистрировать HoloLens в клиенте Intune. Когда устройства поступают, они готовы к самостоятельному развертыванию в качестве общих устройств в вашем клиенте. Чтобы воспользоваться преимуществами саморазвертывания, устройство должно подключаться к сети во время первого экрана в программе установки, используя порт USB-C-to-Ethernet.

После того как пользователь запустит процесс саморазвертывания автоматически, процесс выполняет следующие действия:

1. Присоедините устройство к Azure Active Directory (Azure AD).
1. Используйте Azure AD для регистрации устройства в Microsoft Intune (или другой службе MDM).
1. Скачайте политики для целевых устройств, сертификаты и сетевые профили.
1. Подготавливает устройство.
1. Предоставление пользователю экрана входа.

Дополнительные сведения см. в разделе [руководства по ознакомлению с Windows Автопилот для HoloLens 2](https://docs.microsoft.com/hololens/hololens2-autopilot).

*Обратитесь к менеджеру по учетным записям, чтобы присоединиться к предварительной версии для Pilot. Готовые устройства, подготовленные для автопилота, скоро начнут доставляться.*

### <a name="fido2-security-key-support"></a>Поддержка ключа безопасности FIDO2

Некоторые пользователи используют устройство HoloLens совместно с другими пользователями в рабочей или учебной среде. Поэтому важно, чтобы пользователи могли без труда вводить длинные имена пользователей и пароли. Служба быстрого удостоверения Online (FIDO) позволяет любому пользователю в вашей организации (клиенту Azure AD) легко войти в HoloLens, не вводя имя пользователя или пароль.

Ключи безопасности FIDO2 — это стандартный метод проверки подлинности без фишинга, который может приходиться на любой форм-фактор. FIDO — это открытый стандарт для проверки подлинности с паролем. Это позволяет пользователям и организациям входить в свои ресурсы без имени пользователя или пароля. Вместо этого они используют внешний ключ безопасности или ключ платформы, встроенный в устройство.

Чтобы приступить к работе, см. [раздел Включение входа с помощью ключа безопасности без пароля](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-security-key).

### <a name="improved-mdm-enrollment-via-provisioning-package"></a>Улучшенная регистрация в MDM с помощью пакета подготовки

Пакеты подготовки позволяют задать конфигурацию HoloLens с помощью файла конфигурации, а не с помощью встроенного интерфейса HoloLens. Ранее подготовка пакетов должна быть скопирована в внутреннюю память HoloLens. Теперь они могут находиться на USB-накопителе, чтобы их было проще использовать на нескольких устройствах HoloLens, а устройства можно подготавливать параллельно. Подготовка пакетов теперь также поддерживает поле для регистрации в управлении устройствами, поэтому после подготовки не выполняется ручная настройка.

Для этого:

1. Скачайте последнюю версию конструктора конфигураций Windows из Магазина Windows на компьютер.
1. Выберите **подготавливать устройства hololens** для  >  **инициализации устройств hololens 2**.
2. Создайте профиль конфигурации. Затем скопируйте все файлы, созданные на устройстве хранения USB-C.
3. Подключите устройство USB-C к любому обновленному устройству HoloLens. Затем нажмите кнопки питания для **уменьшения громкости**,  +   чтобы применить свой пакет подготовки.

### <a name="hand-tracking-improvements"></a>Усовершенствования отслеживания руки

Этот выпуск включает в себя несколько улучшений отслеживания:

- Назначится **стабильность:** Теперь система выдает изгиб пальца по индексу, когда он получает перекрыто от Palm. Это изменение повышает точность при нажатии кнопок, вводе, прокрутке содержимого и т. д. 
- **Уменьшено случайное число касаний воздуха:** Мы улучшили обнаружение жеста касания воздуха. Теперь в нескольких распространенных сценариях осталось меньше случайных активаций, например, когда вы передаете свои руки на свои стороны.
- **Надежность коммутатора пользователя:** Теперь система работает быстрее и надежнее при обновлении размера руки при совместном использовании устройства.
- **Понижение кражи руки:** Мы улучшили обработку случаев, когда датчиков имеется более двух рук. Если несколько людей работают близко друг к другу, теперь существует намного более низкая вероятность того, что отслеживание руки будет "переходить" от пользователя к руки другого человека в сцене.
- **Надежность системы:** Исправлена проблема, из-за которой отслеживание перестает работать, если устройство находится в режиме высокой нагрузки.
