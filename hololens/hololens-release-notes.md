# Заметки о выпуске HoloLens 2

Для обеспечения эффективной работы с устройствами HoloLens мы продолжаем выпусти компоненты, ошибки и обновления для системы безопасности. На этой странице вы можете увидеть новые возможности для HoloLens в каждом месяце. Чтобы получить последнюю версию устройства с полным обновлением флэш-памяти (FFU) для [флэш-памяти на устройстве с помощью расширенного помощника по восстановлению](hololens-recovery.md#clean-reflash-the-device), [скачайте его здесь](https://aka.ms/hololens2download). Загрузка актуальна и предоставляет последнюю доступную сборку, как обычно.

>[!NOTE]
> Чтобы прочитать заметки о выпуске эмулятора HoloLens, [откройте архив](https://docs.microsoft.com/windows/mixed-reality/hololens-emulator-archive).

## Windows holographic, версия 2004 — Обновление за Сентябрь 2020
- Сборка 19041,1117

Усовершенствования и исправления в обновлении.

- Устраняет ошибку, препятствующую отладке приложения в Visual Studio, если в appxmanifest присутствует SupportsMultipleInstances = "true".
- Этот выпуск включает исправление обнаружения прокси-сервера NCSI для устранения ошибки обнаружения в Интернете через сетевой прокси-сервер. NCSI может использовать прокси-сервер и прокси-сервер для каждого профиля для обнаружения подключения к Интернету. Прокси-сервер для каждого пользователя будет поддерживаться в этом выпуске NCSI в будущем.
- На большинстве устройств с Windows Mixed Reality вектор направления находится параллельно с заземлением, когда заголовком пользователя присвоена независимая должность. Тем не менее, более ранние версии HoloLens 2 выровнены по вектору, перпендикулярному панелям отображения, и, в отличие от идеальной ориентации, наклонен вниз по меньшей мере на несколько градусов. Более поздние версии HoloLens 2 исправляют это для обеспечения семантической согласованности на разных конструктивных параметрах.
- Повышенная надежность отслеживания, которая приводит к уменьшению количества потерь в определенных сценариях.
- Этот выпуск содержит исправление, позволяющее улучшить качество отметки времени звука, которое может быть обнаружено для проблем с видеозахватом.

### Поддержка Windows для автопилота

Windows Pilot для HoloLens 2 позволяет каналу продаж устройства предварительно зарегистрировать HoloLens в клиенте Intune. Когда устройства прибывают, они готовы к развертыванию как общие устройства в рамках вашего клиента. Чтобы воспользоваться преимуществами автоматического развертывания, устройство должно подключаться к сети на первом экране при установке с помощью набора USB-C-to-Ethernet или USB-C-to-LTE.

После того как пользователь запустит самоустанавливающийся самовернутый процесс, процесс завершит указанные ниже действия.

1. Присоединение устройства к Azure Active Directory (Azure AD).
1. Регистрация устройства в Microsoft Intune (или другой службе MDM) с помощью Azure AD.
1. Скачивание политик для устройств, сертификатов и сетевых профилей.
1. Подготовка устройства.
1. Демонстрация экрана входа пользователю.

Дополнительные сведения о [пробных руководствах по Windows "Автопилот" для HoloLens 2](https://docs.microsoft.com/hololens/hololens2-autopilot).

*Чтобы присоединиться к программе автопилота, обратитесь к своему менеджеру по учетным записям. Готовые устройства для автоматического пилотного развертывания скоро будут доставляться.*

### Поддержка ключа безопасности FIDO2

Некоторые пользователи в рабочей или учебной среде имеют доступ к устройству HoloLens другим пользователям. Поэтому важно, чтобы пользователи могли без проблем вводить длинные имена пользователей и пароли. Быстрая идентификация Online (FIDO). позволяет любому пользователю в вашей организации (Azure AD) легко войти в HoloLens, не вводя имя пользователя или пароль.

Ключи безопасности FIDO2 — это способ проверки подлинности без фишинга, который может быть основан на любом конструктивном интерфейсе. FIDO — это открытый стандарт для проверки подлинности с парольной учетной записью. Это позволяет пользователям и организациям входить в свои ресурсы без имени пользователя или пароля. Вместо этого они используют внешний ключ безопасности или ключ платформы, встроенный в устройство.

Чтобы приступить к работе, ознакомьтесь с [разделом Включение входа в систему ключа безопасности без пароля](https://docs.microsoft.com/azure/active-directory/authentication/howto-authentication-passwordless-security-key).

### Улучшенная регистрация в MDM с помощью пакета подготовки

Пакеты подготовки позволяют настроить конфигурацию HoloLens с помощью файла конфигурации, а не с помощью встроенного интерфейса HoloLens. Ранее пакеты подготовки пришлось скопировать в внутреннюю память HoloLens. Теперь они могут быть на USB-накопителе, чтобы их было проще использовать на нескольких устройствах HoloLens, и вы можете подготавливать устройства параллельно. Пакеты подготовки теперь также поддерживают поле для регистрации в управлении устройством, поэтому после подготовки Настройка вручную отсутствует.

Попробуйте сами:

1. Скачайте последнюю версию конструктора конфигураций Windows из Магазина Windows на компьютер.
1. Выберите **подготовить устройства hololens**для  >  **подготовки устройств hololens 2**.
2. Создайте профиль конфигурации. Затем скопируйте все файлы, созданные на устройстве хранения USB-C.
3. Подключите устройство USB-C к любому обновленному устройству с флэш-памятью. Затем нажмите кнопки **volume down**  +  **управления** электрорядом громкости, чтобы применить свой пакет подготовки.

**Поставщик служб конфигурации NetworkQoSPolicy**

Поставщик услуг конфигурации NetworkQoSPolicy создает сетевые политики качества обслуживания (QoS). Политика QoS выполняет ряд действий над сетевым трафиком, основываясь на наборе выполняемых условий. Дополнительные сведения можно найти в разделе [CSP NetworkQoSPolicy](https://docs.microsoft.com/windows/client-management/mdm/networkqospolicy-csp).

### Расширенная поддержка USB Ethernet для устройств с модемом 5G/LTE

Добавлена поддержка некоторых мобильных устройств мобильной связи, таких как 5G/LTE телефоны и Wi-Fi hotpots, когда они подключаются к HoloLens 2 через USB. Эти устройства теперь отображаются в **параметрах сети** как другое подключение Ethernet. (Устройства мобильной широкополосной связи, которым требуется внешний драйвер, не поддерживаются.) Эта функция обеспечивает подключение с высокой пропускной способностью, если Wi-Fi недоступен, и модемы Wi-Fi не работают достаточно производительно. Подробнее о поддерживаемых USB-устройствах можно узнать [в статье подключение к устройствам Bluetooth и USB – C](https://docs.microsoft.com/hololens/hololens-connect-devices).  

### Улучшенные функции отслеживания

Этот выпуск включает в себя несколько улучшений отслеживания.

- **Стабильность — наведите указатель на пункт позы.** Теперь система будет превышена изгибует указатель пальца, когда он получает Загороженная с помощью Palm. Это изменение повышает точность при нажатии кнопок, вводе, прокрутке содержимого и т. д. 
- **Уменьшение случайного касания воздуха:** Мы улучшили обнаружение жеста касания в воздухе. В некоторых распространенных сценариях, например при перетаскивании на свои стороны, наносятся меньше случайных активаций.
- Повышенная **надежность пользовательского переключателя.** Система становится более быстрой и надежной при обновлении размера при предоставлении общего доступа к устройству.
- Перенос **уменьшенной руки:** Мы улучшили обработку ситуаций, когда датчиков доступно более двух представлений. Если несколько людей работают вместе, стало еще более низким вероятность того, что отслеживаемая рука будет «переход» от пользователя к руки другого человека в сцене.
- **Надежность системы:** Устранена проблема, из – за которой отслеживание перестает работать, когда устройство находится в высоком загрузку.

### Улучшенное качество и устранение проблем

Усовершенствования и исправления, также описанные в статье обновление.  
- Добавлена активная система калибровки экрана. Эта функция повышает стабильность и выравнивание голограмм. Теперь они остаются на месте, когда вы перемещаете голову из одной стороны в другую.
- Исправлена ошибка, из-за которой поток передачи данных Wi-Fi для HoloLens был периодически прерван. Если приложение указывает на то, что требуется потоковая задержка с небольшим временем, реализуйте исправление, вызвав [функцию SetSocketMediaStreamingMode](https://docs.microsoft.com/windows/win32/api/socketapi/nf-socketapi-setsocketmediastreamingmode).
- Устранено зависание устройства, которое возникло во время потоковой передачи в режиме справочных материалов.
- Исправлено сообщение об ошибке, в котором в некоторых случаях при возобновлении сеанса на экране входа не отображается правильный пользователь.
- Устранена проблема, из которой пользователи не смогли экспортировать журналы MDM с помощью **параметров**.
- Исправлена проблема, связанная с тем, что средство отслеживания взгляда сразу после завершения настройки может быть менее ожидаемым.
- Исправлена ошибка, связанная со сбоем инициализации подсистемы отслеживания взгляда или выполнение калибровки при определенных условиях.
- Исправлена проблема, из-за которой при калибровке глаз Вам будет предложено выбрать уже откалиброванный пользователь.
- Устранена проблема, из-за которой драйвер аварийно завершает работу при калибровке глаз.
- Устранена проблема, из-за которой при нажатии на клавишу "Повтор" может произойти сбой в 60-секундном времени и сбое оболочки.
- Улучшена стабильность для буферов глубины.
- Добавлена кнопка " **поделиться** " в центре отзывов, чтобы пользователи могли проще делиться с отзывами.
- Устранена ошибка, из-за которой RoboRaid wan't установлен правильно.

### Известные проблемы

- Неполадка, связанная с системным языком zh-CN, не позволяет голосовым командам получить смешанный захват или отобразить IP-адрес устройства.
- Для устранения проблем требуется запустить приложение Кортаны после запуска устройства, чтобы использовать активацию голосовой связи "Привет, Кортана!". Если вы обновили сборку 18362, вы также можете увидеть вторую плитку приложения для предыдущей версии приложения Кортана, которая больше не работает в **начале**.
