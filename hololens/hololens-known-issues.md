### <a name="windows-device-portal"></a>Портал устройств Windows

- Функция интерактивной предварительной версии в записи смешанной реальности может демонстрировать несколько секунд задержки.

- На странице Виртуальный ввод элементы управления жестом и прокруткой в разделе Виртуальные жесты не работают. Их использование не будет действовать. Виртуальная клавиатура на виртуальной странице ввода работает правильно.

- после включения режима разработчика в Параметры для включения портала устройств может потребоваться несколько секунд.

### <a name="onedrive-camera-upload"></a>OneDriveная передача камеры

OneDriveное приложение для HoloLens не поддерживает автоматическую отправку камеры для рабочих или учебных учетных записей.

Решения.

- Если это возможно для вашего бизнеса, то в учетных записях пользователей Майкрософт поддерживается автоматическая отправка камеры. вы можете войти в учетная запись Майкрософт в дополнение к рабочей или учебной учетной записи (приложение OneDrive поддерживает двойной вход). из профиля учетная запись Майкрософт в OneDrive можно включить автоматическую отправку фоновой камеры.

- если вы не можете безопасно использовать потребительскую учетная запись Майкрософт для автоматической отправки фотографий, вы можете вручную отправить фотографии в рабочую или учебную учетную запись из приложения OneDrive. для этого убедитесь, что вы вошли в рабочую или учебную учетную запись в приложении OneDrive. Нажмите **+** кнопку и выберите **upload**. Найдите фотографии или видео, которые вы хотите передать, перейдя к **рисунку > рулон камеры**. Выберите фотографии или видео, которые нужно отправить, а затем нажмите кнопку **Открыть** .

### <a name="unable-to-connect-and-deploy-to-hololens-through-visual-studio"></a>не удается подключиться к HoloLens и выполнить развертывание в Visual Studio

> [!NOTE]
> последнее обновление: 8/8 @ 5:11 вечера-Visual Studio выпущена VS 2019 версии 16,2, которая включает исправление этой проблемы. Рекомендуется выполнить обновление до этой последней версии, чтобы избежать этой ошибки.

в Visual Studio выпущена версия VS 2019 версии 16,2, которая включает исправление этой проблемы. Рекомендуется выполнить обновление до этой последней версии, чтобы избежать этой ошибки.

проблема root-причина: пользователи, которые использовали Visual Studio 2015 или ранние выпуски Visual Studio 2017 для развертывания и отладки приложений на их HoloLens, а затем использовали последние версии Visual Studio 2017 или Visual Studio 2019 с одним и тем же HoloLens будут затронуты. новые выпуски Visual Studio развертывают новую версию компонента, но файлы из старой версии остаются на устройстве, что приводит к сбою новой версии.  Это приводит к появлению следующего сообщения об ошибке: DEP0100: Убедитесь, что на целевом устройстве включен режим разработчика. Не удалось получить лицензию разработчика \<ip\> из-за ошибки 80004005.

#### <a name="workaround"></a>Обходной путь

Наша группа сейчас работает над исправлением. В то же время можно выполнить следующие действия, чтобы устранить эту ошибку и помочь разблокировать развертывание и отладку.  

1. Запустите Visual Studio.

1. Выберите **File** > **New** > **Project** ( Файл > Создать > Проект).

1. выберите **Visual C#**  >  Windows консольное приложение для **настольных компьютеров**  >  **(платформа .NET Framework)**.

1. присвойте проекту имя (например, "хололенсдеплойментфикс") и убедитесь, что для платформы установлено по меньшей мере платформа .NET Framework 4,5, а затем нажмите кнопку **ок**.

1. Щелкните правой кнопкой мыши узел **ссылки** в Обозреватель решений и добавьте следующие ссылки (перейдите к разделу **Обзор** и выберите **Обзор**):

    ```console
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Deploy.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Connectivity.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\SirepInterop.dll
    ```

    > [!NOTE]
    > Если вы не установили 10.0.18362.0, используйте самую последнюю версию.

1. Щелкните правой кнопкой мыши проект в Обозреватель решений и выберите команду **Добавить**  >  **существующий элемент**.

1. перейдите к папке C:\Program files (x86) \ Windows Kits\10\bin\10.0.18362.0\x86 и измените фильтр на **все файлы ( \* . \* )**.

1. Выберите SirepClient.dll и SshClient.dll и нажмите кнопку **Добавить**.

1. Выберите оба файла в обозреватель решений (они должны находиться в нижней части списка файлов) и измените значение **всегда** **Копировать в выходной каталог** в окне **свойств** .

1. В верхней части файла добавьте следующий объект в существующий список `using` инструкций:

    ```console
    using Microsoft.Tools.Deploy;
    using System.Net;
    ```

1. Внутри `static void Main(...)` добавьте следующий код:

    ```PowerShell
    RemoteDeployClient client = RemoteDeployClient.CreateRemoteDeployClient();
    client.Connect(new ConnectionOptions()
    {
        Credentials = new NetworkCredential("DevToolsUser", string.Empty),
        IPAddress = IPAddress.Parse(args[0])
    });
    client.RemoteDevice.DeleteFile(@"C:\Data\Users\DefaultAccount\AppData\Local\DevelopmentFiles\VSRemoteTools\x86\CoreCLR\mscorlib.ni.dll");
    ```

1. Выберите **Сборка** > **Собрать решение**.

1. Откройте окно командной строки и компакт-диск в папку, содержащую скомпилированный файл .exe (например, К:\мипрожектс\хололенсдеплойментфикс\бин\дебуг).

1. Запустите исполняемый файл и укажите IP-адрес устройства в качестве аргумента командной строки. (При подключении по USB можно использовать 127.0.0.1, в противном случае используйте IP-адрес Wi-Fi устройства).  Например, "Хололенсдеплойментфикс 127.0.0.1".

1. после завершения работы средства без каких-либо сообщений (это займет всего несколько секунд), теперь вы сможете развертывать и отлаживать Visual Studio 2017 или более поздней версии.  Продолжение использования этого средства не является обязательным.

Мы предложим дальнейшие обновления по мере их появления.

### <a name="issues-launching-the-microsoft-store-and-apps-on-hololens"></a>проблемы при запуске Microsoft Store и приложений на HoloLens

> [!NOTE]
> Последнее обновление: 4/2 @ 10 с нерешенной проблемой.

при попытке запуска Microsoft Store и приложений на HoloLens могут возникнуть проблемы. Мы определили, что эта проблема возникает, когда фоновое обновление приложения развертывает более новую версию пакетов платформы в определенных последовательностях, в то время как одно или несколько зависимых приложений все еще работают. в этом случае автоматическое обновление приложения доставляет новую версию .NET Native Framework (версия 10.0.25531 в 10.0.27413) привела к тому, что приложения, работающие в, не будут правильно обновляться для всех работающих приложений, использующих более раннюю версию платформы.  Для обновления платформы используется следующий поток:

1. Новый пакет платформы загружается из магазина и устанавливается.

1. Все приложения, использующие более раннюю версию платформы, обновлены для использования более новой версии.

Если шаг 2 был прерван до завершения, все приложения, для которых новая инфраструктура не была зарегистрирована, не будут запущены из меню "Пуск".  мы считаем, что все приложения на HoloLens могут быть затронуты этой проблемой.

Некоторые пользователи сообщили, что закрытие зависших приложений и запуск других приложений, таких как центр отзывов, 3D Viewer или фотографии, решают проблему для них, однако это не работает 100% времени.

корень привел к тому, что эта проблема не была вызвана обновлением, но ошибка в ос, которая привела к неправильной обработке обновления .NET Native framework. Мы рады сообщить, что мы обнаружили исправление и выпустили обновление (ОС версии 17763,380), содержащее исправление.  

Чтобы узнать, может ли устройство выполнить обновление:

1. перейдите в приложение Параметры и откройте **обновление & безопасность**.

1. Выберите **проверить обновления**.

1. Если доступно обновление до 17763,380, обновите эту сборку, чтобы получить исправление ошибки зависания приложения.

1. После обновления до этой версии ОС приложения должны работать надлежащим образом.

кроме того, по мере работы с каждым выпуском HoloLens ос мы разместили образ ффу в [центре загрузки майкрософт](https://aka.ms/hololensdownload/10.0.17763.380).

если вы не хотите принимать обновление, мы выпустили новую версию Microsoft Store приложения UWP на 3/29. После создания обновленной версии хранилища:

1. Откройте магазин и убедитесь, что он загружается.
1. Чтобы открыть меню, используйте жест раскрытия.
1. Попытка открыть ранее работающие приложения.
1. Если вы по-прежнему не можете запустить его, коснитесь и удерживайте значок неработающего приложения и выберите Удалить.
1. Переустановите эти приложения из магазина.

если устройство по-прежнему не может загрузить приложения, можно загружать неопубликованные версию платформы .NET Native и среду выполнения через центр загрузки, выполнив следующие действия.

1. Скачайте [этот ZIP-файл](https://download.microsoft.com/download/8/5/C/85C23745-794C-419D-B8D7-115FBCCD6DA7/netfx_1.7.zip) из центра загрузки Майкрософт. При распаковке будут получены два файла.  Microsoft. NET. Native. Runtime. 1.7. appx и Microsoft. NET. Native. Framework. 1.7. appx.

1. Убедитесь, что устройство разблокировано в приложении dev.  если вы еще не сделали этого, ознакомьтесь [с инструкциями на портале устройств Windows](/windows/mixed-reality/using-the-windows-device-portal) .

1. затем нужно перейти на портал устройств Windows. Наша рекомендация заключается в том, чтобы сделать это через USB и сделать это, введя http://127.0.0.1:10080 в браузере.

1. после того, как на портале устройства Windows, вам нужно будет загрузить два скачанных файла. Для этого необходимо перейти на левую панель вниз, пока не появится раздел **приложения** и выберите **приложения**.

Вы можете сделать это только по одному, поэтому после выбора первого из них щелкните "Go" в разделе "развертывание". Затем сделайте это для второго APPX файла.

### <a name="xbox-wireless-controller"></a>Беспроводной контроллер Xbox

- Необходимо обновить контроллер беспроводной сети Xbox, прежде чем его можно будет использовать с HoloLens. Прежде чем пытаться связать контроллер с HoloLens, убедитесь, что у вас есть [актуальная](https://support.xbox.com/xbox-one/accessories/update-controller-for-stereo-headset-adapter) версия.

- при перезагрузке HoloLens во время подключения контроллера беспроводной сети Xbox контроллер не будет автоматически подключаться к HoloLens. Кнопка "Обзор" будет мигать медленно, пока контроллер не завершит работу через 3 минуты. Чтобы немедленно подключиться к контроллеру, выключите контроллер, нажимая кнопку "переключить", пока не будет выключена лампочка. При повторном включении контроллера снова будет подключаться к HoloLens.

- если HoloLens переходит в ждущий режим во время подключения контроллера беспроводной сети Xbox, любые входные данные на контроллере будут пробудить HoloLens. Это можно предотвратить, выключив контроллер после завершения его использования.
