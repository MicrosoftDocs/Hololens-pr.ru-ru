## Известные проблемы для всех поколений HoloLens

### Unity

- См. ["Установка средств](https://docs.microsoft.com/windows/mixed-reality/install-the-tools) для самой новой версии Unity, рекомендуемой для разработки HoloLens".
- Известные проблемы с Unity HoloLens Technical Preview описаны на форумах [HoloLens Unity.](https://forum.unity3d.com/threads/known-issues.394627/)

### Портал устройств Windows

- Функция Live Preview при захвате смешанной реальности может демонстрировать задержку в несколько секунд.

- На странице "Виртуальный ввод" элементы управления "Жесты" и "Прокрутка" в разделе "Виртуальные жесты" не работают. Использование этих методов не влияет на их использование. Виртуальная клавиатура на той же странице работает правильно.

- После включения режима разработчика в параметрах может потребоваться несколько секунд, прежде чем переключатель включит портал устройств.

### Отправка с камеры OneDrive

Приложение OneDrive для HoloLens не поддерживает автоматическую отправку камеры для учетных записей на работе или в учебном зачете.

Обходные пути:

- Если это целесообразно для вашей компании, автоматическая отправка с камеры поддерживается в учетных записях Майкрософт потребителя. Вы можете войти в свою учетную запись Майкрософт в дополнение к вашей работе или учебной учетной записи (приложение OneDrive поддерживает двойной вход). В профиле учетной записи Майкрософт в OneDrive можно включить автоматическую отправку с камеры в фоновом режиме.

- Если вы не можете безопасно использовать учетную запись Майкрософт потребителя для автоматической отправки фотографий, вы можете вручную отправить фотографии в свою учетную запись для работы или учебного заведения из приложения OneDrive. Для этого убедитесь, что вы вписались в свою учетную запись работы или учебного заведения в приложении OneDrive. Выберите **+** кнопку и выберите **"Отправить".** Найдите фотографии или видео, которые вы хотите отправить, перейдите в > **с камеры.** Выберите фотографии или видео, которые вы хотите **** отправить, а затем кнопку "Открыть".

## Известные проблемы с устройствами HoloLens 2

### Не удается запустить Microsoft Edge

Несколько клиентов сообщили о проблеме, из-за которой Microsoft Edge не удалось запустить. Для таких клиентов проблема сохраняется посредством перезагрузки и не решается обновлениями Windows или приложений. Если вы столкнулись с этой проблемой и убедитесь, что [Windows](hololens-updates.md#manually-check-for-updates)обновлена, в файле приложения ["Центр](hololens-feedback.md) отзывов" есть следующая категория и под категория: установка и обновление > Загрузка, установка и настройка Центра обновления Windows.

Не существует известных обходных пути, так как мы не могли до сих пор причину проблемы. Подача ошибки через Центр отзывов поможет нашему расследованию!

### Клавиатура не переключается на специальные символы

Во время запуска при включении компьютера существует проблема, из-за которой после того как пользователь выбрал учетную запись для работы или учебного заведения и вводит пароль, при попытке переключиться на специальные символы на клавиатуре, нажав кнопку &123, не меняется на специальные символы. 

Обойти:
-   Закрой клавиатуру и снова нажав текстовое поле.
-   Неправильно введите пароль. При следующем повторном запуске клавиатуры она будет работать в нужном случае.
- Веб-проверка подлинности, закрой клавиатуру и **выберив вход с другого устройства.** 
-   При вводе только номеров пользователь может нажать и удерживать определенные клавиши, чтобы открыть расширенное меню.
-   С помощью USB-клавиатуры.

Это не влияет на:
- Пользователи, которые решили использовать личную учетную запись.

## Известные проблемы holoLens (1-е поколения)

### Не удается подключиться к HoloLens и развернуть его с помощью Visual Studio

> [!NOTE]
> Last Update: 8/8 @ 5:11PM - Visual Studio has released VS 2019 Version 16.2 which includes a fix to this issue. Мы рекомендуем обновить эту новую версию, чтобы избежать этой ошибки.

Visual Studio выпуск vs 2019 версии 16.2, который содержит исправление этой проблемы. Мы рекомендуем обновить эту новую версию, чтобы избежать этой ошибки.

Первопричина проблемы: пользователи, которые использовали Visual Studio 2015 или ранние выпуски Visual Studio 2017 для развертывания и отладки приложений на своих HoloLens, а затем использовали последние версии Visual Studio 2017 или Visual Studio 2019 с тем же HoloLens, будут затронуты. Новые выпуски Visual Studio развертывают новую версию компонента, но файлы из более старой версии на устройстве покидают устройство, что приводит к сбойу новой версии.  Это приводит к следующему сообщению об ошибке: DEP0100: убедитесь, что на целевом устройстве включен режим разработчика. Не удалось получить лицензию разработчика из-за \<ip\> ошибки 80004005.

#### Обходной путь

В настоящее время наша команда работает над исправлением. В это время можно использовать следующие действия для решения проблемы и разблокировки развертывания и отладки:  

1. Откройте Visual Studio.

1. Выберите **новый**  >  ****  >  **проект файла.**

1. Выберите **приложение Visual C#**  >  **для консоли Windows**  >  **(.NET Framework).**

1. Придание проекту имени (например, HoloLensDeploymentFix) и убедитесь, что для framework установлено по крайней мере .NET Framework 4.5, а затем выберите **"ОК".**

1. Щелкните правой кнопкой мыши узел **"Ссылки"** в обозревателе решений и добавьте следующие ссылки (выберите в разделе **"Обзор"** и выберите **"Обзор"):**

    ```console
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Deploy.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\Microsoft.Tools.Connectivity.dll
    C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86\SirepInterop.dll
    ```

    > [!NOTE]
    > Если у вас не установлена версия 10.0.18362.0, используйте наиболее новую версию. 

1. Щелкните проект правой кнопкой мыши в обозревателе решений и выберите **пункт "Добавить**  >  **существующий элемент".**

1. Перейдите в папку C:\Program Files (x86)\Windows Kits\10\bin\10.0.18362.0\x86 и измените фильтр на **All Files (\*.\*).**

1. Выберите SirepClient.dll и SshClient.dll и выберите **"Добавить".**

1. Найдите и выберите оба файла в обозревателе решений (они должны **** находиться в нижней части списка файлов) и измените "Копировать в выходной каталог" в **окне** "Свойства" на "Копировать **всегда".**

1. В верхней части файла добавьте в существующий список выписки `using` следующее:

    ```console
    using Microsoft.Tools.Deploy;
    using System.Net;
    ```

1. Добавьте в `static void Main(...)` него следующий код:

    ```PowerShell
    RemoteDeployClient client = RemoteDeployClient.CreateRemoteDeployClient();
    client.Connect(new ConnectionOptions()
    {
        Credentials = new NetworkCredential("DevToolsUser", string.Empty),
        IPAddress = IPAddress.Parse(args[0])
    });
    client.RemoteDevice.DeleteFile(@"C:\Data\Users\DefaultAccount\AppData\Local\DevelopmentFiles\VSRemoteTools\x86\CoreCLR\mscorlib.ni.dll");
    ```

1. Выберите **решение**  >  **сборки.**

1. Откройте окно командной подсказки и компакт-диск с папкой, которая содержит скомпилировать EXE-файл (например, C:\MyProjects\HoloLensDeploymentFix\bin\Debug).

1. Запустите исполняемый исполняемый адрес и укажит IP-адрес устройства в качестве аргумента командной строки. (Если подключение подключено с помощью USB, можно использовать 127.0.0.1, в противном случае используйте Wi-Fi IP-адрес устройства.)  Например, "HoloLensDeploymentFix 127.0.0.1".

1. После выхода средства без каких-либо сообщений (это займет всего несколько секунд), вы сможете развернуть и отлалать его с Visual Studio 2017 или более новой.  Дальнейшее использование средства не требуется.

Мы будем предоставлять дополнительные обновления по мере их получения.

### Проблемы при запуске Microsoft Store и приложений на HoloLens

> [!NOTE]
> Last Update: 4/2 @ 10 AM - Issue resolved. 

При запуске Microsoft Store и приложений на HoloLens могут возникнуть проблемы. Мы определили, что проблема возникает, когда фоновые обновления приложения развертывают более новую версию пакетов framework в определенных последовательности, пока одно или несколько зависимых приложений все еще работают. В этом случае при автоматическом обновлении приложения была доставлена новая версия .NET Native Framework (с версии 10.0.25531 до 10.0.27413), из-за чего запущенные приложения не обновлялись правильно для всех запущенных приложений, потребляющих предыдущие версии этой структуры.  Поток обновления структуры: 

1. Новый пакет структуры загружается из Магазина и устанавливается.

1. Все приложения, использующие старую структуру, "обновляются" для использования более новой версии.

Если шаг 2 прерывается до завершения, все приложения, для которых не зарегистрирована новая структура, не будут запускаться из меню "Пуск".  Мы считаем, что эта проблема может затронуть любое приложение на HoloLens.

Некоторые пользователи сообщили, что закрытие приложений зависать и запуск других приложений, таких как Центр отзывов, 3D Viewer или Photos устраняет проблему для них, однако это не &mdash; работает 100 % времени.

Причиной этой проблемы было не само обновление, а ошибка в ОС, которая привела к неправильной обработке обновления .NET Native Framework. Мы рады сообщить, что мы определили исправление и выпустили обновление (ОС версии 17763.380), содержащее это исправление.  

Чтобы узнать, может ли устройство обновиться, с:

1. Перейдите в приложение "Параметры" и откройте приложение **"& Безопасность".**

1. Выберите **"Проверить обновления"**.

1. Если доступно обновление до версии 17763.380, обновите эту сборку, чтобы получить исправление ошибки "Зависать" в приложении.

1. После обновления до этой версии ОС приложения должны работать ожидаемым образом.

Кроме того, как и в каждом выпуске ОС HoloLens, мы опубликовали образ FFU в Центре загрузки [Майкрософт.](https://aka.ms/hololensdownload/10.0.17763.380)

Если вы не хотите обновлять приложение, мы выпустили новую версию приложения UWP для Microsoft Store 29.03.2019. После обновления версии Магазина:

1. Откройте Магазин и подтвердит его загрузку.
1. Используйте жест цветения, чтобы открыть меню.
1. Попытка открыть ранее неубитые приложения.
1. Если его по-прежнему не удается запускать, коснитесь и удерживайте значок разбитого приложения и выберите "Удалить".
1. Повторно установите эти приложения из Магазина.

Если вашему устройству по-прежнему не удается загрузить приложения, вы можете загрузить неогружаемую версию .NET Native Framework и следующую следующую версию в Центре загрузки:

1. Скачайте [этот ZIP-файл](https://download.microsoft.com/download/8/5/C/85C23745-794C-419D-B8D7-115FBCCD6DA7/netfx_1.7.zip) в Центре загрузки Майкрософт. При разгрузке будут создаваться два файла.  Microsoft.NET.Native.Runtime.1.7.appx и Microsoft.NET.Native.Framework.1.7.appx.

1. Убедитесь, что устройство разблокировано для разработчиков.  Если вы еще этого не сделали, см. инструкции на [портале устройств с Windows.](https://docs.microsoft.com/windows/mixed-reality/using-the-windows-device-portal)

1. Затем вам нужно ться на портал устройств с Windows. Мы пишем это по USB, и это можно сделать, введя http://127.0.0.1:10080 текст в браузере.

1. После того как вы загрузите портал устройств с Windows, вам потребуется "загрузить" неоконтруживаемые файлы, которые вы скачали. Для этого необходимо перейти вниз по левой панели, пока не перейти к разделу **"Приложения"** и выбрать **"Приложения".**

1. После этого вы увидите экран, похожий на экран ниже.  Вы хотите перейти к **** разделу "Установка приложения" и перейти к разделу, в котором вы разрезали эти два APPX-файла. Вы можете делать только по одному, поэтому после выбора первого из них нажмите кнопку "Перейти" в разделе "Развертывание". Затем сделайте это для второго APPX-файла.

   ![Портал устройств Windows для установки Side-Loaded приложения](images/20190322-DevicePortal.png)
   
1. На этом этапе мы считаем, что ваши приложения должны снова начать работать, и вы также можете ходить в Магазин.

1. В некоторых случаях необходимо выполнить дополнительный шаг запуска приложения "Просмотр трехмерных устройств", прежде чем будут запущены затронутые приложения. 

Мы ценим ваше терпение по мере того, как мы прошли процесс решения этой проблемы, и с нетерпением ждем продолжения совместной работы с нашим сообществом для успешного создания опытов смешанной реальности.

### Обновление устройства

- Через 30 секунд после нового обновления оболочка может исчезнуть один раз. Выполните жест **цветения,** чтобы возобновить сеанс.

### VisualStudio

- См. ["Установка средств"](https://docs.microsoft.com/windows/mixed-reality/install-the-tools) для самой новой версии Visual Studio, рекомендуемой для разработки HoloLens.

- При развертывании приложения из Visual Studio на HoloLens может возникнуть ошибка: запрашиваемая операция не может быть выполнена с файлом, в который открыт совместимый с пользователем **раздел. (Исключение из HRESULT: 0x800704C8)**. В этом случае попробуйте еще раз, и развертывание, как правило, будет успешным.

### API

- Если приложение задает [](https://docs.microsoft.com/windows/mixed-reality/focus-point-in-unity) точку фокуса за пользователем или норму camera.forward, голограммы не будут отображаться в фото- или видеосъемке смешанной реальности. Пока эта ошибка не исправлена в Windows, если приложения активно устанавливают точку [фокуса,](https://docs.microsoft.com/windows/mixed-reality/focus-point-in-unity) они должны убедиться, что нормальная плоскость установлена на противоположном направлении камеры (например, обычная = -camera.forward).

### Беспроводной геймпад Xbox

- Беспроводной контроллер Xbox S необходимо обновить, прежде чем его можно будет использовать с HoloLens. Перед [попыткой](https://support.xbox.com/xbox-one/accessories/update-controller-for-stereo-headset-adapter) сопряжения контроллера с HoloLens убедитесь, что вы в курсе.

- Если вы перезагружаете HoloLens, когда беспроводной контроллер Xbox подключен, контроллер не будет автоматически подключаться к HoloLens. Свет кнопки "Руководство" будет мигать медленно, пока контроллер не отключается через 3 минуты. Чтобы повторно подключить контроллер немедленно, отключите его, удерживая кнопку "Руководство", пока свет не отключается. Когда вы снова включите контроллер, он снова подключит его к HoloLens.

- Если устройство HoloLens находится в режиме ожидания во время подключения беспроводного контроллера Xbox, любые входные данные на контроллере будут выводить Устройство HoloLens в режим ожидания. Это можно предотвратить, отключив контроллер после его использования.

## Известные проблемы эмулятора HoloLens

- Не все приложения в Microsoft Store совместимы с эмулятором. Например, Юный Конкер и фрагменты недоступны для воспроизведения в эмуляторе.
- В эмуляторе нельзя использовать веб-камеру компьютера.
- Функция Live Preview портала устройств Windows не работает с эмулятором. Вы по-прежнему можете захватывать видео и изображения смешанной реальности.
