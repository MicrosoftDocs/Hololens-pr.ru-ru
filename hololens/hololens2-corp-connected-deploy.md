# <a name="deploy---corporate-connected-guide"></a>Deploy — руководство по корпоративным подключениям

Важной частью каждого развертывания является правильное настройка развертывания перед самим тестированием, чтобы обеспечить бесперебойную работу для конечного пользователя.

Поскольку мы развертываем Wi-Fi с помощью MDM, нам необходимо сначала настроить HoloLens и записать устройства в открытую сеть Wi-Fi или сеть, которая не требует сертификата. После завершения OOBE и регистрации HoloLens устройство получит сетевой сертификат и LOB, настроенные ранее, и мы сможем проверить оба полученных устройства.

После этого вы сможете подтвердить, что можете как автор, так и руководство по проверке.

## <a name="enrollment-validation"></a>Проверка регистрации

Теперь, когда все правильно настроено для регистрации Azure AD и MDM, остальное должно быть оснасткой. Вам потребуется подключение Wi-Fi и устройство HoloLens, а также одна из ранее настроенных учетных записей пользователей Azure AD.

Если ваше устройство в настоящее время не находится в состоянии заводских параметров, сейчас будет хорошее время для перезахощения [устройства.](https://docs.microsoft.com/hololens/hololens-recovery#clean-reflash-the-device)

1. После того как устройство будет в OOBE, вам потребуется приступить к взаимодействию и следуя подсказкам.

2. Подключение к открытой Wi-Fi, для подключения к Wi-Fi не требуются сертификаты. Это позволит устройству скачать сертификат, который будет использоваться в Wi-Fi организации после начальной установки.

3. Критический запрос будет, когда вас **спрашивают, кому принадлежит этот HoloLens?** Выберите **свою работу или школу, владеют ней,** и введите учетные данные учетных записей Azure AD.

4. Когда регистрация будет успешной, вам будет предложено настроить ПИН-код. Этот ПИН-код является уникальным для этого устройства для этого пользователя. Вам также будут предложены параметры сканирования радужной оболочки глаза, голосовых данных и телеметрии, и, наконец, вы сможете узнать, как открыть меню пусков и завершить OOBE.

5. Как только вы приземлитесь в доме смешанной реальности, откройте меню Пуск с помощью жеста **Пуск,** который вы только что узнали.

6. Выберите приложение **Параметры** и выберите **System**. Первая часть информации, которую вы увидите, — это имя устройства, которое для устройства HoloLens 2 будет hololENS, а затем строка из шести &quot; &quot; символов.

7. Обратите внимание на это имя.

    1.  Приложение Параметры.         ![Проверка типа "Регистрация" в Azure AD](./images/hololens2-devices-all-devices.png)

## <a name="wi-fi-certificate-validation"></a>Wi-Fi проверки сертификата

К настоящему времени устройство должно было получить сертификат Wi-Fi. Простейшая проверка, которую вы можете сделать, это попытка подключиться к подключению Wi-Fi, для которого&#39;получил сертификат. Откройте приложение **Параметры** и перейдите в **Сетевой &amp; Интернет**  ->  **Wi-Fi** и выберите подключение к Wi-Fi. После подключения откройте приложение Microsoft Edge и подтвердив, что вы можете перейти на веб-сайт.

Чтобы подтвердить, что вы получили сертификат на устройстве, можно использовать [диспетчер сертификатов.](https://docs.microsoft.com/hololens/certificate-manager)

## <a name="validate-dynamics-365-guides"></a>Проверка руководства по динамике 365

В HoloLens существуют режимы для приложения Guides, которые являются автором и операционным. Вам необходимо завершить авторство руководства перед его эксплуатацией.

### <a name="authoring-the-guide"></a>Автор руководства

Нам не нужно делать много для этой быстрой проверки. Просто выберите руководство, подготовленную на компьютере. Вам потребуется закрепить [руководство,](https://docs.microsoft.comdynamics365/mixed-reality/guides/hololens-app-anchor)для быстрой проверки можно использовать голографический якорь. После этого необходимо разместить [шаги и модели.](https://docs.microsoft.com/dynamics365/mixed-reality/guides/hololens-app-orientation)

>[!NOTE]
> Вам потребуется роль **автора** для входа на компьютер и автора на HoloLens. Роль оператора является доступной только для чтения и не имеет доступа к приложению для ПК.
