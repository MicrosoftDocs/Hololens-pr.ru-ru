# <a name="deploy---corporate-connected-guide"></a>Развертывание — интегрированное с корпоративным руководством

Важной частью каждого развертывания является обеспечение правильной настройки развертывания перед тестированием, чтобы обеспечить бесперебойную работу конечных пользователей.

Поскольку мы развертываем сертификат Wi-Fi с помощью MDM, сначала необходимо настроить HoloLens и зарегистрировать устройства в открытой Wi-Fi сети или в сети, не требующей сертификата. После того как HoloLens завершит работу OOBE и зарегистрировался, устройство получит сетевой сертификат и настроенный ранее бизнес-объект, и мы сможем проверить, были ли они получены устройством.

После этого вы сможете подтвердить создание и работу с руководством по тестированию.

## <a name="enrollment-validation"></a>Проверка регистрации

Теперь, когда все правильно настроено для регистрации Azure AD и MDM, остальное теперь должно быть привязкой. Вам потребуется подключение Wi-Fi и устройство HoloLens, а также одна из ранее настроенных учетных записей пользователя Azure AD.

Если ваше устройство в настоящее время не находится в состоянии заводской настройки, теперь пришло время переносить [устройство на флэш-память](https://docs.microsoft.com/hololens/hololens-recovery#clean-reflash-the-device).

1. После того как устройство находится в OOBE, необходимо начать взаимодействие и выполнить следующие запросы.

2. Подключитесь к открытой Wi-Fi сети, которая не требует сертификатов для присоединения к Wi-Fi. Это позволит устройству загрузить сертификат, который будет использоваться в Wi-Fi Организации после начальной настройки.

3. При появлении запроса на то **, кто владеет этим HoloLens?** Выберите **Моя рабочая или учебная** сеть и укажите учетные данные учетной записи Azure AD.

4. После успешного завершения регистрации вам будет предложено настроить ПИН-код. Этот ПИН-код уникален для этого устройства для этого пользователя. Кроме того, вы получите запрос на сканирование IRI, данные голоса и параметры телеметрии, и наконец, сможете узнать, как открыть меню "Пуск" и завершить OOBE.

5. Когда вы попадете на домашнюю страницу Mixed Reality, откройте меню Пуск с помощью только что полученного **жеста Start** .

6. Выберите приложение **Параметры** и щелкните **система**. Первый фрагмент информации, который вы увидите, — это имя устройства, которое для устройства HoloLens 2 будет содержать &quot; &quot; строку с шестью символами.

7. Запишите это имя.

    ![Экран параметров HoloLens 2](./images/hololens2-settings-about.jpg)

8. Убедитесь, что устройство успешно присоединено к Azure AD. Существует два способа.

    1.  Приложение параметров. На основе **параметров** выберите **учетные записи**  ->  **доступ к рабочей или учебной заведению**. На этом экране вы можете убедиться, что регистрация успешно выполнена, просмотрев &quot; Подключение к намеофаад&#39;s Azure AD. Соединено *yourusername@nameofAAD.onmicrosoft.com* . Это позволит проверить, присоединено ли ваше устройство к вашей организации&#39;s Azure AD.

    1. [Портал Azure](https://portal.azure.com/#home). Перейдите в **Azure Active Directory**  ->  **устройства**  ->  **все устройства** и выполните поиск по имени устройства. В разделе Тип соединения будет отображаться как "присоединение к Azure AD".
        ![Проверка типа присоединение в Azure AD](./images/hololens2-devices-all-devices.png)

9. Убедитесь, что устройство зарегистрировано в MDM. Существует два способа.

    1. На основе **параметров** выберите **учетные записи**  ->  **доступ к рабочей или учебной заведению**. На этом экране вы можете убедиться, что регистрация успешно выполнена, просмотрев &quot; Подключение к намеофаад&#39;s Azure AD. Соединено *yourusername@nameofAAD.onmicrosoft.com* . Чтобы получить доступ к рабочей или учебной учетной записи, выберите &quot; Подключение к намеофаад&#39;s Azure AD. Подключено, yourusername@nameofAAD.onmicrosoft.com &quot; и нажмите кнопку **сведения** .

    1. [Центр администрирования Microsoft Endpoint Manager](https://endpoint.microsoft.com/#home). Войдите в систему и выберите  **устройства**  , а затем  **все устройства**. Здесь можно найти имя устройства HoloLens&#39;s. Вы должны видеть HoloLens, указанный в Intune.

        ![Проверка управления с помощью Intune в Azure AD](./images/hololens2-devices-all-devices2.png)


## <a name="wi-fi-certificate-validation"></a>Проверка сертификата Wi-Fi

На данный момент устройство должно получить сертификат Wi-Fi. Простейший способ проверки — попытка подключения к Wi-Fiному подключению, для которого&#39;получен сертификат. Откройте приложение **параметров** и перейдите к **сети &amp; Интернет**  ->  **Wi-Fi** и выберите подключение Wi-Fi. После подключения откройте приложение Microsoft ребр и убедитесь, что вы можете перейти на веб-сайт.

Чтобы убедиться, что сертификат получен на устройстве, можно использовать [Диспетчер сертификатов](https://docs.microsoft.com/hololens/certificate-manager).

## <a name="validate-lob-app-install"></a>Проверка установки бизнес-приложения

Чтобы увидеть ход установки управляемого приложения, можно проверить, установлено ли приложение или проверьте параметры. Настроив бизнес-приложение в качестве обязательной установки для нашей группы, после регистрации HoloLens с пользователем в назначенной группе приложение будет автоматически скачиваться в HoloLens.

Откройте меню Пуск и выберите **все приложения**. В зависимости от количества приложений может потребоваться использовать кнопки **Page Up** или **Page Down** .

Чтобы проверить установку приложения на устройстве, это можно сделать с помощью **параметров**  ->  **учетные записи**  ->  **доступ к работе или учебные заведения**; выберите учетную запись, а затем нажмите кнопку " **сведения** " и прокрутите вниз, чтобы просмотреть различные конфигурации и приложения, применяемые к устройству из MDM.

Чтобы проверить установку из Intune, перейдите на страницу " [приложения портала MEM](https://endpoint.microsoft.com/#home)"  ->   — > все **приложения**  -> *сенамеофйоурапп*  ->  **состояние установки устройства** .

Дополнительные сведения: [развертывание приложений Intune для HoloLens](https://docs.microsoft.com/hololens/app-deploy-intune)

## <a name="validate-dynamics-365-guides"></a>Проверка руководств по Dynamics 365

Существуют режимы для приложения "руководства" в HoloLens, создания и эксплуатации. Перед началом работы с руководством вам потребуется завершить создание руководств.

### <a name="authoring-the-guide"></a>Создание руководств

Для этой быстрой проверки не нужно выполнять много усилий. Просто выберите программное обеспечение, подготовленное на компьютере. Необходимо [привязать руководство](https://docs.microsoft.comdynamics365/mixed-reality/guides/hololens-app-anchor)для быстрой проверки, в которой можно использовать привязку Holographic. После этого необходимо [разместить шаги и модели](https://docs.microsoft.com/dynamics365/mixed-reality/guides/hololens-app-orientation).

>[!NOTE]
> Вам потребуется роль **разработки** для входа на ПК и создания на HoloLens. Роль оператора доступна только для чтения и не имеет доступа к приложению для ПК.
