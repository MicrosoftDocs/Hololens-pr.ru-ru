# Развертывание и управление предприятием HoloLens 2

Этот обзор призван помочь ИТ-специалистам понять вопросы развертывания устройств Microsoft HoloLens 2 на предприятии и управления им.

HoloLens 2 работает в Windows 10 Holographic, что предоставляет организациям надежные, гибкие встроенные технологии управления мобильными устройствами и приложениями. Windows 10 Holographic поддерживает управление жизненным циклом устройств до конца, чтобы предоставить компаниям возможность управлять устройствами, данными и приложениями. HoloLens 2 можно легко включить в стандартные методики жизненного цикла, начиная с регистрации устройств, настройки и управления приложениями до обслуживания и выхода из эксплуатации с помощью комплексного решения для управления мобильными устройствами.

## Подготовка

При подготовке к развертыванию HoloLens 2 в корпоративной корпоративной среде следует учитывать и понимать некоторые моменты при планировании масштабного развертывания HoloLens 2.

### Инфраструктура essentials

Для HoloLens 2 в сценарии корпоративного развертывания на предприятии требуются некоторые основные службы инфраструктуры для поддержки всего набора возможностей. HoloLens 2 был создан с помощью [управления](https://www.microsoft.com/itshowcase/managing-windows-10-devices-with-microsoft-intune) современными мобильными устройствами для развертывания и управления. С помощью Azure AD Join + MDM в качестве основного средства достижения этой цели для постоянно увеличивающегося числа мобильных сотрудников. В приведенном ниже подмосковном обзоре представлен краткий обзор каждого компонента инфраструктуры, который следует учитывать при планировании развертывания HoloLens 2.

### Azure Active Directory
Azure AD— это облачная служба каталогов, обеспечивающая управление удостоверениями и доступом. Ее можно интегрировать с существующими локальными каталогами, создав тем самым гибридное решение для управления удостоверениями. Организации, использующие Microsoft Office 365 или Intune, уже используют Azure AD с тремя выпусками: Free, Premium P1 и Premium P2 (см. [выпуски Azure Active Directory).](https://azure.microsoft.com/documentation/articles/active-directory-editions/) Все выпуски поддерживают регистрацию устройств Azure AD, но для автоматической регистрации в MDM требуется premium P1. Для работы HoloLens 2 требуется присоединиться к Azure Active Directory, чтобы включить большинство функций и функций корпоративного уровня.

> [!NOTE]
> Локальное присоединиться к Active Directory не поддерживается на HoloLens 2.

### Управление мобильными устройствами
HoloLens 2 разработан специально для управления системами управления мобильными устройствами (MDM) в корпоративной среде. Microsoft [Intune](https://www.microsoft.com/microsoft-365/enterprise-mobility-security/microsoft-intune), часть Enterprise Mobility + Security, это облачная система MDM, которая управляет устройствами на предприятии. Как и Office 365, Intune использует Azure AD для управления удостоверениями, поэтому сотрудники используют те же учетные данные для регистрации устройств в Intune, что и для входа в Office 365. Windows 10 поддерживается несколькими системами MDM, и большинство из них поддерживает сценарии развертывания как персональных, так и корпоративных устройств. Системы MDM также могут управлять развертываниями приложений и обновлениями для HoloLens 2. Другие поставщики MDM, которые поддерживают HoloLens 2, в настоящее время включают: AirWatch, MobileIron и другие. Все поставщики систем MDM имеют одинаковый доступ к поставщикам служб конфигурации управления устройствами (CSP) Windows 10, что дает ИТ-организациям возможность выбирать, какая система лучше всего соответствует их требованиям управления, будь то Microsoft Intune или сторонний продукт MDM.

> [!NOTE]
> Традиционные системы управления на локальном компьютере, такие как System Center Configuration Manager, не поддерживаются в HoloLens 2.

### Центр обновления Windows для бизнеса
Центр обновления Windows для бизнеса, разработанный корпорацией Майкрософт, дает ИТ-администраторам дополнительные возможности управления, например возможность развертывать обновления на группах устройств и задавать временные периоды установки обновлений. Сведения об управлении обновлениями HoloLens 2 можно найти [здесь.](https://docs.microsoft.com/hololens/hololens-updates)

### Сертификаты
HoloLens 2 поддерживает развертывание сертификатов с помощью MDM, если вашей среде требуются сертификаты для проверки подлинности Corp Wi-Fi сети или доступа к другим ресурсам. Для развертывания сертификатов на HoloLens 2 могут потребоваться некоторые конфигурации инфраструктуры MDM. Узнайте, как [подготовить сертификаты и сетевые профили для HoloLens 2.](https://docs.microsoft.com/hololens/hololens-certificates-network) Подробные сведения о Intune см. [здесь.](https://docs.microsoft.com/mem/intune/protect/certificates-configure)

## Настройка

Администраторы MDM могут определять и реализовывать параметры политики на любом корпоративном устройстве, зарегистрированного в системе MDM. Какие параметры конфигурации будут отличаться в зависимости от сценария развертывания. В Windows 10 поставщики служб конфигурации (CSP) — это интерфейс для чтения, настройки, изменения или удаления параметров конфигурации на устройстве. Эти параметры сопоставляются разделам реестра или файлам. Дополнительные сведения о CSP управления устройствами с Windows 10 для HoloLens 2 см. в полном списке [CSP, поддерживаемых устройствами HoloLens.](https://docs.microsoft.com/windows/client-management/mdm/configuration-service-provider-reference#hololens)

HoloLens 2 также поддерживает настройку ограниченного набора конфигураций CSP с помощью пользовательских пакетов подготовка. Пакеты подготовка обычно используются для устройств, не управляемых MDM, и их необходимо вручную применять к каждому устройству. Дополнительные сведения о создании пользовательских пакетов подготовка можно найти [здесь.](https://docs.microsoft.com/hololens/hololens-provisioning)

> [!NOTE]
> HoloLens 2 поддерживает [Windows Autopilot,](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot)обеспечивая простой и простой процесс управления корпоративными конфигурациями устройств с Windows 10.

### Управление удостоверением

Сотрудники могут использовать только одну учетную запись для инициализации устройства, чтобы&#39;обязательно, чтобы ваша организация в первую очередь контролирует, какая учетная запись включена. Выбранная учетная запись определяет, кто контролирует устройство, и влияет на возможности управления. HoloLens 2 поддерживает 3 типа учетных записей: учетную запись локального пользователя, личную учетную запись Майкрософт и учетные записи Azure Active Directory. Настоятельно рекомендуется использовать Azure Active Directory для решения по управлению корпоративными удостоверениями, так как оно обеспечивает все возможности на устройствах HoloLens 2. Дополнительные сведения об удостоверениях на HoloLens 2 можно найти [здесь.](https://docs.microsoft.com/hololens/hololens-identity)

### Сеть и подключение

Так как HoloLens 2 — это первое облачное устройство, сетевой доступ к сетевым ресурсам необходим, чтобы сделать доступными все функции и возможности. При развертывании устройств HoloLens 2 с подключением к корпоративной сети интрасети может потребоваться обновить правила прокси-сервера или брандмауэра, чтобы разрешить доступ к облачным службам HoloLens 2. Список общих конечных точек, необходимых для операционной системы HoloLens 2, можно найти [здесь.](https://docs.microsoft.com/hololens/hololens-offline) Для успешного запуска приложений или других облачных служб на HoloLens 2 может потребоваться доступ к дополнительным конечным точкам.

Некоторые распространенные службы HoloLens 2, требующие дополнительного доступа к конечным точкам, следуют следующим образом:

- [Intune;](https://docs.microsoft.com/mem/intune/fundamentals/intune-endpoints)
- [Руководства по D365](https://support.microsoft.com/en-us/help/2655102/internet-accessible-urls-required-for-connectivity-to-microsoft-dynami)
- [D365 Remote Assist (инфраструктура O365 Teams)](https://docs.microsoft.com/office365/enterprise/urls-and-ip-address-ranges#skype-for-business-online-and-microsoft-teams)

### Развертывание сертификатов

Если для доступа к корпоративным сетям Wi-Fi или другим службам в организации требуются сертификаты, HoloLens 2 поддерживает развертывание сертификатов пользователей и устройств с помощью MDM. Примечание. Вашему решению MDM может потребоваться дополнительная конфигурация инфраструктуры для развертывания сертификатов на устройствах с Windows 10.

### Проверка безопасности

Большинству корпоративных ИТ-отделов потребуется оценка и проверка новых устройств, развертывание в корпоративной корпоративной сети. Если вашей организации требуется проверка безопасности HoloLens 2, дополнительные сведения можно найти здесь для получения утверждений [безопасности.](https://docs.microsoft.com/hololens/security-overview)

#### Аппаратные ограничения

HoloLens 2 использует современные технологии, которые включают популярные аппаратные функции, такие как камеры, микрофоны, динамики, интерфейсы USB, интерфейсы Bluetooth и Wi-Fi. Аппаратные ограничения также можно использовать, чтобы контролировать доступность этих функций.

Ниже перечислены наиболее часто используемые параметры MDM, поддерживаемые HoloLens 2 для настройки аппаратных ограничений. Некоторые из этих аппаратных ограничений обеспечивают возможность подключения и помогают защитить данные.

- [**Разрешить Wi-Fi:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-wifi#wifi-allowwifi) Возможность пользователей включить и использовать Wi-Fi на своих устройствах
- [**Разрешить USB-подключение:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-connectivity#connectivity-allowusbconnection) Включено ли USB-подключение (&#39;не влияет на зарядку USB)
- [**Разрешить Bluetooth:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-connectivity#connectivity-allowbluetooth) Возможность пользователей включить и использовать Bluetooth на своих устройствах

Узнайте больше о других [распространенных ограничениях устройств.](https://docs.microsoft.com/hololens/hololens-common-device-restrictions)

#### Профили Wi-Fi

Большинство корпоративных сетей Wi-Fi требует предоставления сертификатов и прочих сложных сведений, чтобы ограничить доступ пользователей и обеспечить его безопасность. Эти расширенные Wi-Fi сложно настроить для обычных пользователей, но системы MDM могут полностью настроить эти профили Wi-Fi без вмешательства пользователя. В системе MDM можно создать несколько профилей Wi-Fi.

Дополнительные сведения о Wi-Fi windows 10 см. в настройках [Wi-Fi профиля предприятия.](https://docs.microsoft.com/mem/intune/configuration/wi-fi-settings-windows#enterprise-profile)

#### Сертификаты

Сертификаты помогают повысить безопасность, обеспечивая проверку подлинности учетных записей, Wi-Fi проверку подлинности, шифрование VPN и SSL-шифрование веб-содержимого. Хотя администраторы могут управлять сертификатами на устройствах вручную с помощью пакетов&#39;, лучше всего использовать систему MDM для управления этими сертификатами на протяжении всего жизненного цикла ( от регистрации до продления и отзыва). Система MDM может автоматически развертывать эти сертификаты в хранилищах сертификатов&#39; устройств после регистрации устройства (если система MDM поддерживает протокол SCEP или стандарты шифрования открытого ключа #12 (PKCS#12)). MDM также может запрашивать и удалять зарегистрированные клиентские сертификаты или запускать новый запрос на регистрацию до истечения срока действия текущего сертификата.

Узнайте больше о подготовке сертификатов и [сетевых профилей для HoloLens 2.](https://docs.microsoft.com/hololens/hololens-certificates-network)

#### Proxy (Прокси)

Большинство корпоративных сетей интрасети используют прокси для управления внутренним трафиком. С помощью HoloLens 2 можно настроить прокси-сервер для подключений ethernet и Wi-Fi подключения. Эти параметры не применяются к VPN-подключениям.

Дополнительные сведения о параметрах прокси-сервера для Windows 10 см. в [CSP NetworkProxy.](https://docs.microsoft.com/windows/client-management/mdm/networkproxy-csp)

#### VPN

Организации часто используют VPN для управления доступом к приложениям и ресурсам в своей&#39;интрасети. HoloLens 2 поддерживает VPN-подключения SSL, для которых требуется подключаемый модуль для загрузки из Microsoft Store и которые связаны с поставщиком VPN по вашему выбору.

Дополнительные сведения о профилях VPN см. в разделе [Поставщик служб шифрования VPNv2](https://msdn.microsoft.com/library/windows/hardware/dn914776(v=vs.85).aspx)

#### Режим полного экрана

Устройство HoloLens 2 можно настроить для работы в качестве устройства фиксированного назначения( терминала), настроив устройство для работы в режиме терминала. Режим терминала ограничивает приложения (или пользователей), доступные на устройстве. Режим терминала — это удобная функция, которую можно использовать для выделенных устройств HoloLens 2 бизнес-приложениям или использования устройства HoloLens 2 в демонстрационной версии приложения.

Дополнительные сведения о настройке HoloLens 2 в режиме терминала см. в [настройках HoloLens в режиме терминала](https://docs.microsoft.com/hololens/hololens-kiosk)

## Развертывание

### Регистрация устройства MDM

В корпоративных развертываниях рекомендуется [](https://docs.microsoft.com/hololens/hololens-enroll-mdm) зарегистрировать устройства в MDM как корпоративные устройства только с помощью присоединить Azure AD и автоматическую регистрацию в MDM (Azure AD+MDM). Для этого требуется Azure AD Premium и поддержка автоматической регистрации для нескольких поставщиков MDM, включая Intune.

Узнайте больше о методе самостоятельной регистрации [Autopilot.](https://docs.microsoft.com/hololens/hololens2-autopilot)

### Развертывание приложений

Эффективность работы пользователей на мобильных устройствах зачастую зависит от приложений.

Windows 10 позволяет разрабатывать такие приложения, которые поддерживают работу на нескольких устройствах, используя универсальную платформу Windows (UWP) для приложений Windows.

Существует несколько способов развертывания приложений на устройствах HoloLens 2. Приложения можно развертывать напрямую через MDM, Microsoft Store для бизнеса или выгружать с помощью пакета предоставления. Дополнительные [сведения о развертывании приложений можно найти здесь.](https://docs.microsoft.com/hololens/app-deploy-overview)

> [!NOTE]
> HoloLens 2 поддерживает только запуск приложений UWP ARM64.

## Обслуживание

Желание обеспечить пользователей актуальными технологиями, характерное для ИТ-отделов организаций, должно быть согласовано с возможностью обеспечения безопасности и необходимостью контроля расходов. Так как кибератаки становятся повседневными, важно правильно поддерживать состояние устройств с Windows 10. ИТ-отделу требуются инструменты управления параметрами конфигурации, чтобы обеспечивать постоянное соответствие политикам, а также принудительно задать список устройств, у которых есть доступ к внутренним приложениям. HoloLens 2 предоставляет возможности управления мобильными операциями, необходимые для обеспечения соответствия устройств корпоративной политике.

### Параметры обслуживания ОС

**Оптимизированный процесс обновления**

Корпорация Майкрософт оптимизировала цикл разработки и выпуска продуктов таким образом, чтобы еще быстрее выпускать новые компоненты, интерфейсы и функции Windows, востребованные рынком. Корпорация Майкрософт планирует выпускать по два обновления компонентов в год (12 месяцев). **Обновления функций** устанавливают current Branch или CB и имеют связанную версию.

Корпорация Майкрософт также будет доставлять и устанавливать обновления для обеспечения безопасности и стабильности непосредственно на устройства HoloLens 2. Эти **обновления качества,** выпущенные под управлением Майкрософт через Windows Update, доступны ежемесячно. HoloLens 2 потребляет обновления функций и обновления качества в рамках одного стандартного процесса обновления.

Корпоративные клиенты могут управлять процессом обновления на HoloLens 2s с помощью системы MDM. В большинстве случаев для управления процессом обновлениям как компонентов, так и качества применяются политики. Дополнительные сведения [о настройке MDM для обновлений HoloLens.](https://docs.microsoft.com/hololens/hololens-updates)

### Управление приложениями

ИТ-администраторы могут контролировать, какие приложения разрешено устанавливать на Устройстве HoloLens 2 и как они должны быть установлены в последние сроки.

HoloLens 2 поддерживает Защитник Windows управления приложениями [(WDAC),](https://docs.microsoft.com/hololens/windows-defender-application-control-wdac)что позволяет администраторам создавать, разрешать или отопопускать списки приложений из Microsoft Store. Эта возможность также распространяется на встроенные приложения. Возможность разрешить или запретить приложения помогает гарантировать, что люди используют свои устройства для своих целей. Тем не менее, далеко не всегда удается найти баланс между требованиями безопасности и запросами и потребностями сотрудников. Для создания списков разрешенных и запрещенных приложений необходимо всегда быть в курсе новых приложений в Microsoft Store.

Дополнительные сведения [см. в CSP application Control.](https://docs.microsoft.com/windows/client-management/mdm/applicationcontrol-csp)

### Вывод из эксплуатации

Выход из устройства — это последний этап жизненного цикла устройства. Важно&#39;, что устройства, заменяемые более новыми моделями, надежно удаляются, так как вы&#39;не хотите, чтобы любые данные компании оставались на отброшенных устройствах, что может нарушить конфиденциальность ваших данных. ИТ-специалистам также требуется возможность обеспечить надлежащую поддержку тем пользователям, которые хотят уничтожить данные на устройстве в случае его утери или хищения.

HoloLens 2 поддерживает 3 метода стирки устройства

**Протирка фабрики MDM:** Возвращает образ фабрики HoloLens 2 с помощью команды MDM, инициированной администратором. Стирает все хранимые данные на устройстве.

**Сброс устройства из параметров:** Конечные пользователи могут вручную сбросить HoloLens 2 в приложении "Параметры" на устройстве. Стирает все хранимые данные на устройстве.

**Advanced Recovery Companion (ARC):** На компьютере, где запущено средство ARC, пользователь или администратор может запускать устройство HoloLens 2, подключенный к компьютеру, с помощью USB-кабеля. Стирает все хранимые данные на устройстве.
