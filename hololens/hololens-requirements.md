# <a name="hololens-2-enterprise-deployment-and-management"></a>Развертывание и управление предприятием HoloLens 2

Этот обзор предназначен для того, чтобы помочь ИТ-специалистам в понимании вопросов развертывания и управления устройствами Microsoft HoloLens 2 на предприятии.

HoloLens 2 работает в Windows 10 holographic, который предоставляет Организации надежные, гибкие, встроенные технологии управления мобильными устройствами и приложениями. Windows 10 holographic поддерживает комплексное управление жизненным циклом устройств для предоставления компаниям контроля над устройствами, данными и приложениями. HoloLens 2 можно легко включить в стандартные методы жизненного цикла, от регистрации устройств, настройки и управления приложениями до обслуживания и прекращения работы с помощью комплексного решения по управлению мобильными устройствами.

## <a name="prepare"></a>Подготовка.

При подготовке к развертыванию HoloLens 2 в корпоративной корпоративной среде необходимо ознакомиться с несколькими аспектами, которые следует рассмотреть и понять, когда вы начнете планировать масштабирование развертываний HoloLens 2.

### <a name="infrastructure-essentials"></a>Основы инфраструктуры

Для HoloLens 2 в корпоративном сценарии корпоративного развертывания существуют некоторые необходимые службы инфраструктуры, необходимые для поддержки полного набора возможностей. HoloLens 2 был создан с учетом [современного управления мобильными устройствами](https://www.microsoft.com/itshowcase/managing-windows-10-devices-with-microsoft-intune) для развертывания и управления. Присоединение к Azure AD и MDM — основное средство достижения этого в постоянно возрастающем объеме мобильных сотрудников. Приведенные ниже разделы содержат краткий обзор каждого компонента инфраструктуры, который следует учитывать при планировании развертывания для HoloLens 2.

### <a name="azure-active-directory"></a>Azure Active Directory
Azure AD — это облачная служба каталогов, которая обеспечивает управление удостоверениями и доступом. Ее можно интегрировать с существующими локальными каталогами, создав тем самым гибридное решение для управления удостоверениями. Организации, использующие Microsoft Office 365 или Intune, уже используют Azure AD с тремя выпусками: Free, Premium P1 и Premium P2 (см. [Azure Active Directory выпуски](https://azure.microsoft.com/documentation/articles/active-directory-editions/)). Все выпуски поддерживают регистрацию устройств Azure AD, но для включения автоматической регистрации MDM требуется Premium P1. HoloLens 2 требует Azure Active Directory JOIN, чтобы обеспечить большинство функций и функций корпоративного уровня.

> [!NOTE]
> Локальная Active Directory JOIN не поддерживается в HoloLens 2.

### <a name="mobile-device-management"></a>Управление мобильными устройствами
HoloLens 2 разработан специально для управления системами по управлению мобильными устройствами (MDM) в корпоративной среде. Microsoft [Intune](https://www.microsoft.com/microsoft-365/enterprise-mobility-security/microsoft-intune), часть Enterprise Mobility + Security, — это облачная система MDM, которая управляет устройствами на предприятии. Как и в Office 365, Intune использует Azure AD для управления удостоверениями, поэтому сотрудники используют одни и те же учетные данные для регистрации устройств в Intune, которые используются для входа в Office 365. Windows 10 поддерживается несколькими системами MDM, и большинство из них поддерживает сценарии развертывания как персональных, так и корпоративных устройств. Системы MDM также могут управлять развертыванием приложений и обновлениями для HoloLens 2. Другие поставщики MDM, поддерживающие HoloLens 2, в настоящее время включают: AirWatch, MobileIron и другие. Все поставщики систем MDM имеют одинаковый доступ к поставщикам службы настройки управления устройствами Windows 10 (CSP), давая ИТ-организациям возможность выбирать, какая система лучше всего соответствует требованиям к управлению, будь то Microsoft Intune или сторонний продукт MDM.

> [!NOTE]
> Традиционные системы управления компьютерами, такие как System Center Configuration Manager, не поддерживаются в HoloLens 2.

### <a name="windows-update-for-business"></a>Центр обновления Windows для бизнеса.
Центр обновления Windows для бизнеса, разработанный корпорацией Майкрософт, дает ИТ-администраторам дополнительные возможности управления, например возможность развертывать обновления на группах устройств и задавать временные периоды установки обновлений. Дополнительные сведения об управлении обновлениями HoloLens 2 см. в документации по [обновлениям hololens](https://docs.microsoft.com/hololens/hololens-updates) .

### <a name="certificates"></a>Сертификаты
HoloLens 2 поддерживает развертывание сертификатов с помощью MDM, если среда требует сертификаты для корпоративной Wi-Fi сетевой аутентификации или доступа к другим ресурсам. Для включения развертывания сертификатов в HoloLens 2 могут потребоваться некоторые конфигурации инфраструктуры MDM. Узнайте, как [подготовить сертификаты и сетевые профили для HoloLens 2](https://docs.microsoft.com/hololens/hololens-certificates-network). Если вы используете Intune, ознакомьтесь с подробными сведениями о [конфигурации сертификации](https://docs.microsoft.com/mem/intune/protect/certificates-configure) .

## <a name="configure"></a>Configure

Администраторы MDM могут определять и применять параметры политики на любом корпоративном устройстве, зарегистрированном в системе MDM. Используемые параметры конфигурации будут отличаться в зависимости от сценария развертывания. В Windows 10 поставщики службы настройки (CSP) — это интерфейс для чтения, установки, изменения или удаления параметров конфигурации на устройстве. Эти параметры сопоставляются с разделами или файлами реестра. Дополнительные сведения о Cspх управления устройствами Windows 10 для HoloLens 2 см. в полном списке [CSP, поддерживаемых на устройствах hololens](https://docs.microsoft.com/windows/client-management/mdm/configuration-service-provider-reference#hololens).

HoloLens 2 также поддерживает установку ограниченного набора конфигураций CSP с помощью настраиваемых пакетов подготовки. Пакеты подготовки обычно используются для устройств под управлением, не являющихся управляемыми MDM, и их необходимо вручную применять к каждому устройству. Дополнительные сведения о создании пользовательских пакетов подготовки см. в документации по [подготовке HoloLens](https://docs.microsoft.com/hololens/hololens-provisioning) .

> [!NOTE]
> HoloLens 2 поддерживает [Windows автопилот](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot), предоставляя простой и простой процесс управления конфигурациями устройств Windows 10 в корпоративной среде.

### <a name="identity-management"></a>Управление удостоверениями

Сотрудники могут использовать только одну учетную запись для инициализации устройства, поэтому она&#39;, чтобы ваша организация могла выбрать, какая учетная запись включена в первую очередь. Выбранная учетная запись определяет, кто контролирует устройство, и влияет на возможности управления. HoloLens 2 поддерживает 3 типа учетных записей: учетная запись локального пользователя, личная учетная запись Майкрософт и учетные записи Azure Active Directory. Настоятельно рекомендуется использовать Azure Active Directory для решения по управлению корпоративными удостоверениями, так как оно обеспечивает все возможности на устройствах HoloLens 2. Дополнительные сведения [об удостоверениях](https://docs.microsoft.com/hololens/hololens-identity) для hololens 2 см. в этой записи.

### <a name="network-and-connectivity"></a>Сеть и подключение

Так как HoloLens 2 является облачным первым устройством, сетевой доступ к сетевым ресурсам необходим для полной функциональности и возможностей. При развертывании устройств HoloLens 2 с подключением к корпоративной сети интрасети может потребоваться обновить правила прокси-сервера или брандмауэра, чтобы разрешить доступ к облачным службам HoloLens 2. Дополнительные сведения см. в списке [общих конечных точек для операционной системы HoloLens 2](https://docs.microsoft.com/hololens/hololens-offline). Для успешного выполнения в HoloLens 2 приложений или других облачных служб может потребоваться доступ к дополнительным конечным точкам.

Некоторые общие службы HoloLens 2, для которых необходим дополнительный доступ к конечной точке, приведены ниже.

- [Intune](https://docs.microsoft.com/mem/intune/fundamentals/intune-endpoints)
- [Руководства по D365](https://support.microsoft.com/en-us/help/2655102/internet-accessible-urls-required-for-connectivity-to-microsoft-dynami)

### <a name="certificate-deployment"></a>Развертывание сертификата

Если для доступа к корпоративным Wi-Fi сетям или другим службам в Организации требуются сертификаты, HoloLens 2 поддерживает развертывание сертификатов пользователей и устройств с помощью MDM. Примечание. решение MDM может потребовать дополнительной настройки инфраструктуры для развертывания сертификатов на устройствах Windows 10.

#### <a name="hardware-restrictions"></a>Аппаратные ограничения

HoloLens 2 использует современные технологии, которые включают в себя популярные аппаратные функции, такие как камеры, микрофоны, динамики, интерфейсы USB, интерфейсы Bluetooth и Wi-Fi. Аппаратные ограничения также можно использовать, чтобы контролировать доступность этих функций.

Ниже перечислены наиболее часто используемые параметры MDM, поддерживаемые HoloLens 2 для настройки ограничений оборудования. Некоторые из этих ограничений оборудования обеспечивают подключение и помогают в защите данных.

- [**Разрешить Wi-Fi:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-wifi#wifi-allowwifi) Могут ли пользователи включать и использовать радиостанции Wi-Fi на своих устройствах
- [**Разрешить USB-подключение:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-connectivity#connectivity-allowusbconnection) Включено ли USB-подключение (&#39;t влияет на зарядку по USB)
- [**Разрешить Bluetooth:**](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-connectivity#connectivity-allowbluetooth) Могут ли пользователи включать и использовать радиомодули Bluetooth на своих устройствах

Узнайте больше о других [общих ограничениях устройств.](https://docs.microsoft.com/hololens/hololens-common-device-restrictions)

#### <a name="wi-fi-profiles"></a>Профили Wi-Fi

Большинству корпоративных Wi-Fi сетей требуются сертификаты и другие сложные сведения для ограничения и защиты доступа пользователей. Эти дополнительные сведения Wi-Fi трудно настроить для обычных пользователей, но системы MDM могут полностью настраивать эти Wi-Fi профили без вмешательства пользователя. В системе MDM можно создать несколько профилей Wi-Fi.

Дополнительные сведения о параметрах Wi-Fi для Windows 10 см. в разделе [Параметры корпоративного профиля Wi-Fi](https://docs.microsoft.com/mem/intune/configuration/wi-fi-settings-windows#enterprise-profile).

#### <a name="certificates"></a>Сертификаты

Сертификаты помогают повысить безопасность, предоставляя проверку подлинности учетной записи, Wi-Fi проверку подлинности, шифрование VPN и шифрование SSL веб-содержимого. Несмотря на то, что администраторы могут управлять сертификатами на устройствах вручную с помощью пакетов подготовки,&#39;рекомендуется использовать систему MDM для управления этими сертификатами во всем жизненном цикле — от регистрации до продления и отзыва. Система MDM может автоматически развертывать эти сертификаты на устройствах&#39; хранилищах сертификатов после регистрации устройства (при условии, что система MDM поддерживает SCEP (SCEP) или стандарты шифрования с открытым ключом #12 (PKCS # 12)). MDM также может запрашивать и удалять зарегистрированные сертификаты клиентов или запускать новый запрос на регистрацию до истечения срока действия текущего сертификата.

Узнайте больше о [подготовке сертификатов и сетевых профилей для HoloLens 2.](https://docs.microsoft.com/hololens/hololens-certificates-network)

#### <a name="proxy"></a>Proxy (Прокси)

Большинство корпоративных сетей интрасети используют прокси-сервер для управления внутренним трафиком. С помощью HoloLens 2 можно настроить прокси-сервер для подключений Ethernet и Wi-Fi. Эти параметры не применяются к VPN-подключениям.

Дополнительные сведения о параметрах прокси-сервера для Windows 10 см. в разделе [НЕТВОРКПРОКСИ CSP](https://docs.microsoft.com/windows/client-management/mdm/networkproxy-csp).

#### <a name="vpn"></a>VPN

Организации часто используют VPN для управления доступом к приложениям и ресурсам в интрасети компании&#39;s. HoloLens 2 поддерживает VPN-подключения SSL, требующие загрузки подключаемого модуля из Microsoft Store и специфичные для выбранного поставщика VPN.

Дополнительные сведения о профилях VPN см. в разделе [Поставщик служб шифрования VPNv2](https://msdn.microsoft.com/library/windows/hardware/dn914776(v=vs.85).aspx)

#### <a name="kiosk-mode"></a>Режим полного экрана

Устройство HoloLens 2 можно настроить для работы в качестве устройства с фиксированной назначением, которое также называется киоском, настроив устройство для работы в полноэкранном режиме. Режим киоска ограничивает приложения (или пользователей), доступные на устройстве. Режим киоска — это удобная функция, которую можно использовать для выделения устройства HoloLens 2 для бизнес-приложений или для использования устройства HoloLens 2 в демонстрации приложения.

Дополнительные сведения о настройке HoloLens 2 в полноэкранном режиме см. в статье [Установка hololens в качестве киоска](https://docs.microsoft.com/hololens/hololens-kiosk) .

## <a name="deploy"></a>Развертывание

### <a name="mdm-device-enrollment"></a>Регистрация устройств MDM

Для корпоративных развертываний рекомендуется [регистрировать устройства](https://docs.microsoft.com/hololens/hololens-enroll-mdm) в MDM как корпоративные устройства только с присоединением к Azure AD и автоматической регистрацией MDM (Azure AD + MDM). Это требует Azure AD Premium и поддерживает автоматическую регистрацию нескольких поставщиков MDM, включая Intune.

Дополнительные сведения об автоматическом развертывании [метода автоматической](https://docs.microsoft.com/hololens/hololens2-autopilot)подготовки регистрации.

### <a name="application-deployment"></a>Развертывание приложения

Эффективность работы пользователей на мобильных устройствах зачастую зависит от приложений.

Windows 10 позволяет разрабатывать такие приложения, которые поддерживают работу на нескольких устройствах, используя универсальную платформу Windows (UWP) для приложений Windows.

Существует несколько способов развертывания приложений на устройствах HoloLens 2. Приложения можно развертывать напрямую с помощью MDM, Microsoft Store для бизнеса или загруженные неопубликованные с помощью пакета подготовки. Дополнительные сведения см. в документации по [развертыванию приложения](https://docs.microsoft.com/hololens/app-deploy-overview) .

> [!NOTE]
> HoloLens 2 поддерживает запуск только приложений ARM64 UWP.

## <a name="maintain"></a>Техническое обслуживание

Желание обеспечить пользователей актуальными технологиями, характерное для ИТ-отделов организаций, должно быть согласовано с возможностью обеспечения безопасности и необходимостью контроля расходов. Так как кибератакам стала ежедневным событием, важно правильно поддерживать состояние устройств Windows 10. ИТ-отделу требуются инструменты управления параметрами конфигурации, чтобы обеспечивать постоянное соответствие политикам, а также принудительно задать список устройств, у которых есть доступ к внутренним приложениям. HoloLens 2 предоставляет возможности управления мобильными операциями, необходимые для обеспечения соответствия устройств корпоративной политике.

### <a name="os-servicing-options"></a>Параметры обслуживания ОС

**Оптимизированный процесс обновления**

Корпорация Майкрософт оптимизировала цикл разработки и выпуска продуктов таким образом, чтобы еще быстрее выпускать новые компоненты, интерфейсы и функции Windows, востребованные рынком. Корпорация Майкрософт планирует выпускать по два обновления компонентов в год (12 месяцев). **Обновления компонентов** устанавливают Current branch или CB и имеют связанную версию.

Корпорация Майкрософт также предоставит и установит обновления для обеспечения безопасности и стабильности непосредственно на устройства HoloLens 2. Эти **обновления** , выпускаемые под контролем системы управления майкрософт через центр обновления Windows, доступны ежемесячно. HoloLens 2 использует обновления компонентов и исправлений в рамках одного стандартного процесса обновления.

Корпоративные клиенты могут управлять процессом обновления и работать в HoloLens 2S с помощью системы MDM. В большинстве случаев для управления процессом обновлениям как компонентов, так и качества применяются политики. Дополнительные сведения о [настройке MDM для обновлений HoloLens](https://docs.microsoft.com/hololens/hololens-updates).

### <a name="managing-applications"></a>Управление приложениями

ИТ-администраторы могут управлять тем, какие приложения разрешено устанавливать в HoloLens 2, и как они должны поддерживаться в актуальном состоянии.

HoloLens 2 поддерживает [Управление приложениями в Защитнике Windows (WDAC)](https://docs.microsoft.com/hololens/windows-defender-application-control-wdac), что позволяет администраторам создавать, разрешать или запрещать списки приложений из Microsoft Store. Эта возможность также распространяется на встроенные приложения. Возможность разрешать или запрещать приложения помогает гарантировать, что пользователи используют свои устройства в целях их назначения. Тем не менее, далеко не всегда удается найти баланс между требованиями безопасности и запросами и потребностями сотрудников. Для создания списков разрешенных и запрещенных приложений необходимо всегда быть в курсе новых приложений в Microsoft Store.

Дополнительные сведения см. в разделе [Application Control CSP](https://docs.microsoft.com/windows/client-management/mdm/applicationcontrol-csp).

### <a name="retire"></a>Прекратить использование

Прекращение использования устройства — это последний этап жизненного цикла устройства. &#39;важно, чтобы устройства, заменяемые более новыми моделями, были безопасно отменены, так как вы не&#39;t хотят, чтобы данные компании оставались на удаленных устройствах, которые могут нарушить конфиденциальность данных. ИТ-специалистам также требуется возможность обеспечить надлежащую поддержку тем пользователям, которые хотят уничтожить данные на устройстве в случае его утери или хищения.

HoloLens 2 поддерживает 3 метода очистки устройства

**Очистка фабрики MDM:** Сбрасывает значение HoloLens 2 к образу фабрики с помощью команды MDM, инициированной администратором. Стирает все сохраненные данные на устройстве.

**Сброс устройства с помощью параметров:** Конечные пользователи могут вручную сбросить HoloLens 2 в приложении "Параметры" на устройстве. Стирает все сохраненные данные на устройстве.

**Расширенный помощник по восстановлению (ARC):** С компьютера, на котором работает средство ARC, пользователь или администратор может мгновенно подключить HoloLens 2 к компьютеру через USB-кабель. Стирает все сохраненные данные на устройстве.
