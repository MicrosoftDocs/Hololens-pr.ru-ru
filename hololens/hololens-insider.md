### Поддержка автоматического эскзамена

В HoloLens 2 электрооборудование обеспечивает точное размещение голограмм, улучшенного просмотра и качества отображения. Позиции, объединенные в результаты отслеживания глаза, вычисляются в результате отслеживания глаза. Однако для этого требуется, чтобы каждому пользователю было проще отслеживать калибровку, даже если в нем не требуется глаза.

**Функция авторизации (AEP)** позволяет этим сценариям использовать без взаимодействия для расчета различных глаз для пользователя.  Функция авторского размещения начинается автоматически в фоновом режиме, начиная с пользователя, которое помещает устройство. Если у пользователя нет клипов с отслеживанием глаза, авторизация начнется автоматически, покоторой системе постоянно обрабатывает вечеринку по здоровью. Обычно это время обработки обычно находится в интервале от 20 до 60 секунд. Данные пользователя не хранятся на устройстве и происходят, что этот процесс повторяется, если пользователь выходит и переводит устройство или же перезагружает устройство или проигрывает спящий режим.  

При вводе некрупных пользователей на устройство работает несколько системных изменений с функцией авторизации переноса данных. Некорректированный пользователь указывает на человека, который не прошли процесс отслеживания клибровки на устройстве ранее.

 Если пользователь переходит с непосредственно в одно из приложений, которое получает доступ к анализу данных, будет отображено запрос кубров. В этом режиме интерфейс в box не изменится.

Начиная с выпуска Программы предварительной оценки Windows 19041.1361+ можно установить и удалить сертификаты непосредственно в HoloLens 2 с помощью приложения "Настройки". Установка сертификата в настоящее время поддерживает файлы CER и CRT. Владельцы устройств могут устанавливать сертификаты на локальном компьютере и текущем пользователье;  Все остальные пользователи могут установить только текущий пользователь. Пользователи могут удалять только сертификаты, установленные непосредственно из пользовательского пользовательского элемента "Параметры". Если сертификат установлен с помощью другого значения, его также необходимо удалить с помощью того же механизма.

#### Чтобы установить сертификат с помощью средства просмотра сертификатов, сделайте следующее: 
1. Перейдите в **раздел "Параметры обновления приложения"**  ->  **и**  ->  **сертификаты**безопасности, а затем выберите "Установить **сертификат".** 
1. Выберите CER-файл из интерфейса выбора файлов.
1. Выберите "Локальный компьютер" (или на другом сертификате).
1. Выберите **корневой** магазин в качестве магазина сертификата (или хранение, который вы хотите поместить сертификат). 
1. Нажмите кнопку **Установить**.

Сертификат должен быть установлен на устройстве.

#### Чтобы удалить сертификат с помощью средства просмотра сертификатов, сделайте следующее: 
1. Перейдите к **разделу "Параметры обновления**  ->  **приложения"** и  ->  **сертификаты безопасности.**
1. Найдите сертификат по имени в поле поиска.
1. Выберите сертификат.
1. Нажмите **кнопку "Удалить"**
1. При появлении запроса нажмите кнопку "Да", а затем подтвердите подтверждение.

### Tenantlockdown CSP и AutoPilo
Устройства HoloLens 2 теперь поддерживают клиентов CSP на сборке Windows 19041.1366+. 

[TenantLockdown](https://docs.microsoft.com/windows/client-management/mdm/tenantlockdown-csp) Облачные услуги облачных услуг 2 могут быть привязаны к регистрации MDM только с помощью AutoPilot. Если у злоузычка CSP требуются как truantLockdown CSP, требуются значения True или False (изучены) для holoLens 2, это значение остается на устройстве, несмежных на пересылке, обновления Хэш-аналогичного обновлений ОС и т. д. 

После установки узла CSPS locknetworkdownc" настроено значение true в HoloLens 2, OOBE ожидает неопределенного времени, пока профиль AutoPilot должен быть успешно скачан и применен. 

Если для узла клиента TenantLockdown CSPs" установлен оно true на HoloLens 2, указанные ниже операции запрещены в OOBE. 
- Создание локального пользователя с помощью подготовки среды выполнения 
- Выполнение операции соединения AAD с помощью подготовки выполнения 
- Выбор пользователей, владельцев которых принадлежит устройству в работе с OOBE

Убедитесь, что конфигурация устройства успешно применена. После применения этой программы на устройстве Hololens 2 эффекты блокировки tenantLockdown будут активны.

#### Что произойдет во время OOBE, если после неназначения параметра AutoPilot назначение tenantLockdown присваивается значение TenantLockdown?

Чтобы удалить эффекты клиента TenantLockdown, устройство необходимо сначала зарегистрировать в исходном клиенте только при использовании AutoPilot, а требуемый параметр — RequireNetworkInOOBE, как описано в предыдущем действии, прежде чем задавать все ограничения клиента, описанные в предыдущем действии.

### Кэширование членства в группе AAD для автономных киоск

Эта политика определяет количество дней, кэш участия в группе AAD, которая используется для назначенных конфигураций доступа, предназначенного для входа пользователем. Если для этого значения политики задано значение больше 0, кэш будет использоваться иначе.  

Имя: AADGroupMembershipCacheValidityInDays: ./Vendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays

Мин — 0 дней  
Макс - 60 дней 

Инструкции по правильному использованию политики: 
1. Создайте профиль конфигурации устройства для выбора групп AAD и назначьте его устройствам HoloLens. 
1. Создайте настраиваемую конфигурацию URI устройства OMA, задающую значение политики для определенного количества дней (> 0) и назначьте ее устройствам HoloLens. 
    1. URI должно быть введено в текстовое поле OMA-URI как .comendor/MSFT/Policy/Config/MixedReality/AADGroupMembershipCacheValidityInDays
    1. Значение может быть разрешено в диапазоне от min /max.
1. Регистрация устройств HoloLens и проверка применения обеих конфигураций к устройству. 
1. Разрешите вход в AAD при появлении подключения к Интернету, после успешного входа в группу AAD и участия в группе AAD кэш будет создан кэш. 
1. Теперь пользователь AAD 1 может работать в автономном режиме и использовать его в режиме киооска, если значение политики поддерживается в течение X дней. 
1. Для любого пользователя AAD повторите действия 4 и 5. Здесь необходимо выполнить вход в устройство через Интернет, чтобы определить, являются ли они участниками группы AAD, к которой приходится конфигурация Базовой доступности. 
 
> [!NOTE]
> До выполнения шага 4 пользователю AAD вы сможете поведение не отключается в средах AAD, описанных ниже в средах "отключено".
