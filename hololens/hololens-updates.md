# Управление обновлениями HoloLens

HoloLens использует Центр обновления Windows таким же образом, как другие устройства с Windows 10. Когда обновление становится доступным, оно автоматически загружается и устанавливается в следующий раз, когда устройство подключается к компьютеру и получает доступ к Интернету. В этой статье описывается управление обновлениями на предприятиях и в других управляемых средах. Сведения об управлении обновлениями для отдельных устройств HoloLens см. в статье [Обновление HoloLens](hololens-update-hololens.md).

## Автоматическое управление обновлениями

Windows Holographic for Business может использовать [Центр обновления Windows для бизнеса](https://docs.microsoft.com/windows/deployment/update/waas-manage-updates-wufb), чтобы управлять обновлениями. Все устройства HoloLens 2 могут использовать Windows Holographic for Business. Убедитесь, что используется Windows Holographic for Business сборки 10.0.18362.1042 или более поздней. Если вы используете устройства HoloLens (1-го поколения), их необходимо [обновить до Windows Holographic for Business](hololens1-upgrade-enterprise.md), чтобы можно было управлять их обновлением.

Центр обновления Windows для бизнеса подключает устройства HoloLens непосредственно к службе Центра обновления Windows. Центр обновления Windows для бизнеса позволяет управлять различными аспектами процесса обновления&mdash;. Можно выбирать, какие устройства получают обновления, и в какое время. Например, можно развернуть обновления на нескольких устройствах для тестирования, а позднее развернуть обновления на остальных устройствах. Также можно выбрать разные расписания обновления для разных типов обновлений.

> [!NOTE]  
> Для устройств HoloLens можно автоматически управлять обновлениями компонентов (выпускаются дважды в год) и исправлениями (выпускаются ежемесячно или по мере необходимости, включая критические обновления безопасности). Дополнительные сведения о типах обновлений см. в статье [Типы обновлений, которыми управляет Центр обновления Windows для бизнеса](https://docs.microsoft.com/windows/deployment/update/waas-manage-updates-wufb#types-of-updates-managed-by-windows-update-for-business).

Можно настроить параметры Центра обновления Windows для бизнеса для HoloLens с помощью политик решения для управления мобильными устройствами (MDM), такого как Microsoft Intune.

Подробное описание использования Intune для настройки Центра обновления Windows для бизнеса см. в статье [Управление обновлениями программного обеспечения Windows 10 в Intune](https://docs.microsoft.com/intune/protect/windows-update-for-business-configure).

> [!IMPORTANT]  
> Intune поддерживает два типа политик для управления обновлениями: *круги обновления Windows 10* и *обновления компонентов Windows 10*. В настоящее время тип политики обновления компонентов Windows 10 находится на стадии общедоступной предварительной версии и не поддерживается для HoloLens.
>  
> Можно управлять обновлениями HoloLens 2 с помощью политик кругов обновления Windows 10.

### Настройка политик обновления для HoloLens 2 или HoloLens (1-го поколения)

В этом разделе описываются политики, с помощью которых можно управлять обновлениями для HoloLens 2 и HoloLens (1-го поколения). Сведения о дополнительной функциональности, доступной для HoloLens 2, см. в статье [Планирование и настройка развертывания обновлений для HoloLens 2](#plan-and-configure-update-rollouts-for-hololens-2).

[Поставщик службы конфигурации политик (CSP)](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update) определяет политики настройки Центра обновления Windows для бизнеса.

> [!NOTE]  
> Сведения о конкретных политиках, поддерживающихся определенными выпусками HoloLens, см. в статье [Политики, поддерживаемые устройствами HoloLens](https://docs.microsoft.com/windows/client-management/mdm/policy-configuration-service-provider#policies-supported-by-hololens-devices).

#### Настройка автоматической проверки обновлений

Можно использовать политику **Update/AllowAutoUpdate** для управления автоматическим обновлением, включая проверку наличия обновлений, их загрузку и установку.

Эта политика поддерживает следующие значения:

- **0** — уведомить пользователя о появлении применимого к устройству обновления, готового для загрузки.
- **1** — автоматически установить обновление, затем уведомить пользователя о том, что нужно запланировать перезапуск устройства.  
- **2** — автоматически установить обновление, затем перезапустить устройство. Это рекомендуемое значение, оно используется по умолчанию.  

- **3** — автоматически установить обновление в указанное время, затем перезапустить устройство. Укажите день и время установки. Если не указать день и время, установка будет по умолчанию производиться каждый день в 3:00.  

- **4** — автоматически установить обновление, затем перезапустить устройство. В этом случае страница "Параметры" становится доступной только для чтения.

- **5** — отключить автоматические обновления.

Дополнительные сведения о доступных параметрах этой политики см. в статье [Update/AllowAutoUpdate](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-allowautoupdate).

> [!NOTE]  
> В Microsoft Intune можно изменить эту политику с помощью параметра **Поведение автоматического обновления**. Дополнительные сведения см. в статье [Управление обновления программного обеспечения в Intune](https://docs.microsoft.com/intune/windows-update-for-business-configure).

#### Настройка расписания обновления

Можно настроить порядок и время применения обновлений с помощью следующих политик:

- [Update/ScheduledInstallDay](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-scheduledinstallday).  
   - значения: **0**–**7** (0 = каждый день, 1 = воскресенье, 7 = суббота)
   - Значение по умолчанию: **0** (каждый день)
- [Update/ScheduledInstallTime](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-scheduledinstalltime).
   - Значения: 0–23 (0 = полночь, 23 = 23:00)
   - Значение по умолчанию: 3 (15:00)

#### Только для устройств под управлением Windows 10 версии 1607

Можно использовать следующие политики обновления, чтобы получать обновления из службы Windows Server Update Service (WSUS) вместо Центра обновления Windows:

- [Update/AllowUpdateService](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-allowupdateservice)
- [Update/RequireUpdateApproval](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-requireupdateapproval)
- [Update/UpdateServiceUrl](https://docs.microsoft.com/windows/client-management/mdm/policy-csp-update#update-updateserviceurl)

### Планирование и настройка развертывания обновлений для HoloLens 2

HoloLens 2 поддерживает больше возможностей автоматизации обновления, чем HoloLens (1го поколения). Это, в частности, касается использования Microsoft Intune для управления политикой Центра обновления Windows для бизнеса. Эти функции упрощают планирование и реализацию развертывания обновлений в организации.

#### Планирование стратегии обновления

Центр обновления Windows для бизнеса поддерживает политики отсрочки. После выпуска обновления корпорацией Майкрософт можно использовать политику отсрочки, чтобы определить, сколько времени ждать перед установкой этого обновления на устройства. Связывая подмножества устройств (*круги обновления*) с различными политиками отсрочки, можно скоординировать стратегию развертывания для организации.

Например, предположим, что в организации есть 1000 устройств, которые необходимо обновлять пятью способами. В этой организации можно создать пять кругов обновления, как показано в следующей таблице.

|Группа |Количество устройств |Отсрочка (дни) |
| ---| :---: | :---: |
|Группа 1 (ИТ-персонал) |5 |0 |
|Группа 2 (ранние последователи) |50 |60 |
|Группа 3 (основная 1) |250 |120 |
|Группа 4 (основная 2) |300 |150 |
|Группа 5 (основная 3) |395 |180 |

Вот как развертывание будет последовательно осуществляться во всей организации.

![Сроки развертывания обновлений](./images/hololens-updates-timeline.png)

#### Настройка политики отсрочки обновлений

Политика отсрочки определяет количество дней между датой выпуска обновления и датой, когда будет предложено установить это обновление на устройство.

Для обновлений компонентов и исправлений можно настроить разные значения отсрочки. В следующей таблице перечислены определенные политики для каждого типа и указано максимальное время отсрочки.

|Категория |Политика |Максимальная отсрочка |
| --- | --- | --- |
|Обновления компонентов |DeferFeatureUpdatesPeriodInDays |365 дней |
|Исправления |DeferQualityUpdatesPeriodInDays |30 дней |

####  Примеры: управление обновлениями с помощью Intune

**Пример 1. Создание и назначение круга обновления.**

Дополнительные сведения об этом примере см. в статье [Создание и назначение кругов обновления](https://docs.microsoft.com/mem/intune/protect/windows-update-for-business-configure#create-and-assign-update-rings).

1. Войдите в [Центр администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) и перейдите к вашим профилям Intune.
1. Выберите **Обновления программного обеспечения** > **Круги обновления Windows 10** > **Создать**.
1. В разделе **Основные сведения** укажите имя и описание (не обязательно), затем нажмите кнопку **Далее**.
1. В разделе **Параметры круга обновления** для параметра **Канал обслуживания** выберите значение **Semi-Annual Channel**, затем измените значение параметра **Период отсрочки обновления компонентов** на **120**. Нажмите кнопку **Далее**.
1. В разделе **Назначения** выберите **+ Выберите группы для включения** и назначьте круг обновления одной или нескольким группам. Для точной настройки назначения используйте функцию **+ Выберите группы для исключения**. Нажмите кнопку **Далее**.
1. В разделе **Проверка и создание** проверьте параметры. Когда все будет готово для сохранения конфигурации круга обновления, выберите **Создать**.

Теперь список кругов обновления включает новый круг обновления для Windows 10.

**Пример 2. Приостановка круга обновления.**

Если при развертывании обновления компонента или исправления возникает проблема, можно приостановить обновление на срок до 35 дней (начиная с указанной даты). При этом обновление не будет устанавливаться на другие устройства, пока вы не сможете решить или устранить проблему. Если приостановить обновление компонента, исправления будут по-прежнему предлагаться для устройств, чтобы гарантировать их безопасность. После истечения указанного времени срок действия приостановки автоматически истекает. Процесс обновления возобновится.

Чтобы приостановить круг обновления в Intune, выполните следующие действия.

1. На странице обзора круга обновления нажмите **Приостановить**.
1. Выберите тип обновления (**обновление компонента** или **исправление**), которое нужно приостановить, затем нажмите кнопку **ОК**.

Если обновление определенного типа приостановлено, в области "Обзор" этого круга обновления показано, сколько осталось дней до возобновления.

Пока круг обновления приостановлен, можно выбрать один из следующих вариантов:

- Чтобы продлить период приостановки для какого-либо типа обновления на 35 дней, выберите **Продлить**.
- Чтобы возобновить развертывание для этого круга обновления, выберите **Возобновить**. При необходимости можно снова приостановить круг обновления.

> [!NOTE]  
> Действие **Удалить** для кругов обновления не поддерживается для устройств HoloLens 2.

## Проверка обновлений вручную

Устройства HoloLens периодически проверяют наличие обновлений системы, но в определенных обстоятельствах может потребоваться проверять наличие обновлений вручную.

Чтобы вручную проверить наличие обновлений, выберите **Параметры** > **Обновления и безопасность** > **Проверить обновления**. Если в приложении "Параметры" указано, что устройство обновлено, это означает, что установлены все доступные обновления.

## Откат обновления вручную

В некоторых случаях может потребоваться вернуться на предыдущую версию программного обеспечения HoloLens. Эта процедура зависит от типа устройства: HoloLens 2 или HoloLens (1-го поколения).

### Возврат к предыдущей версии (HoloLens 2)

Можно откатить обновления и вернуться к предыдущей версии HoloLens 2, воспользовавшись приложением Advanced Recovery Companion для сброса HoloLens на более раннюю версию.

> [!NOTE]
> При возврате к более ранней версии личные файлы и параметры будут удалены.

Для возврата к предыдущей версии HoloLens 2 выполните следующие действия:

1. Убедитесь, что к компьютеру не подключены телефоны или другие устройства с Windows.
1. На компьютере загрузите приложение [Advanced Recovery Companion](https://www.microsoft.com/p/advanced-recovery-companion/9p74z35sfrs8?activetab=pivot:overviewtab) из Microsoft Store.
1. Загрузите [последний выпуск HoloLens 2](https://aka.ms/hololens2download).
1. По окончании загрузки откройте **Проводник** > **Загрузки**, щелкните правой кнопкой мыши сжатую ZIP-папку и выберите **Извлечь все** > **Извлечь**, чтобы распаковать файл.
1. Подключите устройство HoloLens к компьютеру кабелем USB-A — USB-C. Кабели такого типа подходят лучше всего, даже если вы раньше подключали HoloLens другими кабелями.
1. Программа Advanced Recovery Companion автоматически обнаружит устройство HoloLens. Выберите плитку **Microsoft HoloLens**.
1. На следующем экране выберите **Выбрать пакет вручную** и откройте папку, развернутую ранее. 
1. Выберите файл установки (с расширением .FFU).
1. Выберите **Установить программное обеспечение** и следуйте инструкциям.

### Возврат к предыдущей версии (HoloLens (1-го поколения))

Можно откатить обновления и вернуться к предыдущей версии HoloLens (1-го поколения), воспользовавшись приложением Windows Device Recovery Tool для сброса HoloLens на более раннюю версию.

> [!NOTE]
> При возврате к более ранней версии личные файлы и параметры будут удалены.

Для возврата к предыдущей версии HoloLens (1-го поколения) выполните следующие действия:

1. Убедитесь, что к компьютеру не подключены телефоны или другие устройства с Windows.
1. На компьютере загрузите приложение [Windows Device Recovery Tool (WDRT)](https://support.microsoft.com/help/12379).
1. Загрузите пакет восстановления [HoloLens Anniversary Update](https://aka.ms/hololensrecovery).
1. По окончании загрузки откройте **Проводник** > **Загрузки**, щелкните правой кнопкой мыши сжатую ZIP-папку и выберите **Извлечь все** > **Извлечь**, чтобы распаковать файл.
1. Подключите устройство HoloLens к компьютеру кабелем micro-USB, который входит в комплект устройства HoloLens. Этот кабель подходит лучше всего, даже если вы раньше подключали HoloLens другими кабелями.
1. Приложение WDRT автоматически обнаружит устройство HoloLens. Выберите плитку **Microsoft HoloLens**.
1. На следующем экране выберите **Выбрать пакет вручную** и откройте папку, развернутую ранее. 
1. Выберите файл установки (с расширением .FFU).
1. Выберите **Установить программное обеспечение** и следуйте инструкциям.

> [!NOTE]
> Если WDRT не обнаруживает устройство HoloLens, попробуйте перезагрузить компьютер. Если это не поможет, выберите **Мое устройство не обнаружено**, выберите **Microsoft HoloLens** и следуйте инструкциям.
